partiqlFile

partiqlFile is a in the DynamoDB extension that allows you to specify PartiQL statements in an external file. In DynamoDB s, use partiqlFile instead of sqlFile.

Uses

partiqlFile is useful for complex changes that are not supported through automated s such as stored procedures. The PartiQL contained in the file can be multi-line.

Run partiqlFile

Available s

Name Type Description Requirement
dbms String Optional
encoding String Optional
endDelimiter String Optional
path String Specifies the file path of the PartiQL file to load. Optional
relativeToChangelogFile Boolean Optional
splitStatements Boolean Optional
stripComments Boolean Optional

<databaseChangeLog
    xmlns="http://www.liquibase.org/xml/ns/dbchangelog"
    xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
    xmlns:dynamodb-pro="http://www.liquibase.org/xml/ns/pro-dynamodb"
    xsi:schemaLocation="http://www.liquibase.org/xml/ns/dbchangelog
    http://www.liquibase.org/xml/ns/dbchangelog/dbchangelog-latest.xsd
    http://www.liquibase.org/xml/ns/pro-dynamodb
    http://www.liquibase.org/xml/ns/pro-dynamodb/liquibase-pro-dynamodb-latest.xsd">

    <changeSet id="1" author="your.name">
        <dynamodb-pro:createDynamoTable tableName="MusicPartiqlFileTest">
            <dynamodb-pro:keySchema hashAttributeName="Artist" rangeAttributeName="SongTitle"/>
            <dynamodb-pro:attributeDefinitions>
                <dynamodb-pro:attributeDefinition attributeName="Artist" attributeType="S"/>
                <dynamodb-pro:attributeDefinition attributeName="SongTitle" attributeType="S"/>
            </dynamodb-pro:attributeDefinitions>
            <dynamodb-pro:provisionedThroughput readCapacityUnits="10" writeCapacityUnits="10"/>
        </dynamodb-pro:createDynamoTable>

        <rollback>
            <dynamodb-pro:deleteDynamoTable tableName="MusicPartiqlFileTest"/>
        </rollback>
    </changeSet>

    <changeSet id="2" author="your.name">
        <dynamodb-pro:partiqlFile path="test.partiql" splitStatements="true" endDelimiter="@@"/>

        <rollback>
            <dynamodb-pro:partiqlFile path="test-rollback.partiql" splitStatements="true" endDelimiter="@@"/>
        </rollback>
    </changeSet>

</databaseChangeLog>

PartiQL file contents

In this example, you can use the following PartiQL for test.partiql:

INSERT INTO MusicPartiqlFileTest value {'Artist' : 'Taras Chubay','SongTitle' : 'Вона'} @@ INSERT INTO MusicPartiqlFileTestXML value {'Artist' : 'Alan Walker','SongTitle' : 'Faded'}

You can use the following PartiQL for test-rollback.partiql:

DELETE FROM MusicPartiqlFileTest WHERE Artist='Taras Chubay' AND SongTitle='Вона' @@ DELETE FROM MusicPartiqlFileTestXML WHERE Artist='Alan Walker' AND SongTitle='Faded'

Database support

This is only supported for Amazon DynamoDB.
