insert

The insert inserts data into an existing table.

Uses

You can typically use the insert when you want to add and deploy the specific data or values to the existing table in your database. It can include columns and another s listed in this documentation.

Running the update

To insert the needed data and update a table in your database, follow these steps:

Step 1: Add the insert to your with the needed s as it is shown in the examples.

Step 2: Deploy your by running the update command.

liquibase update

Now, you should see that your data has been inserted into the table.

Available s

Name Description Required for Supports Since
catalogName The name of the catalog. all 3.0
dbms The logical expression of database type(s) on which the change must be applied. Valid database type names are listed on the supported databases page.It can be a comma-separated list of multiple databases or you can also specify that a change is not applicable to a particular database type by prefixing with !. The keywords all and none are also available. all 3.0
schemaName The name of the schema. all
tableName The name of the table for which to insert data. all all

Nested properties

Name Description Required for Supports Multiple allowed
columns The data to insert into columns. See Information about the <column> tag section for more information.   all all
XML example 
<changeSet  author="liquibase-docs"  id="insert-example">  
    <insert  catalogName="department"  
            dbms="!h2, mysql"  
            schemaName="public"  
            tableName="person">  
        <column  name="address"  value="address value"/>  
    </insert>  
</changeSet>
 YAML example 
changeSet:  
  id:  insert-example  
  author:  liquibase-docs  
  changes:  
  -  insert:  
      catalogName:  department  
      columns:  
      -  column:  
          name:  address  
          value:  address value  
      dbms:  '!h2,  mysql'  
      schemaName:  public  
      tableName:  person
