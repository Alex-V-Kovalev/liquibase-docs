dropPrimaryKey

The dropPrimaryKey drops an existing primary key in the table.

Uses

You can typically use the dropPrimaryKey when you want to remove a field that uniquely defines a record in your database. It can include catalogName and another s listed in this documentation.

Running the dropPrimaryKey

To drop an existing primary key, follow these steps:

Step 1: Add the dropPrimaryKey to your with the needed s as it is shown in the examples.

Step 2: Deploy your by running the update command.

liquibase update

Now, you should see that the information indicating your primary key is removed from the table.

Available s

Name Description Required for Supports Since
catalogName The name of the catalog. all
constraintName The name of the primary key. firebird, informix, sybase all
dropIndex The index to drop. all
schemaName The name of the schema. all
tableName The name of the table from which you drop the primary key. all all

XML example

<changeSet  author="liquibase-docs"  id="dropPrimaryKey-example">  
    <dropPrimaryKey  catalogName="department"  
            constraintName="const_name"  
            dropIndex="true"  
            schemaName="public"  
            tableName="person"/>  
</changeSet>

YAML example

changeSet:  
  id:  dropPrimaryKey-example  
  author:  liquibase-docs  
  changes:  
  -  dropPrimaryKey:  
      catalogName:  department  
      constraintName:  const_name  
      dropIndex:  true  
      schemaName:  public  
      tableName:  person

SQL example

-- changeset DocUser:1:dropPrimaryKeyChangeType
ALTER  TABLE  department.person  DROP  PRIMARY  KEY;
