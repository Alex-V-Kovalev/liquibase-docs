createView (Databricks)

createView is a in the Databricks extension that creates a view. It extends the (non-Databricks) createView  with Databricks-specific s.

Uses

You can use this to create a view on one or more tables. This works the same way as the change for createView, just with some extra s for Databricks, like tblProperties. This lets you specify additional properties on your Databricks views.

In the source code, the Databricks change to create a view is implemented as a separate than the non-Databricks change. However, they share most of the same s:

  • In YAML and JSON s, there is no difference in syntax. automatically determines which implementation to use depending on whether you specify Databricks s.
  • In XML s, you must specify a databricks: prefix to use Databricks s. Otherwise, throws an error.

databaseChangeLog:
  - changeSet:
      id: 2
      author: your.name
      changes:
        - createView:
            selectQuery: SELECT id, first_name, last_name, email FROM authors
            viewName: test_view
            tblProperties: 'this.is.my.key'=12,'this.is.my.key2'=true
      rollback:
        - dropView:
            viewName: test_view
