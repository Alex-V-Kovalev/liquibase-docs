update-count

The update-count command sequentially updates a specified number of s on your database.

Uses

The update-count command is mainly used when you want to apply changes and update s sequentially, starting with the s from the top of the file until the number specified is reached.

The image below shows four s: a, b, c, and d. As you can see, a is already deployed on the database. So, running the command update-count 2, deploys b and c without applying d.

Additionally, the best practice is to run the update-count-sql helper command because it allows you to inspect the update-count SQL, so you can correct any issues that may arise before running the command.

Running the update-count command

To run the update-count command, specify the driver, classpath, and URL in the properties file. For more information, see Specifying Properties in a Connection Profile. You can also specify them from the command line.

Then run the update-count command:

liquibase --changelog-file=myChangelog.xml update-count 2

Enter the name of the and number you want to use in place of myChangelog.xml and 2.

update-count global s

Definition Requirement
--changelog-file * The root Required
--url The JDBC database connection URL Required
--username The database username Required
--password The database password Required

* will check nested s for definitions of the s to update.

update-count command s

Definition Requirement
<value> The integer indicating the number of changes you want to update Required
