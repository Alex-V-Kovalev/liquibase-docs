The Update Commands

The purpose of the update commands are to apply new database changes specified in the changelog.

When an update command is executed, will read the changesets in the changelog file and then compare the unique identifiers of id, author, and path to the filename to the values stored in the DATABASECHANGELOG table. If the unique identifiers do not exist in the DATABASECHANGELOG table, will apply the changes to the database.

Available Update Commands

  • update - used to deploy all new changes.
  • update-sql - executed before the update command to inspect the raw SQL for potential issues.
  • update-count - used to deploy a specified number of changes in sequential order.
  • update-count-sql - executed before the update-count command to inspect the raw SQL for potential issues.
  • update-to-tag - used to deploy changes from the newest change in the is reached.
  • update-to-tag-sql - executed before the update-to-tag command to inspect the raw SQL for potential issues.
  • update-testing-rollback - tests rollback support by deploying pending changesets, executing a sequential rollback for those changesets, then runs the update again to deploy the changesets.

Since update commands change the schema of a database, provides SQL output commands so users can inspect the SQL will generate before executing the update commands.

Pro Commands

These commands require an active Pro license:

  • update-one-changeset (PRO) - Used to deploy one specific changeset without affecting other undeployed changesets.
  • update-one-changeset-sql (PRO) - Executed before the update-one-changeset command to inspect the raw SQL for potential issues.

Each individual update command page will specify the criteria for using the command successfully including a list of required and optional command-specific arguments.

Update Command Requirements

All update commands require the following command arguments:

  • The root changelog file specified as --changelog-file.
  • The JDBC database connection URL specified as --url.

The username and password are not required for connections and systems which use alternate means of authentication.

Troubleshooting

If an error is returned when executing an update command, use the help parameter to check the correct syntax of the command and the command-specific parameters that can be used when executing the command.

In the CLI: liquibase <command> --help
