rollbackSQL

The rollbackSQL is a helper command that produces the raw SQL would run when using the rollback command.

Uses

The rollbackSQL command is typically used to inspect the SQL uses to revert changes associated with a tag you specify when you run the rollback command. It is best practice to use the rollbackSQL command before running the rollback command to ensure that you eliminate any potential risks.

Running the rollbackSQL command

To run the rollbackSQL command, you can specify the driver, classpath, and URL in your file. For more information, see Specifying Properties in a Connection Profile. You can also specify these properties in your command line.

Then run the rollbackSQL command:

liquibase --changeLogFile=postgres_lbpro_master_changelog.xml rollbackSQL myTag

Enter the name of the and tag that you want to use in place of postgres_lbpro_master_changelog.xml and myTag.

rollbackSQL global s

Attributes Definition Requirement
--changeLogFile* The root Required
--url The JDBC database connection URL Required
--username The database username Required
--password The database password Required
--outputFile** Specifies the file path to where the rollback will be written Optional

* checks nested s for definitions of the s to rollback.

**If not specified, rollbackSQL output goes to STDOUT.

rollbackSQL command s

Definition Requirement
<tag> The tag you can add to s to determine which s in the to evaluate based on their tags. Required
