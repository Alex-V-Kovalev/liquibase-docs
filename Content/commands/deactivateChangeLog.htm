deactivateChangeLog

The deactivateChangeLog command removes the changelogID from your file so it stops sending reports to .

Uses

The deactivateChangeLog command is typically used when:

  • You start using to keep a history of all your activity, but you have registered the wrong file and want to keep it from reporting the activity to .
  • You use a file that is being secluded or refactored into other s and do not want more than one source of s reporting to .

When you run the deactivateChangeLog command, it modifies a specific file by removing changelogID to prevent it from sending the data. The command differs from the liquibase.hub.mode=off property, which is set in your defaults file (the file) or passed as a JAVA property and prevents any from sending data.

The liquibase.hub.mode=off property will not report your activity to , regardless of a file, and will not ask to report the activity to when using a non-registered .

Running the deactivateChangeLog command

Before running the deactivateChangeLog command, you can specify your driver, classpath, and URL in your file. For more information, see Creating and configuring a [%=General.liquiPropFile%] file. You can also specify these properties in your command line.

Next, run the deactivateChangeLog command:

liquibase deactivateChangeLog

The file specified in your file or as CLI argument (--changelogFile=mychangelog) will have its changelogID removed.

If the file is in source control, commit the newly deactivated .

deactivateChangeLog global s
