connect

The connect command allows you to test access to the database without utilizing the .

Uses

This allows the user to test the connection’s readiness prior to including project workflows while simultaneously separating connection troubleshooting from workflow troubleshooting. This command works similarly to liquibase status or liquibase history if they were to operate without a .

Syntax

to run the connect command, specify the user name, password, and URL in the CLI, defaults file, as an environment variable, or any other standard method.

Run the connect command:

liquibase connect

Command arguments

Definition Requirement
--url Required
--format Optional
--password Required
--username Required
