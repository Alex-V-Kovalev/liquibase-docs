set-contexts

The set-contexts command lets you set contexts on your s using your command line. This is an alternative to setting contexts directly in your . You can use these contexts to specify which s to deploy in a later update.

Uses

If you need to apply contexts to many s, manually inserting them into your is slow and can lead to errors. Instead, use set-contexts to quickly add contexts in bulk.

The command is also useful if you want to standardize multiple similar contexts. You can specify command arguments to control the formatting uses when it inserts the contexts into your .

After setting your contexts with set-contexts, you can deploy your changes with the update command. While deploying, specify the --context-filter command argument to control which s are run.

Syntax

To run the set-contexts command, specify the driver, classpath, and URL in properties file. For more information, see Create and Configure a [%=General.liquiPropFile%] File. You can also specify these properties in your command line.

Then run the set-contexts command:

liquibase set-contexts --set-as=dev,stage
