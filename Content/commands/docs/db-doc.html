db-doc

The db-doc command generates documentation in a Javadoc format based on the existing database and s.

Uses

The db-doc command is typically used to generate database documentation with the change information stored in the s and the existing database. It captures the current state of your database including everything that has been performed against it.

For more information, see the changelogDocs.zip file, which shows an example of the database documentation structure with the related directories and files.

Running the db-doc command

To run the db-doc command, specify the driver, classpath, and URL in your properties file. For more information, see Specifying Properties in a Connection Profile. You can also specify these properties from the command line.

Then run the db-doc command:

liquibase --changelog-file=myChangelog.xml db-doc changelogDocs

Enter the name of the and the output directory that you want to use in place of myChangelog.xml and changelogDocs. Also, it is best practice to specify a new empty directory for the files path because the db-doc command creates its own directories, the CSS file, and multiple HTML files.

db-doc global s
