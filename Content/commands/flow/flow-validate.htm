flow validate

flow validate lets you validate a series of commands contained in one or more stages, as configured in a flow file. It is is similar to the validate command. It checks your flow file and identifies any possible errors that may cause the flow command to fail. This command is available in 4.15.0 and later.

Uses

Use the flow validate command to ensure:

  • The YAML is structured correctly
  • Referenced files can be found
  • Referenced commands and parameters are valid

The flow validate command only looks for possible errors in the flow file. It does not check for possible errors that might result from running the flow file on a specific database.

Syntax

To run the flow validate command, specify the driver, classpath, and URL in properties file. For more information, see Create and Configure a [%=General.liquiPropFile%] File. You can also specify these properties in your command line.

Then run the flow validate command:

liquibase flow validate

Global arguments

Definition Requirement
--license-key Required

Command arguments

Definition Requirement
--flow-file Optional
--flow-file-strict-parsing=<true|false> Optional
--flow-shell-interpreter Optional
--flow-shell-keep-temp-files=<true|false> Optional

Output

Flow file liquibase.flowfile.yaml is valid.
Liquibase command 'flow validate' was executed successfully.
