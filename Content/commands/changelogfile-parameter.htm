changeLogFile Parameter

file is a root file that contains a record of all your database changes (s). The changeLogFile parameter helps you create a file using the generateChangeLog command or diffChangeLog command, apply changes to your database, and keep track of your database information.

  • Use the generateChangeLog command and the changeLogFile parameter if you want to capture the current state of a database. The command creates the file with a sequence of s, which describes how to re-create your database. Then, you can apply those s to any number of databases.

To generate a file using Ant and Maven, run the Ant generateChangeLog Task with the outputfile parameter and the Maven generateChangeLog goal with the outputChangeLogFile parameter.

  • Use the diffChangeLog command with the changeLogFile parameter to have the information on differences between the two databases you are comparing. The command creates the file containing deployable s to synchronize multiple databases.

To create an empty file, go to your directory and create a new text file with the .xml, .sql, .yaml, or .json extension.

To apply changes related to a specific file, run the changeLogFile parameter and the needed command. You can also set the parameter in the file.

An example of running the changeLogFile parameter

liquibase --changeLogFile=dbchangelog.xml update

changelog-file CLI Usage in 4.4+

Starting from 4.4, there are changes in how you can configure or how you can type commands and parameters, in particular changeLogFile. The CLI functionality aims to be backward compatible with all previous versions. However, it is a best practice to follow the syntax rules mentioned in the Command Line Interface instruction and in the --help output.

An example of running the changelog-file parameter

liquibase update --changelog-file=dbchangelog.xml
