diffChangeLog

The diffChangeLog command allows you to receive information on differences between two databases you are comparing and creates a file containing deployable s.

The diffChangeLog command points out the differences in general and generates changes to resolve most of them.

Uses

The diffChangeLog command is typically used when you want to create a deployable to synchronize multiple databases. The diffChangeLog command also provides more information about:

  • Missing objects in your database
  • Changes made to your database
  • Unexpected items in your database

Running the diffChangeLog command

Running the diffChangeLog command requires two URLs:

  • referenceURL – the source for the comparison. The referenceURL represents your source (reference) database which is the starting point and the basis for the database you want to compare.
  • url – the target of the comparison. The URL stands for your target database which you want to compare to the source (reference) database. You typically perform actions and run the commands and against this database.

To create a diff :

  • The first option is to run the diffChangeLog command and pass the s needed for your source (reference) database and target database.

    • As an example, you can run the following:

    liquibase
--changeLogFile=dbchangelog.xml
--url="jdbc:oracle:thin:@<IP OR HOSTNAME>:<PORT>:<SERVICE NAME OR SID>"
--username=<USERNAME>
--password=<PASSWORD>
--referenceUrl="jdbc:oracle:thin:@<IP OR HOSTNAME>:<PORT>:<SERVICE NAME OR SID>"
--referenceUsername=<USERNAME>
--referencePassword=<PASSWORD>				
diffChangeLog

    If you want to create an SQL file, add your database type name when specifying the file: liquibase --changeLogFile=mychangelog.oracle.sql diffChangeLog. Replace .oracle.sql with your database type. When in doubt about your database type name, check Supported Databases.

    See the snapshot command topic for an example of using a snapshot file as one of the databases being used in the command.

    It is best practice to put your driver .jar file in the LIQUIBASE_HOME/lib directory. If you put the driver file in any other directory, specify the path to it in the file: classpath:../path_to_drivers/ojdbc<version>.jar. For more information, see Creating and configuring a [%=General.liquiPropFile%] file. When running the command against two different databases, both drivers should be in the LIQUIBASE_HOME/lib directory. Otherwise, the classpath property should reference both .jar files. Use the path separator that is correct for your operating system (a semicolon on Windows, a colon on Mac or Linux).

    classpath: ojdbc7.jar:postgresql-42.2.8.jar

  • Alternatively, configure the file to include your connection information for both databases. Run the following command:
    • liquibase --changeLogFile=file_name.xml diffChangeLog

    Replace file_name.xml with your filename and extension format. If you specify a file name that already exists, will append your changes to the existing file.

    For information on how to configure your file, see the Creating and configuring a [%=General.liquiPropFile%] file topic.

  • Index

Filtering diffChangeLog types

You can filter what objects diffChangeLog generates with the diffTypes :

--diffTypes=<catalog,tables,functions,views,columns,indexes,foreignkeys,primarykeys,uniqueconstraints,data,storedprocedure,triggers,sequences,databasePackage,databasePackageBody>

You can also filter specific objects by name with the includeObjects and excludeObjects s. For example:

--includeObjects=table:tableName,column:columnName,view:viewName,index:indexName
--excludeObjects=table:tableName,column:columnName,view:viewName,index:indexName

This way, you can prevent any undesired objects from ending up in the resulting .

Additional Functionality with

While stores all s in a , creates a directory called Objects and places the directory at the same level as your . The Objects directory contains a subdirectory for each of the following stored logic types:

  • checkconstraint
  • package
  • packagebody
  • function
  • trigger
  • synonyms

Not all database platforms support all stored logic types that are listed.

The diffChangeLog command structures stored logic files into timestamped directories every time you run the command.

diff-changelog CLI usage in 4.4+

Starting from 4.4, there are changes in how you can configure or how you can type commands, including diffChangeLog. The CLI functionality aims to be backward compatible with all previous versions. However, it is a best practice to follow the syntax rules on the Command Line Interface page and in the output of the --help command.

Running the diff-changelog command

Running the diff-changelog command requires two URLs:

  • referenceURL represents your source database which is the starting point and the basis for the database you want to compare.

  • url stands for your target database which you want to compare to the source (reference) database. You typically perform actions and run the commands against this database.

To compare two databases:

  • The first option is to run the diff-changelog command and pass the parameters needed for your source (reference) database and target database:
    • liquibase diff-changelog
--changelog-file=dbchangelog.xml
--url="jdbc:postgresql://<host>:<port>/<dbname>"
--username=<USERNAME>
--password=<PASSWORD>
--reference-url="jdbc:postgresql://<host>:<port>/<dbname>"
--reference-username=<USERNAME>
--reference-password=<PASSWORD>

    See the snapshot command documentation for an example of using a snapshot file as one of the databases being used in the command.

    If you want to create an SQL file, add your database type name when specifying the file: liquibase diff-changelog --changelog-file=mychangelog.oracle.sql. Replace .oracle.sql with your database type. When in doubt about your database type name, check Supported Databases.

    It is best practice to put your driver .jar file in the LIQUIBASE_HOME/lib directory. If you put the driver file in any other directory, specify the path to it in the file: classpath:../path_to_drivers/ojdbc<version>.jar. For more information, see Creating and configuring a [%=General.liquiPropFile%] file. When running the command against two different databases, both drivers should be in the LIQUIBASE_HOME/lib directory. Otherwise, the classpath property should reference both .jar files. Use the path separator that is correct for your operating system (a semicolon on Windows, a colon on Mac or Linux).

    --classpath=ojdbc7.jar:postgresql-42.2.8.jar

  • Alternatively, you can configure the file to include the connection information for both databases:
    • liquibase.command.changelogFile: dbchangelog.xml
liquibase.command.url: jdbc:postgresql://<host>:<port>/<dbname1>
liquibase.command.username: <username>
liquibase.command.password: <password>
liquibase.command.referenceUrl: jdbc:postgresql://<host>:<port>/<dbname2>
liquibase.command.referenceUsername: <username>
liquibase.command.referencePassword: <password>

    Then run the following command:

    liquibase diff-changelog

    For information on how to configure your file, see the Creating and configuring a [%=General.liquiPropFile%] file topic.
