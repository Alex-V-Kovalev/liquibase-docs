history

The history command lists all deployed s and their deploymentIds. The history command does not modify the database.

Uses

The history command is typically used when you want to inspect a particular group of changes to ensure that they have been applied to the database.

Running the history command

To run the history command, specify the driver, classpath, and URL in the properties file. For more information, see Specifying Properties in a Connection Profile. You can also specify these properties in your command line.

Then run the history command:

liquibase history

history global s

Definition Requirement
--url Required
--username Required
--password Required
--output-file Optional

Output

When successful, the history command produces the following output:

Liquibase Version: 4.9.1
Liquibase Community 4.9.1 by Liquibase
Liquibase History for jdbc:h2:tcp://localhost:9090/mem:dev

- Database updated at 5/10/22, 2:25 PM. Applied 3 changeset(s) in 1.582s, DeploymentId: 2214257654
