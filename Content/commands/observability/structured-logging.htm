Structured Logging

Structured Logging makes operation data easily available for automated monitoring and analysis tools to read, query, and act upon in automated workflows. This feature significantly improves the way logs its actions and events to make records machine-readable, easily-ingested, and queryable by industry standard observability and analysis tools. These tools include AWS Cloudwatch, Granfana, Opensearch, Sematext, Splunk, ElasticSearch and other analysis instruments. Existing monitoring and analysis tools can easily determine and act upon both real-time and long-term trend data for usage with Structured Logging. Other data that can also act on are performance, errors, security, tracking for auditablility and outcomes, and even standard DORA devops metrics. Liquibase does not just do the tricky work of db schema versioning and management, it also helps you understand the data around these operations and how they fit into your overall DevOps and CICD performance.

It is important to note that Structured Logging affects log output, not command output. The status, history, diff, etc commands deliver command output directly in the console. Log output is fine-grained and often is code or meta information related.

Log output example:

[2023-02-27 16:37:00] FINE [liquibase.integration] Performance monitoring disabled [2023-02-27 16:37:00] FINE [liquibase.configuration] No configuration value for liquibase.outputFile found [
2023-02-27 16:37:00] FINE [liquibase.configuration] No configuration value for liquibase.strict found

Enable Structured Logging

To enable Structured Logging, set the log format property to JSON or JSON_Pretty via the file, environment variable, or the CLI.

  • Environment Variable: LIQUIBASE_LOG_FORMAT=JSON or JSON_Pretty
  • CLI: liquibase --log-format=JSON or JSON_Pretty
  • Properties file: liquibase.logFormat=json or JSON_Pretty

Structured Logging is enabled successfully!

Not all commands are structured yet, so some regular logs will still appear.

--mirror-console-to-log Property

The --mirror-console-to-log property allows you to set all conventional output to STDOUT or STDERR to capture structure logs in the desired file.
