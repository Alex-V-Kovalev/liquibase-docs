The init Command

The init command sets up configuration files and parameters required to run and access a database. The command works with subcommands for specific functions – connecting to , getting a trial key, setting up a project. ( 4.7.0 and later versions)

Subcommands

Command Description
init hub The hub subcommand connects your local activity to with an API key and sets up the environment to use .
init project The project subcommand finds or creates your project folder with the and properties files and delivers an example H2 database to help you get started.
init start-h2 The start-h2 subcommand starts the in-memory H2 database included in the installation package to get started and test how works.
