status --verbose

The status --verbose command lists all undeployed s. It also lists the id, author, and file path name for each undeployed . The status --verbose command does not modify the database.

Uses

The status --verbose command is typically used when s were added to a through source control by another developer. The command confirms what has been deployed and what s are pending per author and corresponding IDs.

Running the status --verbose command

To run the status --verbose command, you can specify the driver, classpath, and URL in your file. You can also specify these properties in your command line.

Then run the status --verbose command:

liquibase --changeLogFile=postgres_lbpro_master_changelog.xml status --verbose

Enter the name of the you want to use in place of postgres_lbpro_master_changelog.xml.You can also add the changeLogFile to your liquibase.properties file.

status --verbose global attributes

Atrributes Definition Requirement
--changeLogFile The root Required
--username The database username Required
--password The database password Required
--url The JDBC database connection Required
