rollback-one-changeset-sql


The rollback-one-changeset-sql command is a helper command that allows you to inspect the SQL will run to revert the specified in the rollback-one-changeset command. It is only available for users.

Uses

The rollback-one-changeset-sql command is typically used when you want inspect the raw SQL that uses to revert your when you run the rollback-one-changeset command, so that you don't unintentionally make a mistake.

Running the rollback-one-changeset-sql command

Before running the rollback-one-changeset-sql command, gather the following information from your table or by running the history commands:

  • The Author of the you want to revert
  • The ID of the you want to revert
  • The file name ( path) of the you want to revert

Then run the rollback-one-changeset-sql command, with your information:

liquibase --changelog-file=changelog.xml rollback-one-changeset-sql --changeSetAuthor="LiquibaseProUser" --changeSetId="createProc-proschema" --changeSetPath=changelog.xml

For more command specific help, type liquibase rollback-one-changeset-sql --help into the command prompt.

rollback-one-changeset-sql global s
