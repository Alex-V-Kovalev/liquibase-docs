rollback-sql

The rollback-sql tag is a helper command that produces the raw SQL would run when using the rollback command.

Uses

The rollback-sql command is typically used to inspect the SQL uses to revert changes associated with a tag you specify when you run the rollback command. It is best practice to use the rollback-sql command before running the rollback command to ensure that you eliminate any potential risks.

Running the rollback-sql command

To run the rollback-sql command, specify the driver, classpath, and URL in the properties file. For more information, see Specifying Properties in a Connection Profile. You can also specify these properties in your command line.

Then run the rollback-sql command:

liquibase --changelog-file=postgres_lbpro_master_changelog.xml rollback-sql myTag

Enter the name of the and tag that you want to use in place of postgres_lbpro_master_changelog.xml and myTag.

rollback-sql global s

Attributes Definition Requirement
--changelog-file* The root Required
--url The JDBC database connection URL Required
--username The database username Required
--password The database password Required
--output-file** Specifies the file path to where the rollback will be written Optional

* checks nested s for definitions of the s to rollback.

**If not specified, rollback-sql output goes to STDOUT.

rollback-sql command s

Definition Requirement
<tag> Tags allow the user to identify which s in the to evaluate. Required
