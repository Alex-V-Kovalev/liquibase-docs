sync-hub

The sync-hub command synchronizes the local table with .

Uses

The sync-hub command is typically used when you want to:

  • Ensure that shows the latest results from your table.
  • Synchronize the table of a new project that has pre-existing data in the table.
  • Synchronize your local data if update, rollback, changelog-sync, or drop-all were run while was offline.

When you run the sync-hub command, you will not see any operations added to the Project associated with your . sync-hub cannot associate previous operations. You will only see s added to the s tab in your project.

Running the sync-hub command

To run the register-changelog command, specify the following parameters in your file, environment variables, or the command prompt while running the command:

For more information about the options to store parameters, see Creating and configuring a [%=General.liquiPropFile%] file, [%=General.Liquibase%] Environment Variables, and JAVA_OPTS Environment Variable. To find the format of the parameter, see Command Parameters.

  • Driver, URL, and user authentication information such as username and password
  • API key
  • file

    • If you have not registered your file, use the register-changelog command. The register-changelog command connects your local activity to a specific Project.

  • Target ID specified by the hubConnectionId parameter

    • If you have the changeLogFile parameter, there is no need to specify the hubConnectionId parameter with the command. If you use hubConnectionId, add it to the sync-hub command at the command prompt.

    To find the hubConnectionId, select Projects > View Details for the required project > Targets > View Status > Target ID.

Run the sync-hub command:

liquibase sync-hub

sync-hub command attributes

Attribute Definition Requirement

hubConnectionId

Specifies the target (also called connection) needed for the sync-hub command to synchronize the local table with .
