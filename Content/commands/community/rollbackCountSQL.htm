rollbackCountSQL <value> command

The rollbackCountSQL <value> command is a helper command that allows you to inspect the SQL will run while using the rollbackCount <value> command.

Uses

The rollbackCountSQL <value> command is used when you want to inspect the raw SQL before running the rollbackCount <value> command, so you can correct any issues that may arise before running the command. uses the raw SQL to revert any s between the most recent and the value you specified.

Running the rollbackCountSQL <value> command

Before running the rollbackCountSQL <value> command, you can run the history command to see all your previously executed changes.

RollbackCountSQL <value> works from the bottom of the list towards the top. Also, unlike the rollbackCount <value> command, there is no impact of this command. See the rollbackCount <value> topic for more information.

To run the rollbackCountSQL <value> command, you can specify the driver, classpath, and URL in your file. For more information, see Creating and configuring a [%=General.liquiPropFile%] file. You can also specify these properties in your command line.

Then run the rollbackCountSQL <value> command:

liquibase –changeLogFile=mychangelog.postgresql.sql rollbackCountSQL 1

Enter the name of the and value you want to use in place of mychangelog.postgresql.sql and 1.

rollbackCountSQL <value> global s

Attribute Definition Requirement
--changeLogFile * Specifies the root Required
--url Specifies the JDBC database connection URL Required
--username Specifies the database username Required
--password Specifies the database password Required
--outputFile ** Specifies the file path to where the Rollback SQL will be written Optional

* checks the and any nested s for the definitions of the s to roll back.

* * If not specified, rollbackCount SQL output goes to STDOUT.

rollbackCountSQL <value> command s

Attribute Definition Requirement
--value The integer indicating the number of changes you want to rollback Required
