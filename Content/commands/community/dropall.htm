dropAll command

Running the dropAll command

To run the dropAll command, you can specify the driver, classpath, and URL in your file. For more information, see Creating and configuring a [%=General.liquiPropFile%] file. You can also specify these properties in your command line.

Then run the dropAll command:

liquibase dropAll

dropAll global s

Attribute Definition Requirement
--url Specifies the JDBC database connection URL Required
--username Specifies the database username Required
--password Specifies the database password Required

Using the dropAll command with

If you use and want to receive the dropAll operation reports, you need to specify one of the following s when running the dropAll command.

Definition Requirement
--hubConnectionId The that identifies the specific Connection in which to record your data at . The is available in your Project at https://hub.liquibase.com. The is required only for the reporting
--hubProjectId The that identifies the specific Project in which to record your data at . The is available in your account at https://hub.liquibase.com. The is required only for the reporting
--changeLogFile A registered file that identifies the specific Project in which to record your data at Hub. The is optional to run the dropAll command. If you specify a registered , it will enable reporting.

Additionally, you need to be a user with a Hub API Key set in the file or passed in the command line.

example

liquibase.hub.apiKey: <HubAPIKey>
liquibase.hub.connectionId: <connectionId>
liquibase.hub.projectId: <projectId>

CLI example

liquibase dropAll --hubProjectId=<projectId> --liquibaseHubApiKey=<addYourHubApiKeyHere>
liquibase dropAll --hubConnectionId=<connectionId> --liquibaseHubApiKey=<addYourHubApiKeyHere>

You can also set your properties as Environment variables:

LIQUIBASE_HUB_API_KEY="<HubAPIKey>"
LIQUIBASE_COMMAND_HUB_PROJECT_ID="<projectId>"
LIQUIBASE_COMMAND_HUB_CONNECTION_ID="<connectionId>"

To retrieve or manage your API keys, go to API Keys by selecting your account in the upper-right corner of the page. To find the hubProjectId, select Projects > View Details. To find the hubConnectionId, select Project > View Details > Connections > View Status.

dropAll example output

When successful, the dropAll command produces the following output:

drop-all CLI usage in 4.4+

Starting from 4.4, there are changes in how you can configure or how you can type commands, in particular dropAll. The CLI functionality aims to be backward compatible with all previous versions. However, it is a best practice to follow syntax rules mentioned in the Command Line Interface instruction and in the --help output.
