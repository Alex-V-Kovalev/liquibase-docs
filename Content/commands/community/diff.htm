diff command

The diff command in allows you to compare two databases of the same type, or different types, to one another.

Uses

The diff command is typically used at the completion of a project to verify all expected changes are in the or to detect drift between a model schema and a database's actual schema. The diff command is also useful for the following tasks:

  • Finding missing objects between one database and another
  • Seeing that a change was made to your database
  • Finding unexpected items in your database

Running the diff command

Running the diff command requires two URLs:

  • referenceURL – the source for the comparison. The referenceURL represents your source (reference) database which is the starting point and the basis for the database you want to compare.

  • url – the target of the comparison. The URL stands for your target database which you want to compare to the source (reference) database. You typically perform actions and run the commands against this database

To compare two databases:

  • The first option is to run the diff command and pass the parameters needed for your source (reference) database and target database. You can do this by running the following:
    • liquibase diff
--url="jdbc:oracle:thin:@<IP OR HOSTNAME>:<PORT>:<SERVICE NAME OR SID>"
--username=<USERNAME>
--password=<PASSWORD>
--referenceUrl="jdbc:oracle:thin:@<IP OR HOSTNAME>:<PORT>:<SERVICE NAME OR SID>"
--referenceUsername=<USERNAME>
--referencePassword=<PASSWORD>

    See the snapshot command documentation for an example of using a snapshot file as one of the databases being used in the command.

    It is best practice to put your driver .jar file in the LIQUIBASE_HOME/lib directory. If you put the driver file in any other directory, specify the path to it in the file: classpath:../path_to_drivers/ojdbc<version>.jar. For more information, see Creating and configuring a [%=General.liquiPropFile%] file.
    When running diff against two different databases, both drivers should be in the LIQUIBASE_HOME/lib directory. Otherwise, the classpath property should reference both .jar files. Use the path separator that is correct for your operating system (a semicolon on Windows, a colon on Mac or Linux).

    classpath: ojdbc7.jar:postgresql-42.2.8.jar

  • Alternatively, you can configure the file to include your connection information for both databases. Run the following command:
    • liquibase diff

    For information on how to configure your file, see the Creating and configuring a [%=General.liquiPropFile%] file topic.

The diff command produces a list of categories along with one of the following descriptions:

  • Missing: there are objects on your source database (referenceURL) that are not on your target database (URL).
  • Unexpected: there are objects on your target database (URL) that are not on your source database (referenceURL).
  • Changed: the object as it exists on the source database (referenceURL) is different than as it exists in the target database (URL).

The changed description will not specify the type of change applied to your database. Run the diffChangeLog command to generate a that will apply the changes to the target database.

Filtering diff types

allows you to use diffType to filter the types of objects you want to compare. Multiple filters can be added to the as a comma-separated list. If no diffTypes are specified, all objects are considered.

liquibase --diffTypes=tables,indexes,views diff

 diff CLI usage in 4.4+

Starting from 4.4, there are changes in how you can configure or how you can type commands, including diff. The CLI functionality aims to be backward compatible with all previous versions. However, it is a best practice to follow the syntax rules on the Command Line Interface page and in the output of the --help command.

Running the diff command

Running the diff command requires two URLs:

  • referenceURL represents your source database which is the starting point and the basis for the database you want to compare.

  • url stands for your target database which you want to compare to the source (reference) database. You typically perform actions and run the commands against this database.

To compare two databases:

  • The first option is to run the diff command and pass the parameters needed for your source (reference) database and target database:
    • liquibase diff
--url="jdbc:postgresql://<host>:<port>/<dbname>"
--username=<USERNAME>
--password=<PASSWORD>
--reference-url="jdbc:postgresql://<host>:<port>/<dbname>"
--reference-username=<USERNAME>
--reference-password=<PASSWORD>

    See the snapshot command documentation for an example of using a snapshot file as one of the databases being used in the command.

    It is best practice to put your driver .jar file in the LIQUIBASE_HOME/lib directory. If you put the driver file in any other directory, specify the path to it in the file: classpath:../path_to_drivers/ojdbc<version>.jar. For more information, see Creating and configuring a [%=General.liquiPropFile%] file.
    When running diff against two different databases, both drivers should be in the LIQUIBASE_HOME/lib directory. Otherwise, the classpath property should reference both .jar files. Use the path separator that is correct for your operating system (a semicolon on Windows, a colon on Mac or Linux).

    --classpath=ojdbc7.jar:postgresql-42.2.8.jar

  • Alternatively, you can configure the file to include the connection information for both databases:
    • liquibase.command.url: jdbc:postgresql://<host>:<port>/<dbname1>
liquibase.command.username: <username>
liquibase.command.password: <password>
liquibase.command.referenceUrl: jdbc:postgresql://<host>:<port>/<dbname2>
liquibase.command.referenceUsername: <username>
liquibase.command.referencePassword: <password>

    Then run the following command:

    liquibase diff

    For information on how to configure your file, see the Creating and configuring a [%=General.liquiPropFile%] file topic.
