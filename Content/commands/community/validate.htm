validate command

The validate command checks and identifies any possible errors in a that can cause the update command to fail.

Uses

The validate command is mainly used when you want to detect if there are any issues with a before running the update command.

With the help of the validate command, you can avoid a partial update, where only some s are applied due to an error in your file.

Use the validate command to ensure that:

  • The XML/YAML/JSON/formatted SQL is structured correctly
  • Referenced files can be found
  • There are no duplicated id/author/file combinations
  • There aren't any checksum errors
  • Any required or not allowed attributes are correct for your database

Running the validate command

To run the validate command, you need to specify your driver, class path, URL, and user authentication information in your liquibase.properties file. For more information, see Creating and configuring a liquibase.properties file. You can also specify these properties in your command line.

Then run the validate command:

liquibase --changeLogFile=oracle_runWith_changelog.xml validate

Enter the name of the you want to use in place of oracle_runWith_changelog.xml.

validate global attributes

Attribute Definition Requirement
--changeLogFile * The root Required
--url The JDBC database connection URL Required
--username The database username Required
--password The database password Required

* will check nested s for definitions of the s to validate.
