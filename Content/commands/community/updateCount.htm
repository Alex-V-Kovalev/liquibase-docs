updateCount <value> command

The updateCount <value> command updates a specified number of s, where <value> is the number of s you want to update sequentially on your database.

Uses

The updateCount <value> command is mainly used when you want to apply changes and update s sequentially, starting with the s from the top of the file until the number specified is reached.

The image below shows four s: a, b, c, and d. As you can see, a is already deployed on the database. So, running the command updateCount 2, deploys b and c without applying d.

Additionally, the best practice is to run the updateCountSQL <value> helper command because it allows you to inspect the updateCount <value> SQL, so you can correct any issues that may arise before running the command.

Running the updateCount <value> command

To run the updateCount <value> command, you need to specify your driver, class path, URL, and user authentication information in your liquibase.properties file. You can also specify these in your command line.

Then run the updateCount <value> command:

liquibase --changeLogFile=myChangelog.xml updateCount 2

Enter the name of the and number you want to use in place of myChangelog.xml and 2.

updateCount <value> global attributes

Attribute Definition Requirement
--changeLogFile * The root Required
--url The JDBC database connection URL Required
--username The database username Required
--password The database password Required

* will check nested s for definitions of the s to update.

updateCount <value> command attributes

Attribute Definition Requirement
<number> The integer indicating the number of changes you want to update Required
