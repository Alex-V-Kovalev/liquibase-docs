generateChangeLog command


The generateChangeLog command creates a file that has a sequence of s which describes how to re-create the current state of the database.

Uses

The generateChangeLog command is typically used when you want to capture the current state of a database, then apply those changes to any number of databases. This is typically only done when a project has an existing database, but hasn’t used before. See Adding [%=General.Liquibase%] on an existing project for more details.

Note: When using the update command to apply the changes in the , will not create a new database or schema. You must create them before applying the to it.

  1. Open your command prompt or Linux terminal and run the following command:

Output

The generateChangeLog command generates a that contains all of your Objects (represented as s) and places the file in the same directory where the command was ran.

The extension provided determines the format of the , so if you specify the filename as changelog.xml you will get an XML formatted . However, if you specify the filename as changelog.yaml or changelog.json or changelog.postgresql.sql you will get s formatted in YAML or JSON or SQL, respectively.

When generating an SQL formatted , you must specify the short name of the targeted database type as part of the filename (as shown above).

Additional functionality with

While stores all s in a , creates a directory called Objects and places the directory at the same level as your . The Objects directory contains a subdirectory for each of the following stored logic types.

  • package
  • packagebody
  • function
  • stored procedure
  • trigger
  • view

Not all database platforms support all of these types.
