unexpectedChangeSets command

The unexpectedChangeSets command produces a list of s that were run in the database but do not exist in the current .

Uses

The unexpectedChangeSets command is typically used to detect and compare the changes between the DATABASECHANGELOG table and the current . If any of the s in the DATABASECHANGELOG table do not exist in the current , the unexpectedChangeSets command detects those s and produces them in your output.

The unexpectedChangeSets command also produces all the deleted s that were previously deployed from the current .

The image below shows the that is already applied to the database—changelog_release1. If you want to apply a new changelog_release2 and it requires information about the previous s applied against the database, the unexpectedChangeSets command will list those s from changelog_release1 and changelog_release2.

By running the unexpectedChangeSets command, you can review any s in the DATABASECHANGELOG table applied from any other s outside the scope of your current .

Running the unexpectedChangeSets command

To run the unexpectedChangeSets command, you need to specify your driver, class path, URL, and user authentication information in your liquibase.properties file. For more information, see Creating and configuring a liquibase.properties file. You can also specify these properties in your command line.

Then run the unexpectedChangeSets command:

liquibase unexpectedChangeSets

To see the list of all changes applied, add the --verbose flag while running the command:

liquibase unexpectedChangeSets --verbose
