unexpectedChangeSets command

The unexpectedChangeSets command produces a list of s that were run in the database but do not exist in the current .

Uses

The unexpectedChangeSets command is typically used to detect and compare the changes between the table and the current . If any of the s in the table do not exist in the current , the unexpectedChangeSets command will detect those s and produce them in your output.

The unexpectedChangeSets command also produces all the deleted s that were previously deployed from the current .

The following image shows the file, which includes s for release 1 that are already applied to the database and new s for release 2 that are not applied to the database. Running the unexpectedChangeSets command will produce the number of s applied for release 1.

So, if you want to identifying unexpected s that were previously applied to the database, run the unexpectedChangeSets command.

Running the unexpectedChangeSets command

To run the unexpectedChangeSets command, you need to specify your driver, class path, and URL in your file. For more information, see Creating and configuring a [%=General.liquiPropFile%] file. You can also specify these properties in your command line.

Then run the unexpectedChangeSets command:

liquibase --changeLogFile=changelog.xml unexpectedChangeSets

Enter the name of the you want to use in place of changelog.xml. You can also specify the file in your file. The output will show the number of unexpected s.

To see the list of unexpected s, add the --verbose flag while running the command:

liquibase --changeLogFile=changelog.xml unexpectedChangeSets --verbose

unexpectedChangeSets global s
