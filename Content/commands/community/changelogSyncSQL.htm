changelogSyncSQL command

The changelogSyncSQL is a helper command that produces the raw SQL would run when using the changelogSync command.

Uses

The changelogSyncSQL command is typically used when you want to inspect the SQL will use to baseline a new database environment.

It is best practice to inspect any SQL that would run when using the changelogSync command so you can review any changes the command would make to your database before running the command.

Running the changelogSyncSQL command

To run the changelogSyncSQL command, you can specify the driver, classpath, and URL in your file. For more information, see Creating and configuring a [%=General.liquiPropFile%] file. You can also specify these properties in your command line.

Then run the changelogSyncSQL command:

liquibase --changelogFile=mychangelog.postgresql.sql changelogSyncSQL

Enter the name of the that you want to use in place of mychangelog.postgresql.sql.

changelogSyncSQL global s

Definition Requirement
--changeLogFile The root Required
--url The JDBC database connection URL Required
--username The database username Required*
--password The database password Required*
--outputFile* The that specifies the file path to where the changelogSync SQL will be written Optional

*If not specified, the changelogSyncSQL output goes to STDOUT.

changelog-sync-sql CLI usage in 4.4+

Starting from 4.4, there are changes in how you can configure or how you can type commands, in particular changelog-sync-sql. The CLI functionality aims to be backward compatible with all previous versions. However, it is a best practice to follow syntax rules mentioned in the Command Line Interface instruction and in the --help output.

Running the changelog-sync-sql command

To run the changelog-sync-sql command, you can specify the driver, classpath, and URL in your file. For more information, see Creating and configuring a [%=General.liquiPropFile%] file. You can also specify these properties in your command line.

Run the changelog-sync-sql command:

liquibase changelog-sync-sql --changelog-file mychangelog.xml

Enter the name of the that you want to use in place of mychangelog.xml.

changelog-sync-sql command s

Definition Requirement
--changelog-file The root Required
--url The JDBC database connection URL Required
--username The database username Required*
--password The database password Required*
--output-file* The that specifies the file path to where the changelogSync SQL will be written Optional

*If not specified, the changelog-sync-sql output goes to STDOUT.
