Quality Checks Packages

4.25.0 +

Checks-packages is a YAML file that allows you to create a list of multiple checks-settings files which can run in one command, either checks run or checks show. This allows different teams to own, control, update, and automatically synchronize their checks with their dependent teams. You also can centralize Quality Check resources. The check package file is the central hub that reaches out to the local and remote checks-settings files needed for different jobs, pipelines, and teams.

Checks-packages rules

Below are the rules to keep in mind that apply to the checks-packages file. Review these rules before moving on to creating a checks-packages file.

checks-packages only works with checks run and checks show.

It's important to note that removes duplicated copies at the file level, not the check level. So two EXACT paths will be removed, but different files can have checks named the same, such as “myLabelCheck,” and will not remove the copy automatically.

That means you cannot use commands such as checks customize, copy, bulkset, edit, or delete when the checks-settings file you specify is a check-package file.

Create a checks-package file

Run the liquibase checks create command to create a new yaml checks-package file.

You can set the following properties for the checks-package file:

  • --package-file
    • the filename of the yaml file to be created. defaults to liquibase.checks-package.yaml if not set.
  • --package-name
    • the name of the checks-package object inside the file to be created
  • --package-contents
    • the file, files, or dir of files to be included in the checks-package object

Checks-packages features

There is a column in the checks show table that displays the source check-settings file. There is also a line in the console output for checks run that highlights the check-settings file which holds the enabled checks and other check run console output. Learn how to Use the Checks Settings Configuration File here.
