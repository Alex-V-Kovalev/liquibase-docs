Quality Checks Packages

In 4.25.0 checks packages are available. Checks-packages is a YAML file that allows you to create a list of multiple checks-settings files which can run in one command such as checks run or checks show. This allows different teams to own, control, update, and automatically synchronize their checks with their dependent teams. This means that different aspects of checks can be controlled by different teams. You also can centralize Quality Check resources. The check package file can be centralized as the main hub that has spokes reaching out to the Quality Check files needed for different jobs, pipelines, and teams. You may also reach local and remote locations with one command because each team can refer to the same files.

Implement Checks-packages

checks packages only work with checks run and checks show.

There is a column in the checks show table that displays the source check-settings file. There is also an extra display line to the console output for checks run to highlight the check-settings file which holds the enabled checks and other check run console output. Learn how to Use the Checks Settings Configuration File here.

It's important to note that we dedupe at the file level, not the check level. So two EXACT paths will be deduped, but different files can have checks named the same, say “myLabelCheck,” and we wont try to dedupe that check itself.

Checks Settings File Example

## This is a Liquibase checks packages formatted yaml file. ## It contains just one or more 'checksPackages' objects ## with one or more named checks-settings-files. ## These checks-settings-files listed in a package are processed sequentially ## during 'liquibase checks run' or 'liquibase checks show' commands ## which specify this file as the checks-settings-file property. ## Learn more at https://docs.liquibase.com/quality-checks ## Uncomment and change names with your checks-settings-files checksPackages: - name: "master-checks-Sept2023.pkg" files: - "./liquibase.checks-settings.conf" - "./liquibase.more.checks-settings.yaml" - hashicorp,secret/liquibase/ChecksSettingsFile,checksSettingsFile
