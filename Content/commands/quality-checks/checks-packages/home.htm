Quality Checks Packages

In 4.25.0+ Checks-packages are available. Checks-packages is a YAML file that allows you to create a list of multiple Checks Settings Configuration Files. You can apply multiple checks with a single checks run or checks show command. This allows different teams to own, control, update, and automatically synchronize their checks with their dependent teams. Centralizing Quality Check resources has never been easier. The checks-packages file is the central hub that reaches out to the local and remote checks-settings files needed for different jobs, pipelines, and teams. For example, if your Security team stores their checks in one place, the checks-packages file that points to that security checks file will pick up the latest version. This allows your distinct teams to reproduce the same outcome even if files are located separately.

Checks-packages rules

Below are the rules to keep in mind that apply to the checks-packages file. Review these rules before moving on to creating a checks-packages file.

  • checks-packages only works with checks run and checks show.
  • removes duplicated copies of checks-packages at the file level, not the check level. So two EXACT paths will be removed, but different files can have checks named the same, such as “myLabelCheck,” and will not remove the copy automatically.
    • That means you cannot use commands such as checks customize, copy, bulkset, edit, or delete when the checks-settings file you specify is part of a checks-packages file.

Create a checks-packages file

Run the liquibase checks create command to create a new yaml checks-package file.

You can set the following properties for the checks-package file in the CLI, file, or as Environment Variables. In the example below the CLI input is used.:

  • --package-file
    • the filename of the yaml file to be created. defaults to liquibase.checks-package.yaml if not set.
  • --package-name
    • the name of the checks-package object inside the file to be created
  • --package-contents
    • the file, files, or dir of files to be included in the checks-package object

Additional Checks-packages features

There is a column in the checks show table that displays the source check-settings file. There is also a line in the console output for checks run that highlights the check-settings file which holds the enabled checks and other check run console output. Learn how to Use the Checks Settings Configuration File here.

Checks Packages File Example

## This is a Liquibase checks packages formatted yaml file. ## It contains just one or more 'checksPackages' objects ## with one or more named checks-settings-files. ## These checks-settings-files listed in a package are processed sequentially ## during 'liquibase checks run' or 'liquibase checks show' commands ## which specify this file as the checks-settings-file property. ## Learn more at https://docs.liquibase.com/quality-checks ## Uncomment and change names with your checks-settings-files checksPackages: - name: "master-checks-Sept2023.pkg" files: - "./liquibase.checks-settings.conf" - "./liquibase.more.checks-settings.yaml" - hashicorp,secret/liquibase/ChecksSettingsFile,checksSettingsFile
