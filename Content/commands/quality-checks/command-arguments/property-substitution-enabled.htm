property-substitution-enabled

property-substitution-enabled is a command argument specific to checks run that allows you to determine when property substitution occurs in a . You can change the default timing of property expansion in your s. This argument is for the whole of checks run, not a specific quality check.

Learn more here: Substituting Properties in Changelogs

--property-substitution (Boolean)

Alias:

--property-sub

true CLI: liquibase checks run --property-substitution-enabled=true|false

Property: liquibase.command.checks.property.substitution

Environment Variable: LIQUIBASE_COMMAND_CHECKS_RUN_PROPERTY_SUBSTITUTION

 Allows you to check to see if text is evaluated pre or post property substitution. This argument is for the whole of checks run, not a specific quality check.

Use Cases

Example: When property substitution is set to true

When property-substitution-enabled is set to true, the value of the substitution is evaluated after by the checks run command.

When you create a stored procedure in SQL Server, you want to confirm that developers follow the best practice to drop the procedure if it exists, create the new procedure, and assign privileges to roles on the new procedure in the same . property-substitution allows you to reuse the name of that procedure to enforce those three requirements.

You can use property substitution tokens to replace database links.

In this example, @supe replaces PE in the UAT environment and @tppe replaces PE in the PROD environment.

<property  name="PE"  value="@supe"  context="UAT"/>
<property  name="PE"  value="@tppe"  context="PROD"/>

Notice @supe in the example below. That is the database name used in the UAT environment.

SELECT TO_DATE(lookup_str_value1||TO_CHAR(p_indate,'YYYY'),'MMDDYYYY')
INTO v_annual_tag_date
from ALPS_MML_LOOKUP@supe
WHERE lookup_group = 'MARKET_ANNUAL_TAG'
AND lookup_code = p_market;

Once the SQL script has implemented the property-substitution token, replaces @supe with ${PE}.

Example: When property substitution is set to false

When property-substitution-enabled is set to false, the value of the substitution is evaluated before by the checks run command.
