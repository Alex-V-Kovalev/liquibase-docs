create

The create subcommand allows you to create a new YAML checks package file. Learn more about Checks packages here: Quality Checks Packages. It is available in 4.25.0 and later.

Uses

Using the create subcommand alongside the checks command creates a new yaml checks package file. This checks package file contains any Quality Check you would like to include so that you can run a bundle of checks at once rather than one at a time.

Syntax

CLI: 

liquibase checks create

A new YAML checks package file is available.

You can set the following properties for the checks package file in the CLI, file, or as Environment Variables. In the example below the CLI input is used.:

Checks package property CLI example Description
--package-file liquibase checks run --packages-file=security-checks.pkg

The file name of the yaml file to be created. defaults to liquibase.checks-package.yaml if not set.
--package-name liquibase checks show --package-name=changelog-checks.pkg

The name of the checks-package object inside the file to be created
--package-contents liquibase checks show --package-contents=database-checks.pkg

The file, files, or dir of files to be included in the checks-package object

For best results, specify all commands and parameters in the --kebab-case format in the CLI. If your preference is camelCase, it also works in the CLI.
