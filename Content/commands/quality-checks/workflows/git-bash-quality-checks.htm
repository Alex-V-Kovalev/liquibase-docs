Use Git Bash with Quality Checks

Without proper configuration, Git Bash shell on Windows does not display the correct prompts to configure Checks and allow the customization of Quality Checks. This guide covers how to work around this limitation so you can customize Quality Checks and use Git Bash. The instructions allow Git Bash to work like the native Windows command prompt or Linux terminal.

By default, the --headless parameter is set to false and you can use Quality Checks configuration commands in GitBash or any other terminal desired.

However, if Quality Checks prompts do not appear in Git Bash as expected, configure the Git Bash TERM_PROGRAM environment variable in the Git Bash terminal like so:

TERM_PROGRAM=mintty
