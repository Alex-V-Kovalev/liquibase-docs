maxAffectedRowsAllowed

This check triggers when the Max Affected Rows Allowed value is exceeded by, per user setup, either any , any pending , or just one specific in the changelog. This means that you can prevent DML statements from being committed to your database if more than a specified number of rows are affected by that statement.

Run the maxAffectedRowsAllowed check

To run the maxAffectedRowsAllowed check, ensure you have a license, a valid url property that points to a live JDBC connection or a snapshot, and that the checks-scope property includes or database. The checks-scope property is set in the default properties file, environment variable, or any standard method. Then check if any searchable object types match the supplied pattern.

Enable the maxAffectedRowsAllowed check by running the following command in the CLI:

liquibase checks enable --check-name=maxAffectedRowsAllowed
prompts you to establish a copy of this check and to initiate the customization work flow. To do this, select options for each of the attributes below in the CLI.

This check can not be enabled directly because one or more fields does not have a default value. Creating a copy of this check initiates the customization work flow.
