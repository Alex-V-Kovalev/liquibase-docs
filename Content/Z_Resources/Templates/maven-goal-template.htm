Maven [goal]

Description

The sample goal executes quality checks using the checks settings file and the file you specify.

Uses

Use the sample goal to run static and dynamic checks against the specified in XML, SQL, YAML, or JSON format. reports the outcome of the checks to STDOUT. The return code helps you detect non-compliant changes and stop deployment jobs in automation.

If your files are not stored in the working directory, specify the relative path to them. See How [%=General.Liquibase%] Finds Files: [%=General.Liquibase%] Classpath for more details.

To see the list of available checks, run mvn liquibase:checks.show.

Running the [sample] Maven goal

Running the sample goal requires you to have a Maven project implemented with the parameter specified.

To run the goal, enter the following at your command prompt:

mvn liquibase:sample

Property Definition

CLI: -Dliquibase.checksSettingsFile

Property in the pom.xml file: <checksSettingsFile>path/to/checks-settings.conf</checksSettingsFile>

Environment variable: LIQUIBASE_COMMAND_CHECKS_SHOW_CHECKS_SETTINGS_FILE

This is the parameter that specifies the needed checks settings file to work with goals.

Set the checksSettingsFile parameter to the relative path of the checks settings file which you want to use or modify.

For more information, see Using the Checks Settings Configuration File.

CLI: -Dliquibase.changeLogFile

Property in the pom.xml file: <changeLogFile>your changelog file</changeLogFile>

Environment variable: LIQUIBASE_COMMAND_CHECKS_RUN_CHANGELOG_FILE

 The file against which you execute checks when running the checks.run goal.
