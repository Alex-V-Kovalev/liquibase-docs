How to Apply Your License Key

license key allows using all the functionality of as well as the following features:

  • Access to reporting, monitoring, and insights with
  • Targeted rollbacks for one change or set of changes
  • Enhanced stored logic capabilities
  • Database drift detection
  • SQL Plus integration

To take advantage of all features, it is recommended to use the latest version of .

There are four ways to apply the license key:

  1. Include the license key liquibaseProLicenseKey: [paste the license key] in your file and save it. For example:
    liquibaseProLicenseKey: aei76ou32thp785463214
  2. Pass the license key as an argument in the command line during runtime: liquibase --liquibaseProLicenseKey=[paste the license key] [command]. For example:
    liquibase --liquibaseProLicenseKey=aei76ou32thp785463214 update
  3. Set the license key as an environment variable in the command line. On Windows:
    set LIQUIBASE_PRO_LICENSE_KEY=aei76ou32thp785463214

    On Linux/macOS:

    export LIQUIBASE_PRO_LICENSE_KEY=aei76ou32thp785463214

    For more information, see [%=General.Liquibase%] Environment Variables.

  4. If you use Maven, include the license key in the <properties> section of your pom.xml file: <liquibase.liquibaseProLicenseKey>key-goes-here</liquibase.liquibaseProLicenseKey/>. For example:
    <plugin>
    <groupId>org.liquibase</groupId>
    <artifactId>liquibase-maven-plugin</artifactId>
    <version>0-SNAPSHOT</version>
    <configuration>
        <changeLogFile>com/example/mysql_lbpro_master_changelog_v4.6.xml</changeLogFile>
        <liquibaseProLicenseKey>key-goes-here</liquibaseProLicenseKey>
    </configuration>
</plugin>

To verify that your license key works as expected and to check its expiration date, you can run the status --verbose command.
