Creating and configuring a file

A file is a text-based file that allows you to store properties that don't change often. You can typically use this file to specify your database connection information. Using the file saves you time and potential typing errors by removing the need to enter these properties as command line arguments.

Creating the file

To create a file, generate a new text file in your project directory and name it .

The file can also be named something other than and be in a completely different directory by using the --defaultsFile attribute every time you run a command. For more information, see Command Line Interface.

Properties

The most common properties that you might specify in the are listed below. The best practice is to configure the file to include your driver class path, URL, and user authentication information for the database you want to capture.

Any property that could be specified on the command line can also be specified in the properties file. If a property is specified in both the properties file and the command line, the command line value will override the value in the file.

Adding your license key

As you can see in the example above, to add your License Key, just add the liquibaseProLicenseKey: attribute to your file and add your License Key. Then save your file.

Additional properties

You can include the following properties in the file as of 4.1. If you use the previous versions, set these properties as java system properties. For example, in the CLI, the properties can be set with JAVA_OPTS=-“Dliquibase.x=a -Dliquibase.y=b”.

Property Definition

liquibase.ignoreRecycleBinWarning=<false|fasle>

Doesn't output the warning if the value is set to true, and if you connect to Oracle using a user that does not have access to the system recycle bin.

liquibase.shouldRun=<true|false>

Defines whether will run. If the value is set to false, will not run.

liquibase.outputLineSeparator

Specifies the line separator for the output. Default value is set to OS default one.

liquibase.changeLogLockWaitTimeInMinutes

Specifies a number of minutes to wait for the lock to be available. Default value is: 5.

liquibase.changeLogLockPollRate

Specifies a number of seconds to wait between checks to the lock when it is locked. Default value is: 5.

liquibase.convertDataTypes

Defines whether converts to/from STANDARD data types. Applies to both snapshot and update commands.

liquibase.outputEncoding

Specifies the encoding to output text in. Default value is set to file.encoding system property or UTF-8.

liquibase.generateChangesetCreatedValues

Defines whether includes a 'created' attribute in diff/generateChangeLog s with the current date time. Default value is: false.

liquibase.autoReorg

Defines whether automatically includes REORG TABLE commands when it is needed. Default value is: true.

liquibase.diffColumnOrder

Defines whether compares column order in the diff operation. Default value is: false.

Liquibase.alwaysOverrideStoredLogicSchema

Defines whether the procedure schema is forced to the default schema if no schemaName attribute is set and when an SQL is generated for createProcedure. Default value is: false.

liquibase.generatedChangeSetIdsContainDescription

Defines whether includes the change description in the id when generating s. Default value is: false.

liquibase.shouldSnapshotData

Defines whether makes snapshots of data by default. Default value is: false.

liquibase.filterLogMessages

Defines whether filters log messages for potentially insecure data. Default value is: true.

liquibase.hub.mode

Controls the level of data sent to Hub. The values are all, meta, or off. Default value is: all.

liquibase.pro.markUnusedNotDrop

Calls markUnused if a column will be dropped in diffChangeLog and if the value if set to true. Default value is: false.

liquibase.pro.synonyms.drop.public

Doesn't drop public synonyms in diffChangeLog or dropAll if the value is set to false. Default value is: true.

liquibase.pro.sql.inline

Generates s with SQL-based changes embedded instead of saving them to an external file if the value is set to true. Default value is: false.
