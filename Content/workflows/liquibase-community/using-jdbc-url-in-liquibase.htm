Using JDBC URL in

A JDBC URL is a database connection URL that identifies a database so that the appropriate driver recognizes it and connects to it. requires a JDBC driver to establish a connection between and your database.

All JDBC URLs begin with jdbc:. The format of other string elements depends on the JDBC driver. In most cases, the format of the database connection URL resembles the following example:

jdbc:<database>://<host>:<port>/<database_name>;<URL_attributes>
database name

Name of the database to which you want to connect. Can also be the file system path to the database.
URL attributes

Supported s of the database connection URL, such as upgrade=true and create=true.
host

Name or address of the machine on which the server is running.
port

Port number that the server framework uses.

To use the JDBC URL, complete one of the following tasks:

  • Run the --url parameter in the CLI:

    • --url=jdbc:<database>://<host>:<port>/<database_name>;<URL_attributes>

  • Configure LIQUIBASE_COMMAND_URL as an environment variable to set it once and pass connection information to and your database. Environment variables are only available in .

    • LIQUIBASE_COMMAND_URL =jdbc:<database>://<host>:<port>/<database_name>;<URL_attributes>

  • Specify liquibase.command.url in the file to set it once and pass connection information to and your database.

    • url: jdbc:<database>://<host>:<port>/<database_name>;<URL_attributes>

Command line arguments override environment variables and the file data. Environment variables override the file data. The file does not override any values.

JDBC URL examples

Database URL
Apache Derby jdbc:derby://localhost:1527/yourdatabase;create=true
AWS Redshift

jdbc:redshift://endpoint:5439/yourdatabase

AWS example: jdbc:redshift://yourcluster-identifier.us-east-1.redshift.amazonaws.com:5439/yourdatabase
Cassandra

jdbc:cassandra://localhost:9042/yourKeyspace;DefaultKeyspace=yourKeyspace

Cassandra on DataStax Astra

jdbc:cassandra://localhost:9042;DefaultKeyspace=keyspace;AuthMech=2;UID=username;
PWD=password;SecureConnectionBundlePath=path/to/secure-connect-database_name.zip;TunableConsistency=6

For more information, see Using [%=General.Liquibase%] with Cassandra on DataStax Astra.
