Quality Checks Workflow

The typical quality checks workflow looks as follows:

DBAs, Architects, Developers and others in your organization may have set specific standards for database changes and security roles. quality checks can be configured (copied, customized, enabled, disabled) on the command line to meet these standards, which are stored in an auto-generated configuration file called liquibase-checks-settings.conf. Your organization's teams can have multiple configuration files.

Next, a team obtains configuration files for a specific project from source code control. It is best practice to keep the checks configuration with the in source code control so that it is delivered to anyone who works with the .

Then, from the command line, you can navigate to the location of the and configuration files and run checks subcommands to configure and execute checks. For more information, see Running [%=General.Liquibase%] Quality Checks Commands.

Changes, such as new copies, customizations, enabled status, and disabled status, are saved in the checks settings file, which you can commit to source control to be used in development, on-demand, and automated quality checks executions.
