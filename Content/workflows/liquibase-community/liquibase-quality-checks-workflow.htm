Quality Checks Workflow

The typical quality checks workflow looks as follows:

Your organization may have specific standards for database changes and security roles defined and given to an Architect, DBA, Developer, or any other employee. When those standards or rules are defined, anyone can configure the checks for a given application or team.

The next steps will be to obtain configuration files for a specific project from source code control. It is best practice to keep the checks configuration with the in source code control so that it is delivered to anyone who works with the .

Then, you can open the command line, navigate to the location of the and configuration files and run checks subcommands to configure and execute checks. For more information, see Running [%=General.Liquibase%] Quality Checks Commands.

Commands save the changes in the checks settings file, and you can commit the checks settings file to source control to be used in development and automation.
