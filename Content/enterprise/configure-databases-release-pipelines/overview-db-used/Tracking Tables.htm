Tracking Tables

Required for . Tracking tables are created in every database managed by and track the deployment status of change sets in the change log for the database. The change log represents the state of the database as a set of all changes (change sets) used to achieve the current state.

  • Single schema projects - use the same schema as the database changes.

  • Multi-schema projects - create or identify a separate schema to use for the tracking tables.

See Tracking Tables.
