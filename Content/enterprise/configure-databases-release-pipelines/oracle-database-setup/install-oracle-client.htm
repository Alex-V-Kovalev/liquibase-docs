Installing the Oracle Database Client

Client requirements are the same for on-premise Oracle and Oracle on RDS.

The Oracle client version must match the version of Oracle you are running. See the versions of Oracle supported at Hardware, Software, and Database Requirements.

Deploy Packager Backup and Restore options for Oracle

There are two different configuration options for the backup and restore options in Deploy Packager. Only the second option requires installing the Oracle EXPDP/IMPDP client.

1. Datapump API Backup and Restore

With 6.7 or higher, you can use the Datapump API Backup and Restore option. With this configuration you do not need to install an Oracle client. This is the simpler configuration option.

Note that an important fix for the Datapump API Backup and Restore method was added in 6.16. We recommend upgrading to 6.16 (or higher) if you are using the Datapump API method.

Please see: Configuring Deployment Packager to Use the Oracle Datapump API for Backup and Restore

2. EXPDP/IMPDP Backup and Restore

The default backup and restore option requires installing the Oracle client for EXPDP/IMPDP and putting it in the PATH. Note that the version of the client needs to be compatible with your Oracle database version.

The following instructions are examples for Oracle Database 11g. You may need to use different links to download client files for other versions of Oracle.

Windows

  1. For the packaging database only, install database utilities expdp and impdp from the following link: http://www.oracle.com/technetwork/database/enterprise-edition/downloads/112010-win64soft-094461.html

    1. Select Oracle Database 11g Release 2 Client (11.2.0.1.0) for Microsoft Windows (x64) or current version.

    2. Select Install Type "Custom"

      • Select Component Name "Oracle Database Utilities"

  2. Make sure expdp and impdp are installed and in the PATH. These are Oracle programs that are used to create a full export and perform a full restore.

    1. Note: When packager runs impdp on a remote database, the export file (.DMP) will reside on the remote database.

Linux

  1. For the packaging database only, install database utilities expdp and impdp from the following link:
    http://www.oracle.com/technetwork/database/enterprise-edition/downloads/112010-linx8664soft-100572.html

    1. Select Oracle Database 11gRelease 2 Client (11.2.0.1.0) for Linux x86-64 or current version.

    2. Download linux.x64_11gR2_client.zip

    3. Unzip linux.x64_11gR2_client.zip

      • Refer to included Oracle Database Client Installation Guides e10841.pdf and e10861.pdf

  2. Make sure expdp and impdp are installed and in the PATH. These are Oracle programs that are used to create a full export and perform a full restore.

    1. Note: When packager runs impdp on a remote database, the export file (.DMP) will reside on the remote database.

Connection Identifier Properties for Oracle

When connecting to an Oracle database, fields are set that identify .

  • JDBC connection

    • v$session.program is set to "Datical Client"

  • SQL*Plus connection

    • v$session.client_info is set to "Datical Client"

This information may be useful in the context of database connection audits.
