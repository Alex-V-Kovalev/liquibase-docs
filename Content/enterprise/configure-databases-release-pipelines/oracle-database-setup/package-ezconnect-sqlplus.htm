Packaging: Using EZCONNECT with SQLPLUS for Deployment Packager

When you are packaging files in the sqlplus directory, Deployment Packager runs SQLPLUS to load SQL Scripts into the database for packaging purposes.

requires that EZCONNECT be configured in the sqlnet.ora file before it can successfully connect using SQLPLUS.

NAMES.DIRECTORY_PATH=(TNSNAMES,EZCONNECT)

Also note, there can be timing difference for connections on the laptops when the EZCONNECT value is placed in different orders in the sqlnet.ora file. If you normally rely on tnsnames to resolve your connections and you put the EZCONNECT before these values, you will see longer wait times for your connections. It appears to be going through each of the connection protocols before it finds the proper method, which succeeds.
