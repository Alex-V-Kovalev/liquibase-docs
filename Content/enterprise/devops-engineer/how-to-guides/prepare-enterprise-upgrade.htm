How To: Prepare for a Liquibase Enterprise Upgrade

Before beginning an upgrade, you should complete the following tasks to ensure that the process goes smoothly.

  • Determine the target version for the upgrade.

    • The recommended practice is to upgrade to the latest version available.

    • Check the DMC Compatibility Matrix to make sure that your version of Liquibase Enterprise will work with your version of DMC. In most cases you will need to upgrade both Liquibase Enterprise and DMC.

  • Download the software from software.datical.com.

    • Instructions for downloading the software can be found here: Downloading the Installer .

  • Locate your Liquibase Enterprise license file.

    • You will need to reinstall the license file as part of the upgrade process.

  • Identify all servers and laptops where Liquibase Enterprise and DMC are installed.

    • All servers must be upgraded at the same time. Running multiple versions of Liquibase Enterprise on the same project is not supported.

  • Verify that you can login to the servers where Liquibase Enterprise and DMC are installed.

    • For Liquibase Enterprise, you must be able to login as the owner of the software.

    • For DMC you will need sudo access to perform the upgrade.

  • Schedule downtime for your CI/CD pipeline.

    • CI/CD jobs should not be running while the upgrade is in progress.

  • Review the documentation to make sure that you are familiar with the upgrade steps.

    • Liquibase Enterprise upgrades are performed by installing the new version of the software in a different location.

      • Installing on Windows Systems

      • Installing on Linux Systems

    • DMC upgrades are performed by running the new installer and providing the location where the current version of DMC is installed.
