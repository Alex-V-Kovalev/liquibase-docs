Interaction Between Datical and Customer Components

Customer Components

Datical Service works with your IT infrastructure. 

  • AUTH Service - Active Directory/LDAP authentication service. Datical Service uses it to define users and groups and to authenticate logins. 
  • SCM SQL Code Repository - You check SQL code into an SCM repository to enable automated database change deployment. 
  • Automation Infrastructure - The Automation Client provides the means for ARA solutions to run Datical operations (Forecast, Deploy, other). 
  • DB Release Pipeline - Datical projects define the sequence and database connections to database connections in your release pipeline. Datical operations forecast and deploy database changes through the pipeline.  

Datical Service Components

Datical Kubernetes

(Not Shown in Diagram). Datical Kubernetes manages the Docker images. It monitors them and makes sure they are running, intervening as needed.

Datical Service

Datical Service runs as an application server.  It is made up of a collection of Docker-based micoservices. 

Databases Used by Datical Service

Datical Service requires two databases:

  1. Application database: used as a service component to store all data on projects, users, groups, and system settings. Must be 
  2. Audit database to store historical, reporting and auditing information. This database can be Oracle, SQL Server, IBM DB2, or PostgresSQL. 

Automation Client

Your ARA solution uses the automation client to run Datical operations. Examples of ARA solutions supported are: 

  • IBM UrbanCode Build
  • IBM UrbanCode Deploy
  • Jenkins
  • Oracle Release Automation

The automation client functionality is provided by Datical DB. You install it where the ARA solution can use it and then use Datical CLI commands in the automation steps. 

User Interface - Deployment Management Console (DMC)

DMC provides the user interface to Datical Service. 

Features

  • User Management
    • Authentication: Users and groups are be provided by your authentication service (AD/LDAP). Logins are validated by accessing the service. 
    • Authorization: Role-based authorization allows you to manage permissions  You assign predefined roles to groups and assign users to groups. 
  • Project management
    • Import existing projects (created by Datical DB)
    • Create projects directly
    • Edit projects
    • Manage connections to the databases in the release pipeline. Database connections are configured on steps in a pipeline. A project consists of one or more pipelines. 
  • Datical Operations
    • Enable or disable the capability to perform forecast and deploy operations directly from DMC. Disable it if you use ARA solutions exclusively for operations. 

Datical DB

Datical DB plays the following roles with Datical Service. 

Desktop Client

  • CLI - Run hammer commands from the CLI that use Datical Service credentials to work on projects stored in Datical Service. 
  • GUI - Use as a development platform for projects before importing them into Datical Service. 

Automation Client

  • Automations (using ARA solutions) use Datical DB to run CLI commands in automation steps. 

Packaging

  • Use Datical DB to package SQL code, either from the desktop or through automations. SCM automation may also trigger packaging jobs.
