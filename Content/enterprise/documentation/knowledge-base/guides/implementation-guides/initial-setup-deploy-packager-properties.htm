Implementation Guide: Initial Setup of "deployPackager.properties" File


The Deployment Packager provides end-to-end automation for the onboarding of database changes in with capabilities including: automated change validation & approval; source code control repository management; database environment backup and restore.

For more information on Deployment Packager, please refer to Implementation Guide: Deploy Packager Configuration.

Here is a sample deploypackager.properties file:


databaseBackupRestoreLocation=DATICAL_DATA_PUMP_DIR

# Use this property to avoid installing Oracle Full Client on CI agents
databaseBackupRestoreMethod=BackupRestoreOracleDatapump


sqlScmSystem=git
enableBackupRestore=REF
sqlScmPath=../Datical/datical-sql
sqlScmSQLBaseDir=sql_code

# LastImportID will automatically update after each successful packager job
sqlScmLastImportID=3afe806
sqlScmBranch=release/1.0.0


NOTE: There are a lot of underlying details in order to make this work. Please review the Deployment Packager user guide: Using the Required deployPackager.properties File
