Configure projects to use the

Prior to switching a project from AuditDB to DMC DB, make sure that the Datical DB version is compatible with the version of DMC by checking the compatibility matrix. To switch from Audit DB to DMCDB you will need to remove the old Audit DB settings from the Datical project and configure the project to use the DMC DB. Learn how to do both of those tasks below. Once configuration is complete, your existing projects will use the so that status and reporting information can be shared in the .

Data will NOT be transferred from the to the . You should retain the if you need to reference historical data.

Maintain backwards compatibility with

To maintain backwards compatibility with the the new client will check that database connection information provided in a project corresponds to the new . If it does not, the client will recognize or create the schema objects and will operate in legacy mode using the .

As a result, installation and configuration of the should be completed BEFORE configuring a project to use the . Configuring the project to use the prior to installation and configuration of the will cause the installation to fail due to the presence of the old schema objects.

If your projects were previously configured to use the , you will need to remove the configuration to use the new . You can remove the manually or by using the GUI.

Remove the using the GUI:

  1. Choose the necessary pipeline and go to Settings located above the Console tab.
  1. Select Database, and then select Clear Configuration.

The Clear Configuration button will remove the auditDB or dmcDB entry from the daticaldb.project file without removing information from tracking tables in the auditDB or dmcDB database.
