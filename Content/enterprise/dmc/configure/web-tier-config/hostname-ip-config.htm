Configure your hostname

There are a variety of reasons why you may need to change or update the hostname of your web server.

Changing the hostname

To change the hostname, type the following command from a terminal logged into your host:

sudo datical-control network set-external-hostname --hostname <new-hostname>

When to update the hostname on all existing projects

When working with Liquibase Enterprise projects, it's important to understand when you need to update the hostname. If you make changes to the hostname in DMC, you may need to make corresponding changes to all of your existing projects. Here are some scenarios to help you determine whether or not you need to update your datical.project file.

Scenario 1: is in Production Mode

If is running in production mode, where it runs on one server and its database is stored on another server, there's no need to make any changes to the datical.project file.

Scenario 2: DMC is in Demo Mode and the Hostname Has Been Changed

If is running in demo mode and you change the hostname using the following command: 

sudo datical-control network set-external-hostname --hostname <new-hostname>

then you will need to update the datical.project file.

Scenario 3: DMC is in Demo Mode and the Hostname Has Been Changed, But the IP Address Has Not

If is running in demo mode, and the hostname is set to a specific value but the datical.project file contains the IP address of the server (not the hostname), and you change the hostname but not the IP address, then you don't need to update the datical.project file. The server will still communicate via IP, which has not been changed.

By understanding these scenarios, you can determine whether or not you need to update the datical.project file when making changes to the hostname in DMC.
