Implementation Guide: Create subversion ignore list In Project Directory

            svn propset svn:ignore

While performing normal operations (Packager, Forecast, Deploy, etc.) will generate log files and reports that are not necessary to be checked into Subversion. 

This is implemented using a "svn propset svn:ignore" command. Therefore, files and directories generated as a result of running various tasks should be ignored. This document will help you setup a subversion to ignore those files in your project.

Some files generated as part of Datical operations, such as HTML reports, are important to be archived. However, SVN is your source code repository, not a means to archive reports. Consider archiving reports into a separate directory or use a different tool such as Artifactory.

STEPS

  1. Open command line
  2. Navigate to your Datical Project Directory 

  3. Enter the following command 

    svn propedit svn:ignore .
  4. This will bring up an editor (if not, set your SVN_EDITOR and point it to vi or notepad depending on your OS)

    1. Put the following values in:

      daticaldb*.log
.classpath
Logs
Reports
Snapshots
Profiles
templates

  5. Now, when you check-in your files, SVN will use the "svn:ignore" property to identify which files to prevent from checking in. Steps to check-in your code into SVN is described in this page: /wiki/spaces/DDKB/pages/6520833
