Implementation Guide: Initial Commit of Project into Team Foundation Server (TFS)

Prerequisite:

  1. Designate a new location for project in TFS. Initially, this will be an empty directory.
    1. For example, $\tfsserver\OrgX\TeamY\ProjZ\Datical
  2. Must have access to TFS and permissions to check-in code into designated location.
  3. You should already have a ".tfignore" file in place: Implementation Guide: Create ".tfignore" file In Project Directory
  4. [OPTIONAL] Visual Studio installed locally on your PC in order to access VS Source Control Explorer

Steps:

  1. In GUI, right-click on the project → Team → Share Project
  2. Select Team Foundation Server and select Next
  3. Select Connect to a Team Foundation Server or Team Services account option. If your server is not already listed in the pulldown list below, then select Servers… and add your TFS server there. Once your server is correctly added, click Next.
  4. You will be asked to select a team project from the list. Browse to your team project, e.g., TFS_Proj. Click Next.
  5. You will be prompted to select a TFS workspace. Use an existing TFS workspace or Add a new workspace. Then click Next.
  6. Based on your selected workspace, you will now point to a shared project on TFS. Browse to the folder where you want your project to be stored.
    1. Notice that the project folder path where your projects (e.g., HarmonyAccounts) will be created.
  7. Click Next.
  8. You will now be presented with a Confirmation screen with how your project will be setup in TFS.
  9. Click Finish.
  10. Your project in GUI will now have a new icon to represent that it is now connected to TFS.
  11. We are ready to check-in our project. Right-click on the project name → Team→ Check In Pending Changes
  12. Provide a commit message, usually "Initial check in" for the first check in.
    1. Ensure that all files are checked, i.e., they all have a checkbox next to their name
