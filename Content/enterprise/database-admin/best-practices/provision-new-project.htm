Implementation Guide: Provision New Project

This section is used for creating a new project from scratch.

In order to create a new project, the user is prompted to setup the first project upon launching GUI for the first time. For other users, take the following actions in order to create a new project:This section helps user setup two pipelines in : MAIN and HOTFIX.

  1. Right-click in the Deployment Plans view → New→ Deployment Plan
  2. In the New Deployment Plan window, provide a Project Name and Pipeline Name:
    1. Project Name: Enter project name, e.g., InsWeb_ddb
    2. Pipeline Name (Optional): MAIN
    3. Single or Multiple Schema Management:Multiple Schema
      1. Note: Even if you have only one schema, still use "Multiple Schema" option. 
  3. Click Next.
  4. Provide a name for the first deployment environment. This is the REF database. Select the database platform for the underlying database.
  5. Click Next.
  6. On Where is your REF1 Database Located? window, provide hostname, port, username, password and Oracle-specific information such as Service Name or SID. Note: This window may look different based on which database platform you selected in the previous screen.
    1. Note: Ensure that "Include Table and Index Storage Information" is checked
    2. Note: Ensure that "Contexts" is set to the correct database
    3. Note: Ensure that "Labels" is set to the correct pipeline
  7. Be sure to click Test Connection button to test connectivity with the database. If there are issues connecting with the database, user will be prompted with connection errors here.
  8. Click Next.
  9. On the Which Schema will be managed in this Deployment Plan? window, provide a name for Managed Schema and Tracking Schema.
    1. "Managed Schema" contains list of schema names that will be managed for this database.
    2. "Tracking Schema" contains the schema name which will use for changelogs
      1. For example, DATICALDBTRACKING
  10. Click Finish.

  12. Repeat these steps above to create additional database environments, e.g.:
    1. DEV
    2. QA
    3. UAT
    4. PROD

In a similar manner, create one more pipelines: HOTFIX.

  1. In the Deployment Plan view, under Pipeline, click MANAGE
  2. In Manage Properties window, click Add
  3. In New Pipeline window, enter the name and description of the new pipeline "HOTFIX"
  4. Click OK.
