Viewing Lists of Changesets

In the Pipeline Summary tab, changes are shown by label. 

Click a label (anywhere in the row) to see the list of all changesets that have that label. 

Searching for Changesets

To constrain the list, enter values in one or more of the following fields, then click Search. 

  • Labels
  • Changesets
  • Contexts

Click Back to to back to the Labels view. 

Wildcards

Use the % character in any field to mean all. 

Status

To constrain the view by status, click the Status menu in a step column, then select a status. The number of changesets at each status is shown in parentheses.

  • All
  • Undeployed - changesets that have not been deployed to the selected step
  • Deployed - changesets that have been deployed to the selected step
  • Ignored - changesets in the changelog but set to ignored

Limits

The number of results is reported in a message under the changesets table. If you exceed the limit, the message changes. 

  • Normal message

    59 total

  • Exceeds Limit

    250 shown - Note: max result size exceeded - use more specific filters and search again to limit results.

The maximum number of changesets in a view is 1000 / (number of steps in the pipeline). 

For example, if the pipeline contains four steps, the maximum number of changesets you can view is 250 (1000 / 4).
