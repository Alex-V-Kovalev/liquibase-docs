Status, Forecast, and Deploy Operations on Projects

Operations are available to users who have the necessary roles. A role may give you permissions to perform the operation at any step in a project (Project role) or only at assigned steps in the project (Step role). 

  • Status - Project Deployer, Project Analyst, Step Deployer, Step Analyst
  • Forecast - Project Deployer, Project Analyst, Step Deployer, Step Analyst
  • Deploy - Project Deployer, Step Deployer

Running Status is Not Usually Necessary

Datical Service updates its own status after operations. All users have their views updated automatically. 

If you see a step that is green but the message is  Undeployed rather than Fully Deployed, then click Status. 

The most likely scenario for this occurs during packaging, which is always done outside of Datical Service:

  1. Deployment Packager adds new changesets to a database.
  2. If there was no Datical DB status run after the packager run, then the audit database is not fully updated with the effects of the packaging operation. Datical Service needs to sync with it.  

Running Operations from DMC

To work with a project:

  1. Log on to DMC. 
  2. Open a project. Select Projects > Project List to see a list, then choose one.



  3. Click Status. 



    Note

    While a status operation is in progress, a notice appears that is visible to all project users. The Operations menu is disabled on all steps until the status operation is finished. 

  4. On a step, select Forecast or Deploy from the Operations menu. 

  5. Change options if desired, then click Finish. Changing the options here overrides the options set in the project.
    • Change labels or contexts
    • Change options

  6. Check status and reports after the operation is complete. While the operation is in progress, you can do the following:
    • View in-progress status on the menu (Deploying, Forecasting) while the operation is in progress. 
    • Perform operations on other steps.
