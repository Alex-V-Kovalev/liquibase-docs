Datical Users

The Users pages lists all users who can log in to the system.  Users with System Administrator role can view and edit users. 

  • To locate a user, scroll the list or use the Search field. 
  • Click a user in the list to view and edit information for the user. 

User Information

The following information is shown for the selected user. 

  • Full Name - First name, last name.  
  • Email - Email address 
  • Groups - List of groups the user belongs to. Go to Admin > Groups to administer Datical Service roles assigned to groups.   

Adding a Role

Click a name in the Users list to see the roles assigned to that user.  

It is common practice to assign roles to groups rather than individual users. You can also assign individual roles to a user. 

  1. Select a user. The existing roles are shown. 
  2. Start with Choose Role. The next choices are highlighted as you go. 
    1. Choose Role - Choose the role to assign. 
    2. Choose Project - For project and step roles
    3. Choose Pipeline - For step roles
    4. Choose Step - For step roles
  3. Click Add Role. 

See Roles and Permissions for more information on role capabilities. 

Integration with Active Directory / LDAP

Users, groups, and group membership are created and managed through integration with Active Directory LDAP. You configure the integration in Keycloak. 

You can also manually create users, groups, and group membership manually in Keycloak. 

During installation you created an administrative user for Keycloak.  You may have specified integration information at that time. 

Accessing Keycloak

To access Keycloak, go to Admin > Keycloak and log in as the administrative user. 

See Configuring Datical Service Users in Keycloak for more information on setting up users.
