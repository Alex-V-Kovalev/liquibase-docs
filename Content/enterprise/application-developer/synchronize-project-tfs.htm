User Guide: Synchronize Project (TFS)


This documentation shows you how to synchronize your project from GUI.

Prerequisites

It is assumed that your project is already connect to TFS and that an initial commit was previously performed. See Implementation Guide: Initial Commit of project to TFSfor more information on connecting your project to TFS.

Overview

There are three scenarios that are possible here:

  1. Someone made changes to project and the user would like to obtain those changes from TFS. 
    1. This is a "Get Latest Version" use case
    2. This is typically the case when Deployment Packager ran via an automation tool and it created changes to your project.
  2. Prior to making any changes, user needs to checkout the project from TFS so that the project is no longer in read-only mode
    1. This is a "Check Out for Edit" use case
  3. User makes changes to project and would like to commit those changes into TFS
    1. This is a "Check In Pending Changes" use case


It is a best practice to always perform a "Get Latest Version" prior to doing a "Check In Pending Changes".

Even if you didn't make any previous changes from the Datical GUI, an automation job may have made changes to the project by way of running a Datical Packager job via command line and pushed changes into TFS. Therefore, always perform a "Get Latest Version" prior to doing a "Check In Pending Changes" from Datical GUI.


Obtain Changes from TFS – "Get Latest Version"

  1. Right-click on the project → Team → Get Latest Version
  2. If you get a message that says "All files are up to date", then that means you already have the latest version available to you.

Check Out Project Before Making Changes

In order to modified a TFS-managed project in GUI, make sure to perform a "check out". 

  1. Right-click on the project → Team → Check Out for Edit ...
    2. You will see a confirmation dialog box. 
    3. Click OK.


Check In Changes into TFS - "Check In Pending Changes"

When you have made changes to project you can check in those changes into TFS using built in capabilities implemented in the GUI.

  1. Right-click on the project → Team → Check In Pending Changes
  2. You will be asked to provide a comment for your Check In:
  3. Click Check In.

In order to modified a TFS-managed project in GUI, make sure to perform a "check out". 

  1. Right-click on the project → Team → Check Out for Edit ...
