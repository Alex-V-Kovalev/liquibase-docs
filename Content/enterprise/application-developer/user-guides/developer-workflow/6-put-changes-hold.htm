6.0. Putting one or more changes "on hold" (and re-enabling the change)

If you no longer want to deploy a particular script further into the pipeline (e.g., if you want to hold or prevent you changes from deploying):

  • Check in an empty file (0 byte file) with -ignore in the file name. 
    • For example, if you want to hold/prevent changes in script "member_siblings_tab.sql", then check in an empty file with this name: "member_siblings_tab-ignore.sql"
    • Note: In Datical changeset, this will set the "ignore" attribute to "true"

If you previously ignored a particular script which now needs to be unignored:

  • Check in an empty file (0 byte file) with -unignore in the file name.
    • For example, if you want to unignore script "member_siblings_tab.sql", then check in an empty file with this name: "member_siblings_tab-unignore.sql"
    • Note: In Datical changeset, this will set the "ignore" attribute to "true"


Using GUI

If you use user interface, you can also interactively add or remove ignore="true" attribute to your change sets.

Add ignore="true" attribute

  1. In your project view, go to the "Design" tab. Expand the appropriate change set you wish to ignore, right-click → select Add Attribute → ignore.
  2. You will notice that a new attribute "ignore" is now added. It defaults to empty string.
  3. For its value, type "true", as shown in the screenshot below.
  4. Be sure to save you project (File menu → Save). And also commit or check-in your changes to your source code.


Remove ignore="true" attribute

If you need to now un-ignore your change set:

  1. In your project view, go to the "Design" tab. Expand the appropriate change set you wish to un-ignore, right-click on the "ignore" attribute → select Remove
  2. Be sure to save you project (File menu → Save). And also commit or check-in your changes to your source code.
