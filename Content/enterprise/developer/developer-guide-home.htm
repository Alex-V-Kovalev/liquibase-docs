Get Started with as a Developer

There are a few elements that directly impact the developer's workflow with . Those elements include scripts, metadata.properties files, and packaging. Understanding the behavior and functionality of each of these items are essential to your success. The following documents are designed to help you use .

Specifically, you need to know what folders hold scripts, how to inspect a script if it fails to run, what happens to scripts, and where they are written from. Metadata.properties are also vital to the Enterprise workflow. Developers need to understand the rules in which metadata.properties files are written, and how to read and inspect the metadata.properties file to find out the folder structure. Understanding how packaging works for the DBA and how that affects the Developer workflow is also a key component. You will find each of these topics covered in detail below.

  • how to inspect scripts if they fail to run
  • what happens to scripts and where the scripts are written to

metadata.properties files
