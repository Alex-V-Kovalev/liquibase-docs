diff

Compares two database definitions in a project and produces a text report of the differences.

USAGE 

diff <referenceDB> <targetDb> [--output=/path/to/report]

EXAMPLE 

diff myAppDevDB myAppQaDB --output=<path>

OPTIONS

  • ddlExcludeList – comma-separated list of object types to exclude from the snapshot. Can be all, none, or a list containing any combination of procedure, function, package, packagebody, trigger, or table view.

    non_stored_logic is available in 8.3 and later. When non_stored_logic is provided for the ddlExcludeList property this will exclude Snapshot, Diff, and DiffChangelog.

  • referenceDB – database that represents the desired state

  • targetDB – database that is out of synchronization and needs to be caught up to the state of referenceDB.

  • output – If present, specifies a file for the report. If not specified, the report is sent to STDOUT
