Initial Commit of Project into Git

The following document will help you commit your new project into a Git repository

Prerequisites

  1. Setup a new Git repository. Initially, this will be an empty repository
  2. (OPTIONAL) Configure Git for the first time: 
    git config --global user.name "<user_name>"git config --global user.email "<user_email>"

Steps:

  1. We need to make sure to ignore a few files from going into Git. These include log files, reports, etc. which are generated as part of running Forecast or Deploy operations in .

    1. .gitignore
      Logs/
Snapshots/
Profiles/
Reports/
.auditdb/
.classpath
.metadata
.reporttemplates
daticaldb.properties
daticaldb*.log

    2. For more details, look into Implementation Guide: Create ".gitignore" file In Project Directory to setup a .gitignore file.

  2. In GUI, right-click on the project → Team → Share Project
  3. Click the checkbox for "Use or create repository in parent folder or project"
  4. Select the project. Notice that "Create Repository" button is now enabled.
  5. Click "Create Repository" button and then click "Finish"
  6. At this point, your project is configured for "Local" Git repository. You still need to configure a "Remote" repository in Git.
  7. We are ready to commit our project. Right-click on the project name → Team → Commit
  8. Provide a commit message, usually "Initial commit" for the first commit.
    1. Notice that since we already have the ".gitignore" file in place, we see only a limited number of files in the Unstaged Changes section.
    2. Click on the "+++" icon to add all files to Staged Changes
    3. Provide a Commit Message, e.g., "Initial commit"
    4. Click "Commit and Push" 
  9. Copy the Git repository URL and past it in the URI field:
    3. Be sure to provide username and password for connecting to your Git repository.
      1. SSH only: It may be necessary to setup SSH keys in your Git provider prior to this step. If that's the case, then log into your Git provider's web interface as project admin, go to settings and add your Access Keys. Read this link for more information on using SSH keys.
    4. Click "Next".
  10. Review the information and click "Next".
  11. Final confirmation screen is now displayed before performing the "git push" operation. Click "Finish".
