Getting a Project From Git


This section describes how to set up a Git repository and check out a project (after someone else from your team has already configured and shared that project in Git).

Git Repository Setup in (versions 6.7 and higher)

You do this only once. 

Note

The following instructions use an example Git repository. Substitute the repository and branch for your development environment. 
  1. In the GUI, go to File menu and select Import
  2. Choose Git > Projects from Git
  3. Click Next >
  4. Select Clone URI
  5. Click Next >
  6. In the URI field, copy and paste the Git clone URL from your Git repository. Rest of the fields should automatically populate based on the URI string.
    2. You may need to provide a User/Password for authentication if the URI is http-based. If the URI is ssh-based, then ensure that there is a public key provided for authentication with the Git repository.
  7. Click Next >
  8. Select the desired branch, then click Next. 



  9. Verify the Local Destination for the repository, then click Next. 



  10. The project is cloned from the repository to a local repo:
  11. Click Next >
  12. You will now see the final screen showing the project imported locally:
  13. Verify the Repository shows up in your Deployment Plans.

Check Out Project From Git (only versions 5.12 or older)

  1. Import projects from the new repository.



  2. Choose the project to import 



  3. Verify the final project settings.



  4. Switch to the Deployment Plans perspective and verify the project.
