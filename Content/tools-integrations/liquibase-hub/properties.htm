has a list of properties to be set so that your local activity is reported to a specific Project.

Required properties

Specifying the API Keys is the required property to connect to along with a registered file.

There are three ways to specify your API Key:

  1. Including liquibase.hub.ApiKey value in the file:
liquibase.hub.ApiKey: <addYourHubApiKeyHere>

An example of the file with the API Key

# liquibase.properties
changeLogFile: ../path/to/file/dbchangelog.xml
driver: oracle.jdbc.OracleDriver
url: jdbc:oracle:thing:@192.168.0.22:1521/orcl
username: PRO
password: password
referenceDriver: oracle.jdbc.oracleDriver
referenceurl: jdbc:oracle:thin:@192.168.0.22:1521/orcl
referencePassword: password
liquibaseProLicenseKey: <addYourProLicenseKeyHere>
classpath: ../path/to/file/ojdbc6-11.2.0.3.0.jar
# Add Your First Liquibase Hub API Key
liquibase.hub.ApiKey: <addYourHubApiKeyHere>
  1. Passing liquibaseHubApiKey on the command line:
--liquibaseHubApiKey=addYourHubApiKeyHere

An example of the API Key running on the command line

java -jar liquibase.jar \
--driver=oracle.jdbc.Oracleoriver \
--classpath=../path/to/file/ojdbc6-11.2.0.3.0.jar \
--changeLogFile=com/example/db.changelog.xml \
--url="jdbc:oracle:thing:@192.168.0.22:1521/orcl" \
--username=scott \
--password=tiger \
--liquibaseProLicenseKey=<addYourProLicenseKeyHere>\
--liquibaseHubApiKey=<addYourHubApiKeyHere>\
update
  1. Specifying API key in your JAVA_OPTS Environment Variable as -Dliquibase.hub.apikey.
    1. On Linux or MacOS: JAVA_OPTS="-Dliquibase.hub.apikey=<myapikey>"
    2. On Windows: set JAVA_OPTS="-Dliquibase.hub.apikey=<myapikey>"

You can also manually set JAVA_OPTS as an environment variable in your system.

Optional properties

The URL, which is https://hub.liquibase.com, is specified by default when you sign up for an account and connect to with the API Key.

There are two ways to change the location property, select one of the following two options:

  • Include liquibaseHubUrl value in the file:
liquibaseHubUrl:https://hub.liquibase.com
  • Specify URL in your JAVA_OPTS Environment Variable as -DliquibaseHubUrl: 
    • On Linux or MacOS: JAVA_OPTS="-Dliquibase.hub.url=https://hub.liquibase.com"
    • On Windows: set JAVA_OPTS="-Dliquibase.hub.url=https://hub.liquibase.com"

You can also manually set JAVA_OPTS as an environment variable in your system.

Another optional property is liquibase.hub.mode, which represents the level of data you want to send to . The default value is: all.

liquibase.hub.mode:[all=default|meta|off]
mode Definition

all

Sends all data, including the body, generated SQL, and logs (the default value)

meta

Does not send the body, generated SQL, or operation-event and change-event logs

off

Does not send any data to

To specify the level of data, select one of the following options:

  • Include liquibase.hub.mode value in the file:
liquibase.hub.mode:meta
  • Specify the mode in your JAVA_OPTS Environment Variable as -DLiquibaseHubMode: 
    • On Linux or MacOS: JAVA_OPTS="-DliquibaseHubMode=[all|meta|off]"
    • On Windows: set JAVA_OPTS="-DliquibaseHubMode=[all|meta|off]"

You can also manually set JAVA_OPTS as an environment variable in your system.

Additionally, you can have different mode settings for different connections.

For example, if sensitive DML updates are only made to the PROD connection and not to lower-level connections, you can have PROD in the meta mode and all lower connections in the all mode.

You can specify the liquibase.hub.mode in your file, as a system property with -DliquibaseHubMode, as an environment variable LIQUIBASE_HUB_MODE, or store it in a liquibase.prod.properties file.

If you specify the mode only in one place, will use it as the default. If you have the mode specified in multiple places, will use the following order of precedence to determine which value to use:

  • The liquibase.prod.properties file
  • The file
  • The LIQUIBASE_HUB_MODE environment variable, which overrides all of the preceding places

If you have policies and compliance requirements that prohibit sensitive information (PII or HIPAA) from leaving the controlled domain, use liquibase.hub.mode=meta. The meta mode is recommended if your scripts contain data that is regulated by government compliance standards, such as HIPAA, PCI, and others.
