Getting Started with Hub

Hub is a secure portal that provides the real-time monitoring, reports, organization, and insights into your team’s database release automation. It provides centralized visibility into distributed operations performed against the multiple environments.

The CLI app sends information to the Hub portal when you run the commands that make changes to your database. For example, update, changeLogSync, rollback, or dropAll.

To set up your account, follow these steps:

Step 1: Sign up for an account

  1. Go to the Hub portal.
  2. Select Sign up and enter your username, email address, and password to confirm registration.
  3. Select Let’s go to receive the email with a link for your email address validation.
  4. Check your email and follow the link to activate an account.

Step 2: Add a Hub API key to the file

  1. Sign in to your Hub account and go to Settings > API Keys by selecting the settings icon on the left side of the page.
  2. Copy the API key, which connects the information generated by your s and other operations to your Hub Projects. The API key is a new concept for organizing your work, and it is unique for each Hub user. Hub will only be in sync with your actions if the API key is added.
  3. Add your API key to the file as follows: liquibase.hub.ApiKey: <addYourHubApiKeyHere>.
# liquibase.properties
changeLogFile: ../path/to/file/dbchangelog.xml
driver: oracle.jdbc.OracleDriver
url: jdbc:oracle:thing:@192.168.0.22:1521/orcl
username: PRO
password: password
referenceDriver: oracle.jdbc.oracleDriver
referenceurl: jdbc:oracle:thin:@192.168.0.22:1521/orcl
referencePassword: password
liquibaseProLicenseKey: <addYourProKeyHere>
classpath: ../path/to/file/ojdbc6-11.2.0.3.0.jar
# Add Your First Liquibase Hub API Key
liquibase.hub.ApiKey: <addYourHubApiKeyHere>

Alternatively, you can pass your API key as a runtime argument and run your commands as usual or you can specify it in your JAVA_OPTS as -Dliquibase.hub.apiKey. For example, to specify JAVA_OPTS on Windows, run SET JAVA_OPTS=”-Dliquibase.hub.apikey”. It is a best practice to add the API key to the file or your build files instead of entering it manually every time.

You can also set the level of data you want to send to Hub but this process is optional:

liquibase.hub.mode:[meta|all|off]
  • meta does not send the body, generated SQL, or operation-event and change-event logs
  • all sends all data, including the body, generated SQL, and logs (default)
  • off sends no data to Hub

Step 3: Run the registerChangeLog command

  1. Run the registerChangeLog command to connect an XML or formatted SQL to a specific Hub Project. Hub relies on a new unique identifier in your called changeLogId, which is automatically added by the registerChangeLog command to provide insights, real-time monitoring, and reports.

Register only the master without registering the nested s, if you use this best-practices structure. You can register a to an existing project or create a new one from the CLI as well.

  1. Once the changelogId identifier is added to your , ensure to commit the to source control, so all team members’ changes are recorded in Hub.

Step 4: Run your regular commands

Now, just use or , and Hub will track the following commands:

  • update
  • updateCount
  • updateToTag
  • rollback
  • rollbackToDate
  • rollbackCount
  • rollbackOneChangeSet
  • rollbackOneUpdate
  • changelogSync
  • dropAll

Each command will automatically synchronize the local table with the Hub’s database, but you can invoke this synchronization manually by using the new syncHub command.

So long as you have a registered , a valid Hub API key, and hub.mode=[all|meta], you can use Hub to see what s passed and what failed in your update, rollback, or changelogsync operations. You can see these Operation reports filtered to a specific Project, database connection, or by dates and users.
