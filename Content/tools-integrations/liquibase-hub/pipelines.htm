Pipelines

A Pipeline is a way to collect, sort, and view the status of s deployed to Targets in a Project. It is a way to monitor the flow of your database schema changes across your databases.

Targets represent databases and include your JDBC URL, the operations performed on the database, and the status of the changes deployed to the database. Learn more about Targets in .

Pipelines are constructed from Targets as follows:

  • A Target can be in any Pipeline only once.
  • You can add the same target to multiple pipelines in the same project.
  • Pipelines do not cross Projects, which means all the Targets you want to track in a Pipeline must be in the same Project.
  • By default, orders Pipeline Targets in the sequence they were added. You can reorder these by dragging them into a different sequence.

A Pipeline is owned by an Organization Admin, like all other Organization resources, regardless of who created it. Only the Organization Admin can deactivate, restore, or delete a Pipeline. Any Organization member can create and edit their Pipeline.

Managing a Pipeline

You can see the following information when using pipelines:

  • The list of all Pipelines in your project.
  • The name and description of the Pipeline.
  • The number of Targets and their overview.
  • The date and time when the Target was created and who created it.
  • The last operation with the count of passing and failed s and its date and time.

Creating a Pipeline

If your Project has targets, you can create a Pipeline with these steps:

  1. Go to Projects, select a Project, select the Pipelines tab, and + Create pipeline.
  1. Enter your Pipeline name and description, and then select Add Target.
  1. Pick the Target from the dropdown list and select Add.

    2. The list represents a collection of all Targets in your Project which are not in your Pipeline. After you add the Target to this Pipeline, it is no longer on the list.

  1. Check your Pipeline for the Target.

Sorting Targets in Your Pipeline

By default, sorts targets in the order they are added to the Pipeline, starting from the first Target. If you have several Targets in your Pipeline, you can alter their order with drag-and-drop functionality.

Deactivating and Deleting a Pipeline

You must be an Organization Admin to deactivate or delete a Pipeline. Before deactivating or deleting a Pipeline, please take into consideration the following:

  • If you deactivate a Pipeline, everyone in your Organization will only see it in a deactivated state from the Pipelines list and will not be able to click it for details. After being reactivated, a Pipeline will be available for detail viewing again.
  • If you delete a Pipeline, removes it from your Project for everyone. No data is lost as the Pipeline is a collected view of individual Targets, so no one can check the flow of changes across these Targets easily.

To delete or deactivate a pipeline, follow these steps:

  1. Go to Projects, choose the Project you need, and then select Pipelines.
  2. Choose the needed pipeline and select Edit.
  1. Select the Delete or Deactivate button to make the change.

Removing a Target in Your Pipeline

You can remove a Target from your Pipeline by selecting the Remove button under the Edit Pipeline view. You will only remove the Target from the Pipeline and will not affect any data or status of the Target.
