Hub Auto-Registration and Report Links

is a free central site to help you organize and monitor your local database change activity in real-time, which increases team collaboration and deployment success. Starting with 4.3, you can use two new capabilities of :

  • Become a registered user of in one step
  • Receive a clickable link to immediately see the report of your activity

You can run liquibase update or any other Hub-enabled command in the CLI (see Using Commands in [%=General.Hub%]), and this will give you the option to start sending data to . If you accept the option and provide your email address, then a User, Organization, Project, Connection, and API key are created automatically, and your is registered to your Project. The API key and changelogID will be stored in the existing file, or if you don’t have the file, will create a new one in the current working directory.

Your command will be run, after which a link to the operation report on is written to the console. When you open the link in a browser, you can see that operation’s report.

If you just auto-registered, please complete your initial sign-in using the one-time password sent to your email.

The use of capabilities is currently limited to CLI and will be extended to additional integrations in future releases.

If you use earlier versions of , to register your account, sign up to , retrieve an API key, and copy it to the file. Next, run the registerChangeLog command and your other commands to start sending data and receiving reports.

If you use CI/CD tools and run in automation, the prompt is skipped and the automation continues. In this case, uses Java’s internal methods (or you can set liquibase.headless="true") to detect when running commands without interaction.

Report

If you use and a API key, you can use to monitor your changes through reports. When you run a successful Hub-enabled command, a link to your report will be shown on the console.
