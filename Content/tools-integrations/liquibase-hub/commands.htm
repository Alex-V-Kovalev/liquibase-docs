Using Commands in

When you use commands that make changes to your database, the data is sent to . You can review the data in in Operation reports. For more information, see [%=General.Hub%] Operations.

tracks the following commands:

  • changelog-sync
  • drop-all
  • rollback
  • rollback-count
  • rollback-one-changeset
  • rollback-one-update
  • rollback-to-date
  • update
  • update-count
  • update-to-tag

The commands specific to functionality:

  • init hub connects your local activity to with an API key and sets up the environment to use
  • register-changelog connects your local activity to a specific Project
  • sync-hub synchronizes the local table with
  • deactivate-changelog removes the changelogID from your file so it stops sending reports to

If you use Maven, refer to the Maven register-changelog, Maven sync-hub, and Maven deactivate-changelog documentation to configure the Maven pom.xml file and another required properties.

Each command automatically synchronizes the local table with . If you lose your internet connection or doesn’t respond, will still execute the command you entered.

Requirements

To run commands, specify your connection information as environment variables or in the properties file. For more information, see [%=General.Liquibase%] Environment Variables and Specifying Properties in a Connection Profile. Otherwise, specify properties at the command prompt.

For the CLI app and , you need the following:

  • The API Key. For more information, see [%=General.Hub%] Properties.
  • The registered file. For more information, see the register-changelog command.

    • If you use source control, make sure to commit the so all team changes are recorded in and the same is not registered with multiple times.

  • The level of data to send to . If you set liquibase.hub.mode=off in the properties file, you do not receive data in when running commands. For more information, see [%=General.Hub%] Properties.

After following requirements, run any command to see what s passed and what s failed in your update, rollback, changelog-sync, or drop-all operations. The operation reports are filtered by a specific project, database target, result, or date.
