Using Commands in

When you use commands that make changes to your database, the data is sent to . You can review the data in in Operation reports. For more information, see [%=General.Hub%] Operations.

tracks the following commands:

  • changelogSync
  • dropAll
  • rollback
  • rollbackCount
  • rollbackOneChangeSet
  • rollbackOneUpdate
  • rollbackToDate
  • update
  • updateCount
  • updateToTag

The commands specific to functionality:

  • init hub connects your local activity to with an API key and sets up the environment to use
  • registerChangeLog connects your local activity to a specific Project
  • syncHub synchronizes the local table with
  • deactivateChangeLog removes the changelogID from your file so it stops sending reports to

If you use Maven, refer to the Maven registerChangeLog, Maven syncHub, and Maven deactivateChangeLog documentation to configure the Maven pom.xml file and another required properties.

Prerequisites

To run the commands, specify your connection information as environment variables or in the file. For more information, see [%=General.Liquibase%] Environment Variables and Creating and configuring a [%=General.liquiPropFile%] file. Otherwise, specify properties at the command prompt.

For the CLI app and , you need the following:

  • The API Key.

    • For more information, see [%=General.Hub%] Properties.

  • The registered file.

    • For more information, see the registerChangeLog command.

    If you use source control, make sure to commit the so all team changes are recorded in and the same is not registered with multiple times.

  • The level of data you want to send to . If you set liquibase.hub.mode=off in your file, you will not receive any data in when running the commands.

    • For more information, see [%=General.Hub%] Properties.

Each command automatically synchronizes the local table with .

If you follow all the prerequisites, you can use to see what s passed and what s failed in your update, rollback, changelogSync, or dropAll operations. You can see these Operation reports filtered to a specific Project, database target, or by dates and users.
