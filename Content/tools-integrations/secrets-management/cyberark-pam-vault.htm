CyberArk PAM Vault Extension

You can natively read your CyberArk PAM (Privileged Access Manager) based vault secrets in with the CyberArk PAM Vault extension. If you connect to your vault using the file, you can store sensitive authentication details such as database url, username, and password s in your vault. For more information, see CyberArk PAM documentation.

Supported products

Requirements

  • 4.10+
  • license
  • CyberArk PAM Vault with API enabled
  • RootCA certificate in your Java security keystore (typically cacerts)

Installation

CLI installation

  1. Navigate to the cyberark-pam-plugin repository on GitHub
  2. Download cyberarkpam-vault-<version>.jar and move it to the liquibase/lib directory

Package Manager

Alternatively, you can install this extension with lpm (Liquibase Package Manager).

lpm update
lpm add cyberarkpam-vault

s

You need a license key to use CyberArk PAM Vault. See How to Apply Your [%=General.LBPro%] License Key.

Definition Requirement
--liquibase-pro-license-key Required
--vault-addr URL for CyberArk PAM Vault Server Required
--vault-certfile Certificate for CyberArk PAM Vault Server Optional

Adding RootCA certificate in your Java security keystore

Example:

sudo keytool -importcert -alias CyberArkRoot -keystore /Library/Java/JavaVirtualMachines/adoptopenjdk-11.jdk/Contents/Home/lib/security/cacerts -storepass changeit -file RootCA.crt

If there is an error message showing PKIX exception, you could have imported into the wrong cacerts file. Run liquibase -version to see which JVM is using

To debug the SSL connection, you can add this Java environment variable to your command line: -Djavax.net.debug=ssl:handshake

Usage

Any property in a file can be stored in CyberArk PAM Vault. Use the syntax VAULT_TYPE,PATH_TO_NAME_IN_PAM,FIELD_TO_MAP to tell the vault plugin which properties to populate from the vault:

# Enable Liquibase Pro functionality
liquibase.licenseKey= ***Insert Pro License Key***

# Properties to store in the vault
username= cyberarkpam,UserName,username
password= cyberarkpam,Content,password

# Authentication
vault.addr= ***Vault URL***
vault.certfile= ***Vault Certificate***
