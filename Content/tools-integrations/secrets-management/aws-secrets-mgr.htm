AWS Secrets Manager Extension

You can natively read your AWS Secrets in with the AWS Secrets Manager extension. If you connect to your vault using the file, you can store sensitive authentication details such as database url, username, and password s in your vault. For more information, see AWS Secrets Manager documentation.

Requirements

  • 4.12+
  • license
  • AWS Credentials

Installation

CLI installation

  1. Navigate to the aws-secrets-manager-extension repository on GitHub
  2. Download aws-secrets-manager-<version>.jar and move it to the liquibase/lib directory

Package Manager

Alternatively, you can install this extension with lpm (Liquibase Package Manager).

lpm update
lpm add aws-secrets-manager

Setup and s

You need a license key to use CyberArk PAM Vault. See How to Apply Your [%=General.LBPro%] License Key.

Configure your AWS credentials in any standard method. aws_access_key_id, aws_secret_access_key and aws_region are required.

Usage

Any property in a file can be stored in AWS Secrets Manager. Use the syntax VAULT,SECRET,KEY to tell the vault plugin which properties to populate from the vault. You can also enable support for AWS Systems Manager with VAULT,PARAMETER.

# Enable Liquibase Pro functionality
liquibase.licenseKey= ***Insert Pro License Key***

# AWS Secrets Manager
url= aws-secrets,liquibase-secret,url
username= aws-secrets,liquibase-secret,username
password= aws-secrets,liquibase-secret,password

# AWS Systems Manager
url= aws-ssm,liquibase-secret-url
username= aws-ssm,liquibase-secret-username
password= aws-ssm,liquibase-secret-password
