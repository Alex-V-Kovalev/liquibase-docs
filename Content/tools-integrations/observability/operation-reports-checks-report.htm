Checks Report

The checks report is a operation report that provides data about the execution of  Quality Checks. You can configure checks for your s and database to ensure standardized and consistent behavior with .

In 4.26.0 and later, you can automatically generate a checks report using the  checks run command. The checks report gives you a human-readable summary of command metadata, warning messages, checks package and other configuration details, and an organized summary of check output. For -scoped checks, the summary identifies s and s that tripped checks, their content, and any s. For database-scope checked, the summary contains entries for specific objects that tripped checks and a count of object types checked. You can view summaries sorted by , database, and individual check.

Uses

Using checks reports can help developers, DBAs, DevOps engineers, and managers in the following ways:

  • Easily scan and understand which checks have run and their results with the summary section
  • Analyze your checks by viewing results by , check, and database
  • Collaborate with your team with shareable reports
