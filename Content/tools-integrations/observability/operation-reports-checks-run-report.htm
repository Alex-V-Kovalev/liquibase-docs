Checks Run Report

The checks run report is a operation report that provides data about the execution of  Quality Checks. You can configure checks for your s and database to ensure standardized and consistent behavior with .

In 4.26.0 and later, you can automatically generate a checks run report using the  checks run command. The checks run report gives you a human-readable summary of command metadata, warning messages, checks package and other configuration details, and an organized summary of check output. For -scoped checks, the summary identifies s and s that tripped checks, their content, and any s. For database-scope checked, the summary contains entries for specific objects that tripped checks and a count of object types checked. You can view summaries sorted by , database, and individual check.

Uses

Using checks run reports can help developers, DBAs, DevOps engineers, and managers in the following ways:

  • Easily scan and understand which checks have run and their results with the summary section
  • Analyze your checks by viewing results by , check, and database
  • Collaborate with your team with shareable reports

Enable operation reports

Checks run reports in automation

To automatically use checks run reports in your CI/CD pipeline, follow these steps in your CI/CD tool:

  1. Create a LiquibaseChecksRunReport job that runs the checks run report. This job must:
    1. Enable the checks run report with --reports-enabled (global scope) or --report-enabled (command scope).
    2. Run the  checks run command against your s and/or database.

    For more information, see the Examples section of this page.

Examples

Setting arguments

You can modify the checks run report output with the arguments listed in the tables on this page. In the CLI, global arguments go to the left of the command and command arguments go to the right of the command. For readability, this page shows them on new lines.

liquibase
    --reports-enabled=true
    --reports-path=reports
    --reports-name=checks_run_report.html
  checks run
    --changelog-file="example-changelog.xml"

You can also set arguments in your file, as environment variables, or in a flow file. For a list of arguments and their syntax in each format, see the Global [%=General.Param/Attribute%]s and Command [%=General.Param/Attribute%]s sections of this page.

Disable reports by default; enable only the checks run report

If you want to keep reports disabled by default and enable only the checks run report, you can use the command argument --report-enabled (singular) on the checks run command. For example:

liquibase checks run
    --report-enabled=true
    --report-name=my_checks_run_report.html
Example flow file running a checks run report: 
stages:
  Checks_Run_Report:
    actions:
      - type: liquibase
        command: checks run
        cmdArgs: { changelog-file: "example-changelog.xml",
                        report-format: "HTML"}
        globalArgs: { mirror-console-messages-to-log: "true",
                        reports-enabled: "true",
                        reports-path: "reports",
                        reports-name: "checks-run-report.html"}
