Observability

Observability enables users to share encapsulated information about operations across teams with clear organization so users can make informed decisions that affect their database. Integrating into the automation, observability, analysis, and productivity systems allows us to create a complete tapestry of modern software delivery.

Operation Reports

Operation Reports let you generate reports of operations you perform on your database. These reports contain information about the status of your database and deployments and may help you resolve errors in your automation pipeline. You can generate the following reports:

  • Drift Report when you run the diff and diff-changelog commands
  • Update Report when you run an update family command
  • Rollback Report when you run a rollback family command
  • Checks Run Report when you run the checks run command

Structured Logging makes operation data easily available for monitoring and analysis tools to read, query, and act upon in automated workflows. This feature improves the way logs its actions and events to make records machine-readable, easily-ingested, and queryable by industry-standard observability and analysis tools. not only does the tricky work of database schema versioning and management, it also helps you understand the data around these operations and how they fit into your overall DevOps and CI/CD performance. For a list of endpoints in a structured log, see Structured Logging Keys.

Custom Log Data

You can also introduce your own custom and private data into s. This allows you to collect, inspect, and analyze the information to better understand and improve your SDLC. To learn how to set up a custom log file, see Custom and Private Data in [%=General.StructuredLog%]s.
