changeLogSyncToTagSQL Task

changeLogSyncToTagSQL is a helper task that produces the raw SQL that would run when using the changeLogSyncToTag task to mark all undeployed s associated with the specified tag as executed in your database.

Uses

The changeLogSyncToTagSQL task is typically used when you want to inspect the SQL will use to baseline a new database environment.

It is best practice to inspect any SQL that would run when using the changeLogSyncToTag task so you can review any changes the task would make to your database before running it. The changeLogSyncToTag task marks all s starting with the first at the top of the file and moving down to the up to and including the tag.

If you donâ€™t have any tag specified in your file, add it by using the tagDatabase as follows:

<changeSet id="13.1" author="liquibase">
<tagDatabase tag="version_2.0"/>
</changeSet>

Running the changeLogSyncToTagSQL task

To execute the changeLogSyncToTagSQL task, include the following values in your Ant default file, which can be build.xml:

<project  name="Example"  xmlns:liquibase="antlib:liquibase.integration.ant">
   <taskdef  resource="liquibase/integration/ant/antlib.xml"  uri="antlib:liquibase.integration.ant">
    <classpath  path="path/to/liquibase/libs"/>
   </taskdef>
</project>
<property name="db.changelog.file" value="oracle_lbpro_master_changelog.xml" />
<target name="changeLogSyncToTagSQL" depends="prepare">
   <liquibase:database id="my-database" driver="${driver.classname}"  url="${jdbc.url}" user="${username}" password="${password}"/>
   <liquibase:changeLogSyncToTagSQL databaseref="my-database" changelogfile="com/example/oracle_lbpro_master_changelog.xml"
   labels="mylabels"
   toTag="version_1.1"
   classpathref="classpath"/>
</target>

This example will sync the referenced database with the s found in the file up to and including the tag .

Next, run the following in the CLI to implement the task and update your database:

ant -f build.xml changeLogSyncToTagSQL

It is important to type the task name in the command prompt exactly like you specify it in the target name of your build.xml file. Ant uses the task name to determine which command it will run.
