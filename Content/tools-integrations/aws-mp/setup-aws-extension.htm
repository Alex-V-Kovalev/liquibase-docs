on AWS with AWS Secrets Manager

You can use AWS Secrets Manager within  on AWS. This allows you to confidently utilize your AWS Secrets credentials in a secure way. There are two methods, one is the classpath method and the other is the LPM Method.

Prerequisites

  • First, purchase a license on AWS by following our Use [%=General.Liquibase%] on AWS guide.
  • AWS Secrets Manager Extension

Step-by-step

Follow these steps until you get to step 10. Then you will choose between the classpath method and the LPM method.

  1. Open your Amazon S3 account.
  2. Navigate to the AWS bucket that you want to associate with this extension.
  3. Save these files to your lib folder so can locate the content easily.
  4. Save these files to this bucket so AWS can find them easily:

    Both liquibasesecrets.docker.properties and liquibase-aws-secrets-manager must be in the same S3 bucket.

    1. liquibasesecrets.docker.properties file
    2. liquibase-aws-secrets-manager file

      You can set your credentials up with the AWS Secrets Manager Extension.


  5. Navigate to Amazon Elastic Container Service on Amazon Marketplace.
  6. Open the Task definitions section from the left navigation menu.
  7. Open a task definition. In this example, we will open the -sync command.
  8. Select one of the revisions to open it's details.

    In AWS you create revisions of each task definition which include any change made to the definition. You can open any revision and edit it.

  9. Click on the JSON tab to open the task definition revision's code.
  10. Click the Create new revision with JSON button.

Default classpath method

  1. Replace the liquibase.docker.properties file with the liquibasesecrets.docker.properties file in the code. This provides a path to locate the user's credentials that enable them to use the AWS Secrets Manager.

    "command": [ "--search-path=/common", "--defaults-file=/common/liquibasesecrets.docker.properties", "changelog-sync", "--changelog-file=example-changelog.xml", ]

  2. Add "classpath: /common/liquibase-aws-secrets-manager-1.0.7.jar" at the bottom of the command brackets. This provides a path to locate the user's credentials that enable them to use the AWS Secrets Manager.

    "command": [ "--search-path=/common", "--defaults-file=/common/liquibasesecrets.docker.properties", "changelog-sync", "--changelog-file=example-changelog.xml", "classpath: /common/liquibase-aws-secrets-manager-1.0.7.jar ]

  3. Click the Create button to create the task definition revision.
  4. AWS will bring you to the cluster that contains the task in progress.
  5. Click on the Tasks tab to view the task running.
    The task will display this message when successful
  6. If your task is unsuccessful you can view the task logs to see more details about what did not work as expected.
  7. Find the task logs by navigating to Clusters, then to the cluster specific to the AWS secrets, and then the task that was unsuccessful.
    View the details of the task definition to troubleshoot why the task was unsuccessful.
