Maven tag

Description

tag marks the current database state so you can roll back or deploy changes in the future.

Uses

tag is typically used to mark the current database state by creating a new row in the table. After setting the tag, you can use the update goal with the liquibase.toTag property specified to deploy all changes to this tag or the rollback goal by specifying the liquibase.rollbackTag property to roll back all changes under this tag.

It is best practice to tag your s before running any other commands against your database.

Maven configuration

Maven can be configured multiple ways. One way is to define your configuration properties in your pom.xml file. You can also indicate the location of your file using the following syntax:

<liquibase.propertyFile>path/to/properties_file</liquibase.propertyFile>

Replace path/to/properties_file with the location of your default properties file. See the example below.

<liquibase.propertyFile>${project.basedir}/local.liquibase.properties</liquibase.propertyFile>

Multiple profiles can be created in the pom.xml which contain different properties. See the example below.

Running the tag Maven goal

Running the tag goal requires a Maven project to be implemented.

To run the goal, type the following in your command prompt:

mvn liquibase:tag -Dliquibase.tag=myTag
