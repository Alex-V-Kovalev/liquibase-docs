Maven changelog-sync-sql

changelog-sync-sql is a helper goal that produces the raw SQL would run when using the changelog-sync goal.

Uses

changelog-sync-sql is typically used when you want to inspect the SQL will use to baseline a new database environment and update the table to mark all changes that are not applied to the database as executed.

It is best practice to inspect any SQL that would run when using the changelog-sync goal so you can review any changes the goal would make to your database before running the goal.

Maven configuration

Maven can be configured in multiple ways. One way is to define your configuration properties in your pom.xml file. To configure your pom.xml file, refer to Configuring [%=General.Liquibase%] [%=General.Param/Attr/Uppercased%]s in your Maven POM File. If your pom.xml does not reference a file, include <changeLogFile> specify your file here</changeLogFile> to the pom.xml.

Running the changelog-sync-sql Maven goal

Running the changelog-sync-sql goal requires a Maven project to be implemented.

To run the goal, type the following in your command prompt:

mvn liquibase:changelog-sync-sql

You can also add the liquibase.migrationSqlOutputFile and specify the file to output the migration SQL script to. The default file is migrate.sql. If it exists, it will be overwritten.
