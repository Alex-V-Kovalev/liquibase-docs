Maven deactivate-changelog

The deactivate-changelog goal removes the changelogID from your file so it stops sending reports to .

Uses

The deactivate-changelog goal is typically used when:

  • You start using to keep a history of all your activity, but you have registered the wrong file and want to keep it from reporting the activity to .
  • You use a file that is being secluded or refactored into other s and do not want more than one source of s reporting to .

When you run the deactivate-changelog goal, it modifies a specific file by removing changelogID to prevent it from sending the data. The goal differs from the liquibase.hub.mode=off property, which is set in your defaults file (the file) or passed as a JAVA property and prevents any from sending data.

The liquibase.hub.mode=off property will not report your activity to , regardless of a file, and will not ask to report the activity to when using a non-registered .

Maven configuration

Maven can be configured in multiple ways. One way is to define your configuration properties in your pom.xml file. To configure your pom.xml file, refer to Configuring [%=General.Liquibase%] [%=General.Param/Attr/Uppercased%]s in your Maven POM File.

Running the deactivate-changelog Maven goal

Running the deactivate-changelog goal requires a Maven project to be implemented. To run the deactivate-changelog goal, you must include the file for the deactivation in the pom.xml or the file referenced by the pom.xml. You can specify the pom.xml property as liquibase.changeLogFile in the global properties section or changeLogFile in the plugin configuration section.

Next, type the following in your goal prompt:

mvn liquibase:deactivate-changelog

The file specified in your or pom.xml file will have its changelogID removed.

If the file is in source control, commit the newly deactivated .
