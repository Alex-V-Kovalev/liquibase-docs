Maven db-doc

db-doc generates documentation based on the existing database and s.

Uses

db-doc is typically used to generate database documentation with the change information stored in the s and the existing database. It captures the current state of your database including everything that has been performed against it. The image below shows an example of the database documentation structure with the related directories and files.

Maven configuration

Maven can be configured in multiple ways. One way is to define your configuration properties in your pom.xml file. To configure your pom.xml file, refer to Configuring [%=General.Liquibase%] [%=General.Param/Attr/Uppercased%]s in your Maven POM File.

Running the db-doc Maven goal

Running the db-doc goal requires a Maven project to be implemented.

To run the goal, type the following in your command prompt:

mvn liquibase:db-doc -Dliquibase.outputDirectory=target/docs

By default, the db-doc Maven goal puts the generated documentation in the target/liquibase/db-doc directory. To change it, enter the name of the output directory that you want to use in place of target/doc.
