Maven validate

The validate goal checks and identifies any possible errors in a file that may cause the update goal to fail.

Uses

Use the validate goal to detect if there are any issues with your file before running the update goal. Validation helps you avoid a partial update, where only some s are applied due to an error in the file.

The validate goal is to ensure:

  • The XML, YAML, JSON, or formatted SQL is structured correctly
  • Referenced files can be found
  • Any required or prohibited s are correct to your database
  • There are no duplicated id, author, and file combinations
  • There are no checksum errors

The validate goal only looks for possible errors in the , it does not check for possible errors that might result from applying the changes to a specific database.

Running the validate Maven goal

Running the validate goal requires you to have a Maven project implemented with the parameter specified.

To include the parameter to the pom.xml file, add <changeLogFile>your changelog file</changeLogFile>. Another option is to specify the parameter in the properties file that is reference from pom.xml. Otherwise, specify the at the command prompt.

To run the goal, type the following at your command prompt:

            mvn liquibase:validate
Output

The validate Maven goal produces one of the following outputs:

                    [ERROR] Validation Failed:
[ERROR]      1 changes have validation failures
[ERROR]           column 'type' is required for all columns, com/example/changelog.xml::22::sample
                
                    [INFO] No validation errors found.
