Maven sync-hub

sync-hub synchronizes the local table with .

Uses

sync-hub is typically used when you want to:

  • Ensure that shows the latest results from your table.
  • Synchronize the table of a new project that has pre-existing data in the table.
  • Synchronize your local data if update, rollback, changelog-sync, or drop-all were run while was offline.

When you run the sync-hub goal, you will not see any operations added to the Project associated with your . sync-hub cannot associate previous operations. You will only see s added to the s tab in your project.

Maven configuration

Maven can be configured in multiple ways. One way is to define your configuration properties in your pom.xml file. To configure your pom.xml file, refer to Configuring [%=General.Liquibase%] [%=General.Param/Attr/Uppercased%]s in your Maven POM File.

There is also a list of the properties you can specify in your pom.xml file under <configuration> in the <project></project> level:

<!-- Hub Mode -->
<!--hub.apiKey>HUB API KEY</hub.apiKey>
<hubMode>all|meta|off</hubMode>
<hubConnectionId>TARGET ID</hubConnectionId>
<hubProjectId>PROJECT ID</hubProjectId-->

Running the sync-hub Maven goal

Running the sync-hub goal requires a Maven project to be implemented.

Also, the following items are required:

  • The API Key

    • There are three ways to specify your API Key:

    • Including liquibase.hub.ApiKey value in the file:
      • liquibase.hub.ApiKey: <addYourHubApiKeyHere>
    • Including liquibase.hub.apiKey value in the pom.xml file:
      • <liquibase.hub.apiKey>HUB API KEY</liquibase.hub.apiKey>
    • Passing liquibaseHubApiKey on the command line:
      • --liquibaseHubApiKey=addYourHubApiKeyHere
    • Specifying API key in your JAVA_OPTS Environment Variable as -Dliquibase.hub.apiKey.
      • On Linux or MacOS: JAVA_OPTS="-Dliquibase.hub.apiKey=<myapikey>"
      • On Windows: set JAVA_OPTS="-Dliquibase.hub.apiKey=<myapikey>"

      • To retrieve or manage your API keys, go to Settings > API Keys by selecting the settings icon on the left side of the page.

  • The file

    • Add your file to the pom.xml or file or specify it as the -Dliquibase.changelog-file while running the goal.

  • The hubConnectionId :

    • If you enter the -Dliquibase.changelog-file , there is no need to specify hubConnectionId with the goal.

    • If you use hubConnectionId, add it to the pom.xml file, the file, or to the sync-hub goal on the command line. You can find the hubConnectionId in Project > View Details for the required project > Targets > View Status > Target ID.

    • If you specify both the hubConnectionId and the file, the file will be ignored and the hubConnectionId will be used.

<!--liquibase.hubConnectionId>TARGET ID</liquibase.hubConnectionId>
<hubConnectionId: <addYourHubTargetId>
liquibase sync-hub --hubConnectionId=myTargetID

Additionally, you can set the level of data, which you want to send to , in your JAVA_OPTS as -Dliquibase.hub.apiKey or in the file:

liquibase.hub.mode:[all|meta|off]
  • all sends all data, including the body, generated SQL, and logs (the default value is all).
  • meta does not send the body, generated SQL, or operation-event and change-event logs. This mode sends the name, author, id, timestamps, version, and others.
  • off sends no data to .

Before running the sync-hub goal, register your file by using the register-changelog goal:

mvn liquibase:register-changelog

The register-changelog goal connects your local activity to a specific Project.

To run the goal, type the following in your command line:

mvn liquibase:sync-hub

sync-hub optional Maven configuration properties

Property Definition
hubConnectionId

Specifies the target (also called connection) needed for the sync-hub goal to synchronize the local table with .
