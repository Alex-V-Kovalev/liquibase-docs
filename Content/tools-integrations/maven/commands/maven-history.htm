Maven history

history is a helper goal that lists out all your deploymentIds and all s associated with each deploymentId.

Uses

history is typically used when you want to inspect a specific group of changes to ensure that they have been applied to the database. The history goal outputs the history of your deployments in the database.

It is best practice to run the history goal before performing rollbacks to avoid any risks of reverting changes that are required to be in the database.

Maven configuration

Maven can be configured in multiple ways. One way is to define your configuration properties in your pom.xml file. To configure your pom.xml file, refer to Configuring [%=General.Liquibase%] [%=General.Param/Attr/Uppercased%]s in your Maven POM File.

Running the history Maven goal

Running the history goal requires a Maven project to be implemented.

To run the goal, type the following in your command prompt:

            mvn liquibase:history
