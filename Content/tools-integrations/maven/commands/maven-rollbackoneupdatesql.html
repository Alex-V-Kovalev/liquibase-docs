Maven rollbackOneUpdateSQL

rollbackOneUpdateSQL is a helper goal that allows you to inspect the SQL will run to revert all s associated with the deploymentID specified when using the rollbackOneUpdate goal.

Uses

rollbackOneUpdateSQL is typically used when you want to inspect the raw SQL that uses to revert all s associated with a specified deploymentId when you run the rollbackOneUpdate command, so you don't unintentionally make a mistake.

Maven configuration

Maven can be configured in multiple ways. One way is to define your configuration properties in your pom.xml file. To configure your pom.xml file, refer to Configuring s in your Maven POM File. If pom.xml does not reference a properties file, include <ProLicenseKey>specify your license key here</ProLicenseKey> to pom.xml.

Running the rollbackOneUpdateSQL Maven goal

Before running the rollbackOneUpdateSQL goal, you can get the deploymentId of the deployment you want to revert by running the history goal:

                        mvn liquibase:history

If you don’t specify the deploymentId, by default, will look up the most recent deploymentId from the table and use it for the rollback.

Next, run the rollbackOneUpdateSQL goal, with your information:

                        mvn liquibase:rollbackOneUpdateSQL -Dliquibase.deploymentId=6408658154

Enter the name of the deploymentId that you want to use in place of 6408658154. For more information, run mvn liquibase:help -Ddetail=true -Dgoal=rollbackOneUpdateSQL.
