Maven update

Description

update deploys any changes that are in the file and that have not been deployed to your database yet.

Uses

update is typically used to apply database changes that are specified in the file to your database.

When you run the update goal, sequentially reads s in the file, then it compares the unique identifiers of id, author, and path to filename to the values stored in the DATABASECHANGELOG table.

  • If the unique identifiers do not exist, will apply the to the database.
  • If the unique identifiers exist, the MD5Sum of the is compared to the one in the database.
  • If they are different, will produce an error message that someone has changed it unexpectedly. However, if the status of the runOnChange or runAlways attribute is set to TRUE, will re-apply the .

Maven configuration

Maven can be configured multiple ways. One way is to define your configuration properties in your pom.xml file. You can also indicate the location of your liquibase.properties file using the following syntax:

<liquibase.propertyFile>path/to/properties_file</liquibase.propertyFile>

Replace path/to/properties_file with the location of your default properties file. See the example below.

<liquibase.propertyFile>${project.basedir}/local.liquibase.properties</liquibase.propertyFile>

Multiple profiles can be created in the pom.xml which contain different properties. See the example below.

Running the update Maven goal

Running the update goal requires a Maven project to be implemented.

To run the goal, type the following in your command prompt:

mvn liquibase:update
