MongoDB Pro Extension User Roles

MongoDB is available in version 4.20.1

You can access data and commands through role-based authorization in MongoDB. This guide covers two easy methods to create a user that has the correct permissions required to run alongside the MongoDB Pro extension. The methods include using built-in roles to Mongo or using the user-defined role method. The built-in role method allows you to provide different levels of access commonly needed in a database system. The user-defined role method allows for custom role creation, useful when and it lends itself to a scenario in which the built-in roles cannot describe the privileges necessary for the job.

readWrite roles and collMod privileges are the minimum required user roles to run liquibase. You can always give more privileges to the user used to run .

Built-in rRoles mMethod

This method allows you to create a user that has readWrite and dbAdmin roles. This user canwill be able to run the Liquibase Pro MongoDB extension directly.

Without dbAdmin role permissions, you cannotwill not be able to run the MongoDB Pro extension directly.

Create a user with the built-in readWrite and dbAdmin roles by adding the following code with your own user, password, and database content to the file:

db.createUser(
	{
		user: "lbuser",
		pwd: "password",
		roles: [
			{
			 role: "readWrite",
			 db: "yourdb"
			},
			{
			 role: "dbAdmin",
			 db: "yourdb"
			}
		]
	}
);
