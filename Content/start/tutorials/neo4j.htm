Using with Neo4j

Neo4j is a property graph database management system with native graph storage and processing. It uses a graph query language called Cypher. For more information, see Neo4j Documentation.

Supported versions

  • 3.5+

Prerequisites

To access Neo4j, do one of the following:

  • Download Neo4j Desktop locally
  • Configure Neo4j locally with Docker
  • Create a Neo4j AuraDB cloud instance
  • Launch a Neo4j Aura sandbox instance

Install drivers

To use and Neo4j, you need two JAR files:

  • JDBC driver (GitHub link)
  • Liquibase extension for Neo4j (GitHub link)
<dependency>
    <groupId>org.neo4j</groupId>
    <artifactId>neo4j-jdbc-driver</artifactId>
    <version>4.0.9</version>
</dependency>
<dependency>
    <groupId>org.liquibase.ext</groupId>
    <artifactId>liquibase-neo4j</artifactId>
    <version></version>
</dependency>

You can either use the JDBC driver for Bolt or the driver for Bolt and HTTP. The previous example shows the driver for both protocols. If you only want to use the Bolt protocol, use the artifact neo4j-jdbc-bolt instead of neo4j-jdbc-driver. See Neo4j JDBC Driver Documentation for more information.

The extension for Neo4j does not currently support connections through HTTP.

For additional JARs to integrate Neo4j with your preferred programming language, see Connecting to Neo4j.
