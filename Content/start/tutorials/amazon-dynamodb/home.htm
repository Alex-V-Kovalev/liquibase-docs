Amazon DynamoDB Extension Overview

Amazon DynamoDB is a fully managed NoSQL database service. You can use Amazon DynamoDB to create a database table that can store and retrieve any amount of data, and serve any level of request traffic. Amazon DynamoDB automatically spreads the data and traffic for the table over a sufficient number of servers to handle the request capacity specified by the customer and the amount of data stored. For more information, see the Amazon DynamoDB documentation page.

You can use Amazon DynamoDB with  s written in XML, YAML, and JSON. However, because Amazon DynamoDB does not use SQL, you cannot use SQL s or commands that generate SQL output. Instead, you can specify arbitrary PartiQL (a SQL-compatible query language) using the partiql and partiqlFile s, similar to the sql and sqlFile s. For a list of supported features like s, see the Features section.

Features

The Amazon DynamoDB extension supports all features and many features, including:

  • Targeted updates and targeted rollbacks (per and per update)
  • [%=General.Liquibase%] Flow Files
  • Secrets Management
  • Using resources stored on AWS S3

Supported s

Normal  s are not supported in Amazon DynamoDB. You must use the following s in your :

  • Tables:
    • createDynamoTable
    • deleteDynamoTable
    • updateDynamoTable
    • updateDynamoTableProvisionedThroughput
  • Indexes:
    • createGlobalSecondaryIndex
    • deleteGlobalSecondaryIndex
    • updateGlobalSecondaryIndex
  • PartiQL:
    • partiql
    • partiqlFile

For more information, see [%=General.Liquibase%] [%=General.changetypes%]s for Amazon DynamoDB.

Supported parameters

You can use all normal [%=General.Liquibase%] Parameters with Amazon DynamoDB.

To configure DynamoDB-specific behavior, such as the billing mode for the Tracking Tables and the behavior of the DynamoDB waiter, you can set these parameters in the CLI, in flow files, in your file, or as environment variables:

  • Tracking tables:
    • --dynamodb-tracking-tables-billing-mode
    • --dynamodb-tracking-tables-provisioned-throughput-read-capacity-units
    • --dynamodb-tracking-tables-provisioned-throughput-write-capacity-units
  • Waiters:
    • For creating objects:
      • --dynamodb-waiter-create-fixed-delay-backoff-strategy-duration
      • --dynamodb-waiter-create-max-attempts
      • --dynamodb-waiter-create-total-timeout
    • For deleting objects:
      • --dynamodb-waiter-delete-fixed-delay-backoff-strategy-duration
      • --dynamodb-waiter-delete-max-attempts
      • --dynamodb-waiter-delete-total-timeout
    • For updating objects:
      • --dynamodb-waiter-update-fixed-delay-backoff-strategy-duration
      • --dynamodb-waiter-update-max-attempts
      • --dynamodb-waiter-update-total-timeout
    • For configuration:
      • --dynamodb-waiter-fail-on-timeout
      • --dynamodb-waiters-enabled

For more information, see [%=General.Liquibase%] Parameters for Amazon DynamoDB.

Supported commands

Amazon DynamoDB supports the following commands:

  • All update and rollback commands, except for those that generate SQL output
  • All change tracking and utility commands, except for execute-sql and those that generate SQL output
  • The flow and flow validate commands
  • All other commands, unless otherwise specified in the Limitations section.

For more information, see About [%=General.Liquibase%] Commands.

Limitations

  • The Amazon DynamoDB extension does not support any normal  s. You must use the s specified in the Supported [%=General.changetypes%]s section.
  • The Amazon DynamoDB extension does not support the following commands:
    • Any command that generates SQL output, such as update-sql and rollback-sql
    • Database inspection commands: diff, diff-changelog, generate-changelog, snapshot, snapshot-reference
    • The execute-sql command
  • The partiql  does not support PartiQL JSON
