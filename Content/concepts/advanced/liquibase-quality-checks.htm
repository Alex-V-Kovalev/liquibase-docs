Quality Checks

quality checks allow you to run checks against s and SQL scripts by flagging conditions and configurations you need. Using the quality checks functionality, you can:

  • Better secure your database by getting immediate notice that user and role privileges are being modified.
  • Prevent data loss and corruption.
  • Enforce internal standards and best practices to ensure that only high-quality, compliant changes are being deployed to your databases.
  • Quickly, consistently, and easily validate your changes from the local development environments.
  • Integrate the check capability into automated workflows to provide quality gates for database changes in automation.

The initial capabilities are available for the following platforms: Oracle, SQL Server, PostgreSQL, MySQL, and MariaDB.

quality checks include static and dynamic checks, the checks command with sub-commands, and the checks settings file that includes defined configurations and parameters. All checks work in the command line, which is interactive and allows the configuration of dynamic checks.

If you are a Community user, the execution is limited to 5 checks.
