liquibase.sql.showAtLogLevel Attribute

The liquibase.sql.showAtLogLevel attribute controls logging of SQL statements and SQL execution output.

logs the SQL it executes in a database to the FINE log level by default, however, you can configure the level where logs an executed SQL with the liquibase.sql.showAtLogLevel attribute.

Using the liquibase.sql.showAtLogLevel attribute

executes SQL statements against a database during operations. The liquibase.sql.showAtLogLevel attribute defines whether the SQL statements executes will result in the output to a log.

The permitted values for liquibase.sql.showAtLogLevel are:

  • FINE
  • INFO
  • WARNING
  • SEVERE
  • OFF
liquibase.sql.showAtLogLevel = [FINE | INFO | WARNING | SEVERE | OFF]

The default value for liquibase.sql.showAtLogLevel is FINE.

The liquibase.sql.showAtLogLevel attribute works with the CLI logLevel attribute as follows:

  • When logLevel set in either file or on the command line is equal to or above liquibase.sql.showAtLogLevel set in the file, executed SQL output is logged
  • When logLevel set in either file or on the command line is below liquibase.sql.showAtLogLevel set in the file, no executed SQL output is generated

To log the executed SQL with the minimum output information, specify logLevel=INFO in either the file or on the command line and liquibase.sql.showAtLogLevel=INFO in the file.

To completely disable the output of the executed SQL, specify liquibase.sql.showAtLogLevel=OFF in the file.

To use the log file to write the executed SQL for the CLI, specify the logFile attribute in either the file or on the command line.

Known issues

There are several limitations when using the liquibase.sql.showAtLogLevel attribute:

  • The liquibase.sql.showAtLogLevel attribute cannot be passed as an attribute to a command. Specify it in the liquibase.properties file.
  • The liquibase.sql.showAtLogLevel attribute is not supported in the Maven integration.
  • SQL executed using the SQL Plus feature will not result in an output.
  • SQL executed by extensions might not give an output if your code is not aligned with the standard JDBC execution classes.
