modifyChangeSets

The modifyChangeSets tag lets you include SQL scripts in your   while using a native executor like PSQL, SQL Plus, or SQLCMD. You can use it to specify an overarching executor for all files you're including.

Uses

If you have many s that need to be run with a native executor, and you organize your s in a "root" and "nested" format using the include and includeAll tags, you can use the runWith  to specify an executor that applies to everything in modifyChangeSets. Since your nested s inherit the executor specified in modifyChangeSets, you don't have to specify it on each individual you include.

If you want to make an exception for a particular you're including, you can specify the default JDBC executor for that . All other s still run with the executor inherited from modifyChangeSets.

You can still include additional files in your root outside of the modifyChangeSets tag. Such files run with the JDBC executor.

Syntax

runWith values

Executor Syntax Notes
JDBC jdbc Default value if none specified
PSQL psql Executor for PostgreSQL
SQL Plus sqlplus Executor for MSSQL Server
SQLCMD sqlcmd Executor for Oracle
