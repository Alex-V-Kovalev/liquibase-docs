Configuration Reference

4.4 and later versions can process configuration values from multiple sources.

The following table provides naming conventions of the parameters for the file, environment variables, and CLI.

Check the related topics for more details when configuring :

  • Creating and configuring a [%=General.liquiPropFile%] file
  • Concepts
  • Command Line Interface
  • [%=General.Liquibase%] Environment Variables
  • JAVA_OPTS Environment Variable
  • Commands

Additionally, you can run liquibase --help to see the specific information.

Global settings

alwaysOverrideStoredLogicSchema   Global

Properties string

liquibase.alwaysOverrideStoredLogicSchema

Environment variable

LIQUIBASE_ALWAYS_OVERRIDE_STORED_LOGIC_SCHEMA

CLI

--always-override-stored-logic-schema

Description

When generating SQL for createProcedure, the parameter defines whether the procedure schema should be forced to the default schema if no schemaName is set.
