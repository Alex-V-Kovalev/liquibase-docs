dbclhistory-capture-extensions

--dbclhistory-capture-extensions is a global Boolean parameter that specifies whether captures information about installed extensions in the [%=General.databasechangeloghistory%] table. It is available in 4.26.0 and later. The default value is true.

Uses

comes pre-installed with some common database drivers to make it easier for you to deploy changes quickly. You can also install more extensions to add more database support, such as the liquibase-mongodb extension. By default, records the filepath and name of all extensions you have installed in the (DBCLH) table.

However, if you want to exclude extension information from your DBCLH, you can set --dbclhistory-capture-extensions to false.
