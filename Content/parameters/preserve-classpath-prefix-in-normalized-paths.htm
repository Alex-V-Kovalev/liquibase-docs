preserve-classpath-prefix-in-normalized-paths

The --preserve-classpath-prefix-in-normalized-paths global parameter is a Boolean value that preserves the prefix classpath: in your resource path when resolving sub-resource locations, allowing you to resolve hierarchical resources under a classpath-based root. It is available in 4.30.0 and later.

The default value of --preserve-classpath-prefix-in-normalized-paths is false.

Uses

uses the Java Virtual Machine (JVM) to resolve a resource path like your classpath, which is a list of file paths where s, configuration files, and JAR files (bundles of Java classes) are stored. The JVM uses the classpath to search for, load, and link each of those JARs to prepare them for use in applications.

However, by default, values in --search-path are also normalized before being sent to the ResourceAccessor, so the classpath: is removed.
