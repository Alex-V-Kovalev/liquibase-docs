custom-log-data-file

The custom-log-data-file parameter allows you to call on a specific file to introduce to a . Learn how to include or exclude your custom data here: Introduce custom data and exclude private data in Structured Logs

Uses

If you would like to include or exclude custom data from structured logs you can introduce that specific data with the custom-log-data-file parameter. This allows you to collect, inspect, and analyze the information to better understand and improve your SDLC. This custom data can be global or command-specific.

If liquibase.customlogdata.yaml is in the working directory, Liquibase uses it by default, and it is not necessary to call the file out via the --custom-log-data-file argument or property.
