trim-load-data-file-header

--trim-load-data-file-header is a global Boolean parameter. You can use it to trim whitespace from CSV headers that you call with the loadData . It is available in 4.29.0 and later. The default value is false.

Uses

loadData accepts input from a Comma-Separated Value (CSV) file, which contains a header (a comma-separated list of column names) and any number of data rows. In 4.4.0 and later, in order to conform to RFC 4180, spaces are considered part of the header name and are no longer stripped by default.

For example, if you have the CSV header id, first, last (with spaces after each comma), parses the column names as id, first, and last (with leading spaces). However, you may not have intended to include spaces in individual column names. If you try to call on the column first (without a leading space) when you specify s for loadData, displays this error:

ERROR: Exception Primary Reason:  ORA-00904: " first": invalid identifier

You can resolve this manually by removing spaces from your CSV header.

Alternatively, you may prefer to keep spaces in your CSV header for aesthetic reasons or don't want to manually update them. To avoid this error, set --trim-load-data-file-header to true. This way, trims the file header to id,first,last so that you can call on the intuitive column names.
