dbclhistory-severity

--dbclhistory-severity is a global parameter that lets you specify the exit code for to use if an exception is encountered while recording events to the [%=General.databasechangeloghistory%] table. Valid values are INFO (0), MINOR (1), MAJOR (2), CRITICAL (3), and BLOCKER (4), from least severe to most severe. It is available in 4.26.0 and later. The default value is 1.
