onMissingIncludeChangelog

An included refers to a file that is tagged with the include tag within a separate . When an included cannot be found by Liquibase, it is considered missing. The onMissingIncludeChangelog is a parameter (or command argument) that controls how behaves when a changelog specified in an include is missing.

If it is set to warn and the Include is not present, a warning message will appear. If it is set to fail and the Include is not present, an error is thrown. The default configuration is fail. We recommend setting it to warn.

Uses

If your Include was removed by accident Liquibase will notify you right away so you know the exact cause of the error. With onMissingIncludeChangelog enabled, you will be able to quickly get to the bottom of why the file is missing and how to restore it.
