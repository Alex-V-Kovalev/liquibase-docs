dynamodb-waiter-log-notification-interval

--dynamodb-waiter-log-notification-interval is a global Boolean parameter for the DynamoDB extension. It specifies the interval in milliseconds for the DynamoDbWaiter to wait between log notifications while it polls resources in your database. The default value is 3000.

Uses

The --dynamodb-waiter-log-notification-interval parameter controls the interval between DynamoDB waiter log notifications while the waiter polls pending resources:

  • If you want more frequent log notifications, set a lower value (shorter interval). This may be useful if you want very specific data about when the waiter polls resources in your database.
  • If you want less frequent log notifications, set a higher value (longer interval). This may be useful if you want to keep your logs cleaner.
