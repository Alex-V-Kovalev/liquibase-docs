Principal AWS extension

The Principal AWS extension encompasses the three AWS S3, AWS Secrets Manager, and Amazon DynamoDB Pro extensions into one. This single, convenient source provides a unified user experience and prevents conflicts between the three tools.

Requirements

  •  4.31.0+
  • license. See How to Apply Your [%=General.LBPro%] License Key
  • AWS Secrets Manager credentials

If you use existing stand-alone extensions in your library or classpath, you do not have to do anything different to continue using the extensions individually. You will receive a console and log message informing you that the individual extensions are deprecated and how to suppress the warning message.

Installation

CLI

  1. Navigate to the liquibase-hashicorp-vaultliquibase-aws-secrets-manager extension on Maven Central
  2. Download liquibase-hashicorp-vaultaws-secrets-manager-<version>.jar and move it to the liquibase/lib directory

Usage

Each element of this extension manages different aspects of AWS content. Find out each of their Usage sections from this list.

  • AWS S3 - Store and retrieve data from remote file locations in a secure, centralized way.
  • AWS Secrets Manager - Store sensitive authentication details and application secrets in .
  • Amazon DynamoDB Pro - Manage DynamoDB objects and data via PartiQL statements.
