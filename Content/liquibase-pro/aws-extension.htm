AWS extension

The AWS extension encompasses the three AWS S3, AWS Secrets Manager, and Amazon DynamoDB Pro extensions into one. This single, convenient source provides a unified user experience and prevents conflicts between the three tools.

Requirements

  •  4.31.0+
  • license. See How to Apply Your [%=General.LBPro%] License Key
  • AWS Secrets Manager credentials

If you currently use any of the individual AWS S3, AWS Secrets Manager, and Amazon DynamoDB Pro extensions we recommend removing them prior to using the AWS extension to prevent possible conflicts.

If you use existing stand-alone extensions in your library or class path, you do not have to do anything different to continue using the extensions individually. delivers a console and log message informing you that the individual extensions are deprecated and how to suppress the warning message.

Installation

CLI

  1. Navigate to the liquibase-aws extension on Maven Central
  2. Download liquibase-aws-<version>.jar and move it to the liquibase/lib directory

Maven

If you use Maven, add the AWS extension dependency in your pom.xml file to download the JAR:

<dependency>
    <groupId>org.liquibase.ext.vaults</groupId>
    <artifactId>liquibase-hashicorp-vault</artifactId>
    <version>1.0.0</version>
</dependency>
<plugin>
     <groupId>org.liquibase</groupId>
	<artifactId>liquibase-maven-plugin</artifactId>
			<version>4.27.0</version>
	<dependencies>
		<dependency>
			<groupId>org.liquibase.ext.vaults</groupId>
			<artifactId>liquibase-aws-secrets-manager</artifactId>
			<version>1.0.5</version>
		</dependency>
	</dependencies>
</plugin>

Package Manager

Alternatively, you can install the HashiCorp VaultAWS Secrets Manager extension with lpm (Liquibase Package Manager).

lpm update
lpm add liquibase-hashicorp-vault
lpm update
lpm add liquibase-aws-secrets-manager

Setup

Configure your AWS credentials in any standard method. aws_access_key_id, aws_secret_access_key and aws_region are required.

Usage

Each element of this extension manages different aspects of AWS content. Find out each of their Usage sections from this list.

  • AWS S3 - Store and retrieve data from remote file locations in a secure, centralized way.
  • AWS Secrets Manager - Store sensitive authentication details and application secrets in .
  • Amazon DynamoDB Pro - Manage DynamoDB objects and data via PartiQL statements.
