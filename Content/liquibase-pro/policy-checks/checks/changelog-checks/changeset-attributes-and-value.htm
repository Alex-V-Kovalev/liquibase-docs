ChangesetAttributesAndValue

This check triggers when specified attributes do not match the user-supplied string or regex pattern.

About ChangesetAttributesAndValue
version required 4.29.0+
Scope (--checks-scope) changelog
Default status disabled
Default severity (exit code) 0 ("INFO")
Customizable settings Yes (dynamic)

Uses

This policy check allows you to easily check whether the attributes in your s each match a particular string or regular expression. This helps you ensure that your s remain standardized.

For example, if you want to ensure that the value of the attribute id always follows the format <integer>-<description> (such as 1-newTable), you could specify the regular expression \d+-\w+.

When customizing this check, simply specify the name of the attribute you want to standardize (ATTRIBUTE) and the string or regular expression (SEARCH_STRING). If you want to apply this check to multiple attributes, run the copy command and specify the name of a different attribute.

If you want to specify multiple attributes in a single check, you can specify them as a comma-separated list. However, you can only specify one value to apply to all the attributes you specify in a single check.

If detects that an attribute does not match the user-supplied string or regex, displays the following message:

The attribute '<attributename>' does not match the required pattern '<PATTERN>.'

Troubleshooting

If you use a property for an attribute value in a (such as runwith=${SOMEVAR}) and have the checks run command parameter --property-substitution-enabled=false, skips the check and displays the following:

WARNING: This check skipped due to unexpanded property in '<attributename>' value. Set '--property-substitution-enabled=true' to properly evaluate this changeset.
