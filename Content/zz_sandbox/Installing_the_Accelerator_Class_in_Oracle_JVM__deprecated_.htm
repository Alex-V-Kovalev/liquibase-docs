Installing the Accelerator Class in Oracle JVM (deprecated)

You needed to remove and reinstall the Oracle JVM for all databases where you wanted to use the Datical Accelerator with versions 7.15 and lower. Typically this was on the database where you run Convert SQL scripts.
For more detailed information, see the following resources:

  • http://stackoverflow.com/questions/11788194/how-to-install-jvm-in-oracle
  • Knowledge Mgmt Note ID 757771.1 How to Reload the JVM in 10.1.0.X and 10.2.0.X in Oracle Support (formerly metalink ). More detailed instructions and video are available there. This note also consists of several other Notes that resolve different issues.

Deprecation of the Oracle Accelerator JDBC Driver

Because of a change in deployment trends and continued improvement of the performance of Datical DB, the Oracle Accelerator JDBC Driver (compression driver) was removed from our driver repository in versions 7.16 and higher.  It was previously available only with versions 7.15 and lower.

Testing that Oracle JVM is Installed with versions 7.15 or lower

You would run the following statement:
select * from dba_registry where COMP_ID='JAVAVM';
If there were no results, then the Oracle JVM is not installed. Go to the installation step.

Removing the Existing Oracle JVM from versions 7.15 or lower


  1. As sys user, run the following:

startup mount;
alter system set "_system_trig_enabled" = false scope=memory;
alter system enable restricted session;
alter database open;
@?/rdbms/admin/catnojav.sql
@?/xdk/admin/rmxml.sql
@?/javavm/install/rmjvm.sql
truncate table java$jvm$status;

  1. Restart the database.

Installing OracleJVM with versions 7.15 or lower


  1. As sys user, run the following:

startup mount
alter system set "_system_trig_enabled" = false scope=memory;
alter database open;
@?/javavm/install/initjvm.sql
@?/xdk/admin/initxml.sql
@?/xdk/admin/xmlja.sql
@?/rdbms/admin/catjava.sql
shutdown immediate;

  1. Restart the database.
  2. Execute the following script and resolve any INVALID objects.

@?/rdbms/admin/utlrp.sql
The JVM would be installed and functional. You could check it using the following statement:
SQL> select comp_name, version, status from dba_registry

Verifying Oracle Permissions for versions 7.15 or lower

The Accelerator loaded a Java class into an Oracle database schema and executed it as part of its operation.
The required permissions were CREATE and EXECUTE permissions for PROCEDURES in order to load and execute the Java code.
