Required Preparation for Installing the GUI Client in v2019.x or v6.x

New Requirements for Installation

The new GUI client requires a new installation. It has been updated to use a newer version of the Eclipse framework and newer plugins.  There is no automated method to update or to revert.  This is necessary if you are currently running Datical 4.x/5.x versions and want to now install the Datical 2019.x/6.x versions.  (For example, when going from Datical version 5.9 to Datical version 6.12.)

The additional preparation steps required are as follows: 

  • Copy and save the existing .metadata directory from the Datical DB workspace (datical directory).
  • Install the new GUI in a different installation directory. 
  • Install database drivers for the new installation. 

This preparation is required for the following reasons:

  • Ability to revert to an older version.


Important

You cannot run the older client (Datical 4.x/5.x) and the newer client (Datical 2019.x/6.x) on the same workspace concurrently. 

Installation Procedure

This variant of the installation procedure uses the existing installation of the Datical DB GUI client. 

See Start Here: Datical DB Installation Overview for more information about installation use cases. 


Note

A procedure specifically for versions 2019.2.2 and 2019.2.3 that includes an extra step needed for those versions is available in the Knowledge Base here: How To: Install Datical DB 2019.2.2 or 2019.2.3 GUI.  


Workspace Location 

The installer creates the following workspace directories for project information.    

  • Windows: C:\Users\<username>\datical

    Windows Note

    Windows treats the .metadata directory as a hidden file. If you do not see it, change your settings in File Explorer:

    1. Open File Explorer
    2. Click the View tab
    3. Select (check) the Hidden Items checkbox

  • Linux: /<user-home>/datical

1. Copy and Save the Existing .metadata Directory

The workspace directory (datical directory) for your Datical DB projects contains a .metadata directory

  1. Close all running Datical DB GUI clients that use that workspace. 
  2. Copy the version 4.x/5.x  .metadata directory from your Datical DB workspace to a location outside of the workspace. 
  3. Keep the location secure and do not make changes to files in directory that has the backup copy of your 4.x/5.x .metadata. 

2. Uninstall the Existing Datical DB

For Windows, choose Start > Datical > Datical DB Uninstaller. 

Uninstalling leaves the workspace in place. Do not remove or change it. 

See Uninstalling Datical DB Components.

Note

If you do not uninstall the existing Datical DB, do not to use it again after the new Datical DB client is installed.

3. Install the New Client in a Different Installation Directory

When you run the installer, you are prompted to provide an installation directory. 

When installing the GUI client for v2019.2 or later, provide a different name than you used previously. We recommend appending the release number.

Example:

  • Old: DaticalDB
  • New: DaticalDB_2019.2  or   DaticalDB_6.12

Enter the installation directory you want when the installer prompts you. 

The installer uses the following installation directories by default. 

  • Windows: C:\Users\<username>\DaticalDB
  • Linux: /<user-home>/DaticalDB

4. Install Database Drivers for the New Installation

  1. Start the new Datical DB GUI
  2. Select Help > Install New Software
  3. In the Available Software window, set the repository to use. You can choose one of the following:
    • Datical Update Site - update.datical.com/latest/thirdparty
    • Datical Composite Repository zip downloaded from software.datical.com
  4. Select Datical DB Drivers. 
  5. Click Next to continue the installation and follow the prompts to the end. 


Troubleshooting

If this procedure fails to preserve your existing projects, re-import them. 

Reverting to an Installation Before 2019.2

If you find you need to stop using the new GUI client (2019.x/6.x) and go back to using the old one (4.x/5.x), do the following. 

  1. Close any running instances of the newer 2019.x/6.x GUI client. 
  2. Replace the newer 2019.x/6.x .metadata directory in the workspace with the older backup copy of the 4.x/5.x.metadata directory you saved and put aside before. 
  3. Start the old 4.x/5.x GUI client (reinstall it if you removed it). 
  4. Remove the new 2019.x/6.x GUI client. See Uninstalling Datical DB Components.
