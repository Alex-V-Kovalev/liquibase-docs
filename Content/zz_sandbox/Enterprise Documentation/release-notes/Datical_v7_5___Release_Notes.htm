Customer Notices for Upgrades

Starting with the 6.10 release, Datical DB will run in the OpenJDK runtime instead of the Oracle Java runtime.

  • If you are running Datical DB version 6.9 or lower, please use the full install process for 7.5 (NOT an upgrade/update process).
  • If you are running Datical DB version 6.10 or higher, you can use whichever process you prefer for 7.5 (full install or upgrade/update).

Datical Deployment Monitoring Console (DMC)

With the 7.0 release Datical included a new, lightweight web-based Deployment Monitoring Console (DMC) that provides status and reporting information for all of your Datical projects through a browser.  The new DMC is a simpler replacement for Datical Service that is much easier to install, manage and use.

Deployment Monitoring Console Update Instructions

To upgrade from version 6.17 or earlier of Datical Service, consult the DMC Upgrade Guide

The process to update the Deployment Monitoring Console from version 7.0 or later to the latest version is very similar to the installation process. Consult the instructions below.

Expand for DMC update instructions
  1. Navigate tosoftware.datical.com, download and copy thedatical-service-unix-7_5_187-master.shfile to your existing DMC server

  2. Run the installer with the following command and enter your existing OS user as the admin user and admin group:

    /home/datical> sudo bash /datical-service-unix-7_5_187-master.sh

  3. When prompted for an install location, enter the directory of the previous DMC installation

  4. The installer will update your installation and restart effected DMC services.  You can begin using the new version of the DMC immediately.


New Features

Deployment Packager Reports and Logs Available in the DMC

The list of operations and associated reports in the Deployment Monitoring Console now includes Deploy Packager operations.  The following is available for each Deploy Packager Operation:

  • Deploy Packager Report
  • packager.log
  • Forecast and Deploy reports for all operations that were executed during Packaging

Enhancements to the runRules command

New parameter option: Lists of Files

The runRules command has been extended to take a comma or semicolon separated list of files in addition to the existing options of individual files and directories. 

runRules - Execute the projects 'sqlRules' against an individual SQL script or a folder of scripts.
USAGE:
  runRules [SQL file | List of SQL files | SQL files folder]
EXAMPLE:
  runRules git/sql/ddl/create_user_table.sql
  OR
  runRules git/sql/ddl/create_user_table.sql,git/sql/ddl/add_user_index.sql,git/sql/ddl/create_co_table.sql
  OR
  runRules git/sql/ddl
NOTES:
  This command requires that the current directory is a valid DaticalDB project directory.

For more information, consult the product documentation for runRules

runRules executed during Deploy Packager in preview mode

runRules will also be executed when the Deploy Packager is run in preview mode. Previously it was only run during Deploy Packager executions that were not in preview mode.
