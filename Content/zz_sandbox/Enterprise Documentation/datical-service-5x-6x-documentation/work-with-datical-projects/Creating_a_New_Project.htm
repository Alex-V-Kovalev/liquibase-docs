Creating a New Project

To create a new project, log in as a user with Project Creator role. 

What You Need First

You need the following information during project creation:

  • SCM Information
    • Repository URL
    • Branch Name
    • Login to use (username/password or SSH key). Create the SCM user before creating the project. 
  • Managed Databases - each step requires a connection to a database. Requirements vary by database platform (Oracle, SQL Server, EDB Postgres, DB2)
    • Host
    • Port
    • Database name (if applicable)
  • Users
    • At least one user must exist in Datical to be assigned the Project Administrator role. 
  • Unique Key scheme - Projects, Pipelines, Steps, and Connections have optional but recommended fields for a unique key. Objects of each type must be able to be identified uniquely. Duplicate keys are not allowed within a type (project, pipeline, step, connection).  


1. Create the Project

  1. Select Projects > New Project.
  2. Project/Schema Settings tab
    • Name
    • Plan Type (default Single-Schema)
    • Project Key - Unique identifier for the project. Optional but recommended when creating a project. Use only lowercase letters, numbers, '_' and '-'.  It is read-only once the project is created. 
    • Schema Naming (default Deployment Step)
  3. Click Save
  4. You go to Project Settings > Pipelines. The Pipelines list is empty. 

2. Create a Pipeline

In Project Settings > Pipelines:

  1. Click Create Pipeline
  2. Enter information for the pipeline:
    • Pipeline Name (required)
    • Pipeline Description
    • Pipeline Key - Unique identifier for the pipeline, optional and recommended, made up of lowercase letters, numbers, '_', and '-'
  3. Click Save. 
  4. You go to Pipeline Settings. The list of Steps is empty. 

3. Create Steps and Database Connections

In Projects > Project > Pipelines > Pipeline > Settings

  1. Click Create Step. You are prompted to create a connection for the step. 
  2. Click Create New Connection. This is required if you have not create a connection before. 
  3. Enter connection information
    • Connection Name (required)
    • Connection Key - Unique identifier for theconnection, optional and recommended, made up of lowercase letters, numbers, '_', and '-'
    • Connection Description
    • Labels - the name of the pipeline is automatically added as a label. 
    • Contexts
    • Default Schema
    • DB Platform (required, the database type). You may be shown additional options for the selected database type. 
    • Hostname (required)
    • Port
    • Database Name (may be required, depending on database type)
    • Username
    • Password
  4. Click Save to continue entering step information. 
  5. Enter step information.
    • Connection (required) - select a connection. 
    • Step Name (required)
    • Step Description
    • Step Type (required, one of DEV, QA, PROD, or uncategorized)
    • Step Key - Unique identifier for thestep, optional and recommended, made up of lowercase letters, numbers, '_', and '-'
    • Enable Row Count
    • Include Table and Index Storage Information
    • Script Execution Timeout - select to enable a timeout limit on executing SQL scripts.
    • Number of seconds - enter the timeout limit, in seconds. 

4. Create a Connection to the SCM Repository

In Projects > Project > Project Settings > SCM Settings

  1. In Project Settings, click the SCM Settings tab.

4a. Get a Jenkins Credential ID for the SCM Repository

For Credential ID, you need to associate login credentials with a string you choose (credential ID). Do the following:  

  1. Click the Jenkins link.The Jenkins administrator console opens in a new tab. 
  2. Click Credentials, then click System under it
  3. Click Global credentials (unrestricted)
  4. Click Add Credentials
    • Select Kind, one of the following depending on how a user logs in to your SCM system:
      • Username with password
      • SSH Username with private key
    • Provide credential information. Set the ID to a string. For example, you could use gitID<projname> to represent credentials to log in to Git for this project. 
    • Click OK.
  5. To verify, go to Credentials > System > Global credentials (unrestricted) to verify that the added credential exists.
  6. Click Update to view the credentials for each ID. 

4b. Complete SCM Connection Information

Go back to the Datical tab. Enter additional information. 

  • SCM Type (required)
  • URL (required)
  • Branch (required for Git only)
  • Project Root (required for TFS only) 
  • Select Create Directories to create standard directories in the repository

5. Assign Users and Groups to the Project

In Projects > Project > Project Settings > Users Settings

  1. In Project Settings, click the Users Settings tab. Project and step roles are listed in a table with the number of members assigned to each role.

  2. Click  the Project Administrators role. Select users and groups as desired, then click Close. 

    The project creator is initally assigned as a Project Administrator. Remove that user if desired. 

  3. Click the other roles as needed. Select users and groups as desired, then click Close. 

6. Add Deployment Settings

In Projects > Project > Project Settings > Deployment Settings

  1. In Project Settings, click the Deployment Settings tab. 
  2. Select values for the desired deployment behavior for this project.

  3. Click Save.

  • Forecast and deploy
    • Require options for forecast and deploy (labels and contexts)
    • Automatically generate SQL for deploy
    • Limit forecast profiling
    • Forecast data modification changes
  • Deployment Threshold (for Full Deploy)
    • Stop on error
    • Stop on warning
    • Always deploy (Not recommended)
  • Check Stored Logic Validity After Deployment (Oracle and SQL Server databases only)
    • Disabled
    • Local - Check the target schema only
    • Global - Check all schema in the database
  • Deployment Mode
    • Full - Perform rules validation and run a forecast before deploying. Deployment threshold applies.
    • Quick - Deploy wihout rules validation or forecast.
