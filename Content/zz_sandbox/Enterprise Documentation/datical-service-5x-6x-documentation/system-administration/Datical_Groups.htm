Datical Groups

The Groups page lists all groups. Users with System Administrator role can view and edit groups. 

The group names and roles shown are examples. You set up your own users and groups in Keycloak

  • To locate a group, open group folders and scroll the list. 
  • Click Open All and Close All to see and hide all subgroup folders in all groups. If a group has no subgroups, it does not have an open/close control. 
  • Click a group name to view and edit roles for the group. 

In the following example, the DBA group has one subgroup, Ops DBAs. The Devops and IT Admin groups do not have subgroups. 

Adding a Role to a Group

You typically assign roles to groups. You can also add roles to individual users. See Datical Users. 

  1. Select Admin > Groups to view the list of groups.
  2. If you have hierarchical groups, navigate to the group folders until you see the group you want.
  3. Click the group. The existing roles are shown. 
  4. Start with Choose Role. The next choices are highlighted as you go. 
    1. Choose Role - Choose the role to assign. 
    2. Choose Project - For project and step roles
    3. Choose Pipeline - For step roles
    4. Choose Step - For step roles
  5. Click Add Role.

Subgroups inherit roles assigned to the parent group. In the Role list, inherited roles include a note, "Assigned by Parent Group."
