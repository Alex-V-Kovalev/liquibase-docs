Configuring Datical Service to Use an Existing Audit Database

During installation, a PostgreSQL database is installed and configured as the audit database for Datical Service. 

If you are already using Datical DB and have configured projects to use an audit database, you need to configure Datical Service to use that database before importing the projects. 

Using the CLI to Configure the Audit Database to Use

See Database Requirements in Prerequisites for Datical Service 5.x/6.x Installation to review the supported database types. 

Run the following command. 

datical-control auditdb configure

You are prompted for the following information for Datical Service to use to access the audit database. 

Common to all database platforms:

  • Hostname
  • Port
  • IP address
  • User name
  • Password

You may also need to include fields that are specific to your database platform (such as service name for Oracle, and databaseName for SQL Server).  Here are example connection URLs for each database platform:

  • jdbc:postgresql://HOST:PORT/DATABASE
  • jdbc:oracle:thin:@//HOST:1521/SERVICE_NAME
  • jdbc:db2://HOST/DATABASE
  • jdbc:sqlserver://HOST;databaseName=DATABASE
