Administrative User Requirements

Creating an Administrative User

We recommend that you create a user for installing and administering Datical Service. 

Before starting the installation, create this user on the host where you will install Datical Service. You may use any user name. Example:

  • User name: datical
  • User directory: /home/datical

Note

You are prompted for the user name during installation. 

Root or Sudo Requirements

The user requires root privileges for installation. Some administrative commands also require root/sudo privileges. 

Note

Sites typically have their own policies about managing sudo privilege. Check with your site's root administrator.

Adding Sudo Privileges Through a Group 

Typically you must do one of the following:

Add the datical user to a group that has sudo privileges. The wheel group is commonly used (shown in the example), but a custom group may also be used.

sudo usermod -aG wheel datical

Adding Sudo Privileges by Adding the User to the sudoers Group

Add the datical user directly to the end of /etc/sudoers. Use a text editor. The visudo editor is provided with most Linux systems and is typically preferred, to guard against critical access errors. 

datical ALL=(ALL)


Test to make sure datical has sudo privileges. 

$ su - datical
Password: 
$ sudo su
#


If the addition fails, you get the following message:

datical is not in the sudoers file.  This incident will be reported.
$

Running CLI for Administration

When you run commands, run them as the administrative user. 

Note that some commands require root privileges.
