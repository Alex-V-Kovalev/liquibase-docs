Exporting a Project from Datical Service for Use with Datical DB

Once a project is imported to Datical Service, you manage it there. If you need to update the project used by Datical DB, export it. 

Important

Do not edit the Project Key for an imported project if you plan to re-export it. 

A project must have at least one pipeline. All pipelines must have at least one step. 


Use the following procedure to export a project. 

In DMC, open a project. 

  1. ClickProject Settings.
  2. ClickExport Project.  
  3. ClickYesto export now. If you have unsaved changes, click No and save them first. 
  4. Select the location of thedatical.projectfile, then clickSave. Do not rename the file. 

After you have exported the project file, use it to update the project as it is stored in your SCM system. The specifics vary by SCM system. 

Note that only the datical.project file is exported. Replace it in the hierarchy of project files. 

  1. Open your SCM client
  2. Go to the local directory for the repository where you have stored the Datical project. 
  3. Update your local copy (refresh it from the SCM server). 
  4. Copy the exported datical.project file and overwrite your local SCM copy with it. 
  5. Check in the datical.project file.  

Once it is checked in, automated Datical operations that check out the project from SCM start using it.
