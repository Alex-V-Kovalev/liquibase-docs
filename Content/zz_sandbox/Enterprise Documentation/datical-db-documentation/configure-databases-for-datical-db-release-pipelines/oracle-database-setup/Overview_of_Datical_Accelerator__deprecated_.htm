Overview of Datical Accelerator (deprecated)


Deprecation of the Oracle Accelerator JDBC Driver:

Because of a change in deployment trends and continued improvement of the performance of Datical DB, the Oracle Accelerator JDBC Driver (compression driver) was removed from our driver repository in versions 7.16 and higher.  It was previously available only with versions 7.15 and lower.


The Datical Accelerator for Oracle improved performance in some environments by compressing the results data produced by an Oracle database before sending them to Datical DB. Using the accelerator involved a tradeoff between host resource usage and network usage.  The accelerator/compression driver:

  • Required additional CPU and memory resources on the database host (for compressing) and the Datical DB host (for uncompressing).
  • Reduced time and network resources for sending the results data over the wire.

The accelerator was helpful in cases where the network connection is slow, such as when lower-speed networks are used or when the Oracle database and Datical DB application are far apart geographically. The performance depended on the results data generated. In some environments, performance could be improved by a factor of 10X or more.

In versions 7.15 or lower, you could install the following components which might improve performance in some environments:

  • Datical DB Driver for Oracle
  • Datical Accelerator, a Java class loaded into the Oracle JavaVM

Once the Datical Accelerator was enabled for a project or database, all Datical commands in that context used it.
