Troubleshooting the Accelerator (deprecated)


The accelerator/compression driver has been deprecated in version 7.16 and higher. It was previously available only with versions 7.15 and lower.

If the driver ran into any errors when loading, it would write the error messages out to the Datical GUI console window with versions 7.15 or lower. Please check the console for additional information that will help to resolve the problem.  The following is a list of the most common issues that could occur when setting up this feature with versions 7.15 or lower:


Java not installed on the database

  • Install the JavaVM as documented in the above Verify & Install Oracle Database JavaVM section.

Feature not licensed correctly

  • Install the Datical Accelerator for Oracle license

No Java permissions (but the JavaVM is installed)

  • Grant the missing permissions to the user account accessing the database.

Error installing the Stored Procedure (permissions or doesn't compile)

  • Grant the missing permissions to the user account accessing the database.
  • If the stored procedure doesn't compile, contact our Technical Support team for assistance.

DeployCommand failure due to cannot find database driver

  • With versions 7.16 or higher, if the accelerator is enabled, you may see these errors.  That is because the accelerator was deprecated and the driver is no longer included with versions 7.16 or higher.
