status

The status command lists all undeployed s. It also lists the id, author, and file path name for each undeployed . The status --verbose command does not modify the database.

Uses

The status --verbose command is typically used when s were added to a through source control by another developer. The command confirms what has been deployed and what s are pending per author and corresponding IDs.

Running the status --verbose command

To run the status --verbose command, specify the driver, classpath, and URL in the properties file. You can also specify these properties in your command line.

Run the status --verbose command:

liquibase --changelog-file=<postgres_lbpro_master_changelog.xml> status --verbose

where <postgres_lbpro_master_changelog.xml> represents the name of the .

Add the changelog-file to the properties file to save time and reduce errors.

status --verbose global attributes

Atrributes Definition Requirement
--changelog-file The root Required
--username The database username Required
--password The database password Required
--url The JDBC database connection Required
Output

When successful, the status --verbose command produces the following output:
