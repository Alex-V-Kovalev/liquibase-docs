liquibase-flow File and Command (For LABS only)

 

The liqubase-flow command and the flow file on which it operates moves Liquibase forward into a more robust, more convenient utility. Multiple steps are setup in another tool, such as a Jenkins file or GitHub actions, to perform the multiple steps is often tasked with in a ci/cd job. The flow command provides the convenience of performing all of the steps in one place. This leads to better repeatability and standardization of the steps in a DevOps pipeline.

You only need to use the liqubase-flow command to perform all actions specified in Jenkins, GitHub actions, or any CI/CD support tool of your choosing. Changes are made in the flow file when updates to Liquibase steps are necessary. You only have to update them in the flow file to bring all jobs up to date. This is especially convenient when flow files are stored in remote locations or shared across teams.
