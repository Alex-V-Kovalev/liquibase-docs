Which Installation Method is for Me?

Review these sections to help plan your installation of Datical Service. 

What Kind of Server Will I Provision?

Choose from these options. 

Your Host Environment
  • Your virtual machine - a VM that contains a supported operating system. You start it and install Datical in it. 
    • Advantages:
      • Allows you to use IT-required images
    • Disadvantages: 
      • Troubleshooting issues may be more difficult because Datical does not know what is or is not installed
        • May have unexpected firewall configurations, missing standard packages, auto-provisioning tools that undo changes we make, etc.
    • Requirements: You need to install OS dependencies needed by Datical Service as the first step in installation. 
  • Your physical server - a host machine running a supported operating system. You install Datical on it. 
    • Advantages:
      • Doesn't require a virtual server infrastructure.
    • Disadvantages:
      • Troubleshooting issues may be more difficult because Datical does not know what is or is not installed.
    • Requirements
      • You need to install OS dependencies needed by Datical Service as the first step in installation. 

See Installing Datical Service on a Server or VM for installation. 

Which Audit Database Do I Use? 

Datical Service now installs an application database and audit database by default. They are managed in a Docker image internally. No further configuration is needed. 

If you have an existing audit database and want to continue using it, you can configure Datical Service after installation. 

See Configuring Datical Service to Use an Existing Audit Database. 

SQL Server Managed Databases with Integrated Security

  • If one of your managed databases is SQL Server running with integrated security and you intend to execute Forecast and Deploy from Datical Service, then you need to install the Jenkins agent. See /wiki/spaces/DSV53/pages/724369471.
