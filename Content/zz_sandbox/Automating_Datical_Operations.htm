Automating Datical Operations

Although Datical operations can be performed manually, the best practice for DevOps environments is to run Datical operations from automation software. 

Typically a workflow in an automation solution includes steps that run Datical operations through the CLI. 

  • Status
  • Forecast
  • Deploy
  • Rollback
  • (Any Datical Command)

Packaging SQL Code into changesets is typically managed by automation in the SCM environment (a "hook" or "trigger" that runs the Deployment Package when changes are checked in). 

Configuring Automation Solutions to Run Datical Operations

An integration provides the mechanism (plug-in) for an automation solution to run Datical CLI commands. 

You typically need to do the following to set up automated Datical operations:

  1. Install the plug-in in your automation solution.
  2. Set global properties that store information about how to run the commands:
    • Location of the hammer command
    • Location of Datical database drivers
  3. Create steps in the automation solution to perform Datical operations. Each step runs one or more Datical Commands.
