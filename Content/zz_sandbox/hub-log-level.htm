hub-log-level

The --hub-log-level parameter specifies the log level for filtering log messages to send to Liquibase Hub during operations. The --hub-log-level can be set to the following values:

  • ERROR (highest level) – Show serious failures that may prevent program execution.
  • WARNING – Show potential problems for program execution.
  • INFO – Show informational messages.
  • FINE (lowest level; formerly DEBUG) – Show tracing information of program execution and minor failures.
  • OFF – Hide all log messages.

If you set --hub-log-level to a particular level, it will show all messages at that level and at higher levels of importance.

The --hub-log-level default value is OFF. However, if you use the --log-file to send your logs to a different file and don't specify the --hub-log-level parameter, will automatically change the --hub-log-level default value from OFF to ERROR.

Uses

The --hub-log-level parameter applies to all commands. It can be used to get more information about the actions that commands perform, as well as provide helpful information to resolve errors.

The value of SEVERE provides messages regarding serious errors. You may find it useful to set the --hub-log-level to INFO to see a more complete list of the operations that a command performs. Use the FINE level to help resolve errors.

Using the --hub-log-level parameter

The --hub-log-level is an optional parameter that is either set in the properties file or specified on the command line. The --hub-log-level specified on the command line overrides the value set in the properties file.

  1. Define environment variables:
    2. LIQUIBASE_HUB_LOG_LEVEL=INFO
  2. Command line argument:
    3. liquibase --hub-log-level=INFO
  3. setting:
    4. liquibase.hub.logLevel=INFO and liquibase.hub.log-level=INFO

Examples:

liquibase --hub-log-level=FINE update
liquibase --hub-log-level=INFO update-testiliquing-rollback

Using the --sql-log-level property alongside --hub-log-level

The SQL log level is set to FINEby default. This behavior can be modified using --sql-log-level property.

the --log-level and sql-log-level values override the hub-log-level value.

For example, if you would like the log SQL statements to have the INFO log level, then you would specify --sql-log-level=INFO in your Liquibase command:

liquibase --sql-log-level=INFO --hub-log-level=INFO update

The --hub-log-level property is required for --sql-log-level to operate successfully.

Other ways to specify --sql-log-level

  1. Define the JAVA_OPTS Environment Variable prior to running your command:
    2. JAVA_OPTS="-Dliquibase.sql.logLevel=INFO -Dliquibase.logLevel=INFO"
  2. Define environment variables:
    3. LIQUIBASE_SQL_LOG_LEVEL=INFO and LIQUIBASE_LOG_LEVEL=INFO
  3. Command line argument:
    4. --sql-log-level=INFO --hub-log-level=INFO
  4. setting:
    5. liquibase.sql.logLevel=INFO and liquibase-logLevel=INFO
