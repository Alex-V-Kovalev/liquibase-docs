ObjectNameMustMatch

The ObjectNameMustMatch subcommand allows you to confirm the listed object names do not conform to the supplied pattern.

  • Subcommands require specific parameters noted in the table.
  • Customizations to checks configuration are stored in the liquibase.checks-settings file.

Uses

Use the subcommand to validate that object naming conventions adhered to and that object names match the supplied pattern. This ensures that youallows the user to ensure they have direct control over their object names and avoid naming issues.


Run the ObjectNameMustMatch subcommand

To run the ObjectNameMustMatch subcommand, ensure you have a license, a valid URL property that points to a live JDBC connection or a snapshot, and the checks-scope property specified as Database and/or Changelog in the Default Properties file. Then check if any searchable object types match the supplied pattern. Enable the ObjectNameMustMatch check by running the following command in the CLI: 

liquibase checks enable --check-name=ObjectNameMustMatch
Liquibase will then prompt the user to establish a copy of this check and to initiate the customization work flow. To do this, the user will be asked to select options for each of the attributes below in the CLI.

This check can not be enabled directly because one or more fields does not have a default value. Creating a copy of this check initiates the customization work flow.

The check is active. Once the attributes are configured, the check will successfully be enabled.

The location to look for the provided SearchString Value

- startsWith - SearchString provided should be found at the beginning of the tlable.

- endsWith - SearchString provided should be found at the end of the tlable.

- contains - SearchString Provided can be anywhere in the tlable.

- regexp - SearchString provided is a regexp that should be matched.
