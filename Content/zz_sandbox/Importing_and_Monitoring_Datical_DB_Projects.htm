Importing and Monitoring Datical DB Projects

In this scenario you have installed Datical Service and want to use it to monitor status of existing jobs that are running through Datical DB, typically through automation software. 

Determining Status for a Datical DB Pipeline

You interact differently with Datical Service to obtain status of a step or pipeline. 

Datical DB Datical Service

Status is derived directly from the managed databases. It is not updated automatically. 

When you run an operation, the GUI is not updated automatically. To get the current status:

  • Click Status on a step (managed database)  in the GUI.
  • Run status commands in the CLI.

Status is derived from the audit database. It is continuously updated. 

  • When you run an operation, the audit database is updated. 
  • Datical Service polls the audit database periodically and updates the status shown in DMC.

A Status button is provided but it is normally not necessary to use it, because Datical Service updates its own status after operations.  

See Status, Forecast, and Deploy Operations on Projects. 


Importing the Project to Datical Service

Datical Service uses the audit database to obtain status of Datical DB projects.  

These instructions assume that the project is configured to use the same audit database as Datical Service. 

  1. Important: Check that Datical Service is using the same audit database that the project uses.  
    • If you have decided to use a different audit database for Datical Service, edit the project to register with the Datical Service audit database before importing the project. 
  2. Import the project to Datical Service. See Importing an Existing Project.
    • DO NOT remove the project from its Datical DB location. 
  3. In DMC on Datical Service, open the project.

All information for the project is shown in DMC. 

Project Status

Open the project normally in DMC. 

Datical Service updates the status periodically.  

Report Availability

The project runs in its Datical DB context. Use Datical DB to view reports for the project. They are in <project-dir>/Reports or a location configured in the project. 

The Reports tab in DMC does not show the reports.
