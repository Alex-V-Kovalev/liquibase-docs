TableColumnLimit

The TableColumnLimit subcommand allows the user to ensure that no table described in a changelog or existing in a database target has more than a threshold number of columns.

  • Subcommands require specific parameters noted in the table.
  • Customizations to checks configuration are stored in the liquibase.checks-settings file.

Uses

Users may use the subcommand when they need to validate that the target tables do not contain more than the allowed number of columns. This gives users more control over the table column count which can be between 1 and 50.
