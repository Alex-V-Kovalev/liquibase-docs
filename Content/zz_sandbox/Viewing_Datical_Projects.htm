Viewing Datical Projects

Projects Overview

A project defines the workflow of database changes through release stages. A project typically maps to a defined scope of development work. 

  • A collection of applications that share the same database
  • An application
  • A part of an application

Project Settings

See Configuring Project Settings for information on individual settings shown in the project, pipeline, and step sections below.  

Project

A project is made up of one or more pipelines. Each pipeline typically represents a discrete development process. An example of how work may be subdivided:

  • Current Release
  • Next Release
  • Hotfix

The pipelines can be mapped to development work represented in branches in a Source Control Management repository. 

Viewing a Project

  1. Click Projects > Projects List



  2. Click a project.  The Project Summary is shown, including the list of pipelines in the project. For more information, navigate using the menus. 

Project Menu Page Sample

Projects List - Go back to the top level and view the list of projects

Project Summary - Summary view of deployment status for each step in each pipeline.

  • Click Status to update. 
  • Click a pipeline name to go to the Pipeline Summary page for it
  • Click a step (column head) to view step reports for that step. 
  • Choose a view for summary statistics: Labels or Changesets.

Statistics for each step are shown:

  • Undeployed - Number of labels that have not been deployed to that step
  • Success Rate - Percent of deployments that pass (PASS / TOTAL)
  • Velocity - Average time to move from the first step to a selected step.
  • Last Operation - Time and type of the last operation performed at that step (Deploy, Forecast). Click to see the report for that operation. 
  • Operations menu (if visible, requires the Step Deployer role) - Allows you to Forecast and Deploy at each step. 
    • If a Status operation is in progress, all users see a progress bar while it is running. Operations are not available. 
    • Operations menu on a step shows Deploying or Forecasting while an operation is in progress.

Project Reports - View the operations history and reports for the project. 

Each entry in the operations list identifies the Pipeline and Step for the operation.

Click a step to show the list of operations for just that step. 

  • Search the list by the following criteria:
    • Changesets
    • Labels
    • Contexts
    • Result Type (Pass, Warning, Fail)
    • Report Type
    • Start Date
    • End Date
  • View reports under Report Type
    • Forecast
    • Deploy
    • Package
  • More Files (click menu to choose)
    • Output Log
    • Deploy Script (if specified in Deployment Options)
    • Rollback Script (if specified in Deployment Options)

Pipeline

A pipeline is made up of steps.  

Viewing a Pipeline

  1. Click Projects > Project List
  2. Click a project.  The Project Summary is shown, including the list of pipelines in the project. For more information, navigate using the menus on the project and the pipelines. 
  3. Click one of the Pipeline names to go to the Pipeline Summary. 
Pipeline Menu Sample Page

Pipeline Status - View labels, changesets, step information, status, and deployment velocity for each step.

  • Labels view - The status of each label is shown at each step (column). 
    • Filter by Label
    • Use the Status filter in the step column head to filter labels by status (All, Deployed, Undeployed, Ignored)
    • Click a label to view all changesets for that label
  • Changesets view - The status of each changeset is shown at each step (column).
    • Filter by Label, Changesets, and Contexts
    • Click a label to view all changesets for that label
    • Click Labels to switch views
  • Status filter: in either view, click the Status menu, then choose what to view (All, Deployed, Undeployed, Ignored)

Step

Each step is aligned with a stage in a release process. An example release process:

REF > DEV > QA > PROD

  • REF - for Datical use, minimal representation of database schema and data. Used in packaging. 
  • DEV - Development environment
  • QA - Test environment
  • PROD - Production (live environment). 

Each step uses a connection to a database. 

Connections can be re-used across steps. 

Viewing a Step

  1. Click Projects > Project List
  2. Click a project.  
  3. Click a pipeline.
  4. Click Pipeline Settings. 
Step Menu Sample Page

Pipeline Settings - Click a step in the list to view and edit. Click Create Step to add a step to the pipeline. 

  • Click the Actions menu to choose the following:
    • Delete the step
    • Move the step (change step ordering)
    • Go to Step Users for the step. 
  • Connection Information - infomation on the database connection used by this step.


Connections

  • Click Create New Connection to add a connection to the list of connections available for steps.
