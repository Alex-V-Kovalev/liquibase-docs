Using Datical Service 5.x/6.x to Monitor Datical DB Operations

If you cannot immediately migrate to Datical Service completely, you can use Datical Service to monitor your existing release pipelines (typically operations built into automation software). 

Importing and Maintaining Projects in Datical Service 5.x/6.x

  1. Import the project to Datical Service
  2. Maintain the project in Datical Service (if you need to edit the project).
  3. Export any changes to re-integrate them into the existing deployment pipeline. Typically you export to a datical.project file and check it in to your SCM system. 

Requirement for Projects to Use the Datical Service Audit Database

Projects must be configured to use the audit database that Datical Service uses before you import them. See Database Requirements in Prerequisites for Datical Service 5.x/6.x Installation. 

Depending on how you have configured Datical DB projects, you may need to reconfigure them before importing them. 

  • You used an audit database for your Datical DB projects and configured Datical Service to use that audit database. 
    • You can import Datical DB projects to Datical Service immediately. 
  • You never used an audit database for your Datical DB projects. You installed Datical Service and configured it to use an audit database, as required. 
    1. Configure the Datical DB projects to use the Datical Service audit database.
    2. Run a Status operation from Datical DB to get records into the audit database. 
    3. Import the project.
  • You used an audit database for your Datical DB projects but configured Datical Service to use a different audit database (not recommended).
    1. Configure Datical DB projects to use the Datical Service audit database.
    2. Run a Status operation from Datical DB to get records into the audit database. 
    3. Import the project. 

Known Side Effects

The import and export process may result in changes in the following areas. They may require additional work on the exported datical.project file before integrating it back into your release pipeline. 

We plan to address these side effects in a future release. 

Database Credentials

Datical Service does not provide for storing database credentials in the project. All managed projects use dynamic credentials. Datical operations can then get credentials from the following sources:

  • System environment variable
  • User prompt when an operation runs
Condition In a single pipeline, you have steps that use stored credentials AND steps that use dynamic credentials.
Result After export, all steps are configured to use dynamic credentials. 

Release Labels

Datical Service does not provide the Release Label capability for projects. 

Condition You used the Datical DB GUI to assign release labels to pipelines in a project. You provide a label name and a description. 
Result The release label names are represented in the exported project file, but the descriptions are not. 

Enable Row Count During Forecast Option

Datical Service does not import or export the Enable Row Count option.

In Datical DB, you can set the following deployment option:

  • GUI (Forecast/Deploy section):
    • Collect row counts for all tables during Forecast
  • CLI: set enableRowCount to true. 

In Datical Service, you can create a project and set the following option:

  • Enable Row Count

The forecast operation uses the row count to calculate the impact of the changes. For example, if you drop a column and there is data for that column in five of 100 rows in the table, the forecast report tells you that. 

The default value is true. For very large tables, you may turn it off to improve performance for forecast operations. 

Condition You set the option to False in Datical DB.  
Result For the exported project, the option is not set. Therefore, forecast operations use the default behavior of calculating and reporting row counts. 

Status Unknown for a Step After Running changeLogSync

When you run changeLogSync, a step status should be Fully Deployed (green). Instead it is grey. 

Condition You run changeLogSync on a step. 
Result In DMC, the step status is grey (unknown) rather than green (Fully Deployed)

Workaround: Click Status on the project. The status is updated correctly. 

Oracle Accelerator on a Step 

Configuration for Oracle Accelerator on a step results in it being set for all steps after export and re-import into Datical DB.  Note that the Oracle Accelerator is only available with versions 7.51 or lower.

Condition A project has a step enabled for Oracle Accelerator
Result After export and re-import to Datical DB, all steps in the project are enabled for Oracle Accelerator
