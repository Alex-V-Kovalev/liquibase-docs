Choosing Stored or Runtime Database Credentials

Stored Credentials

By default, Datical DB stores your database connection credentials (usernames and passwords) in the project. 

  • User names are stored as-is (clear text).
  • Passwords are encoded.

In the datical.project file, the dbdef statement for each step contains the credentials required for the database connection. 

During operations, the user name and password are retrieved whenever database access is needed. 

Runtime Credentials

If you do not want to store the database credentials, configure the project to use runtime credentials.

  • GUI: In the Settings tab, Credential Managment section, check Specify database credentials at operation runtime.



  • CLI: Use the runtimeCredentials option. 

    hammer set runTimeCredentials true


Important

If you have previously set passwords in the DbDefs (steps) for the project, they are removed when you set the option to use runtime credentials.  If you are using the GUI, you are warned when you turn this options off. 


Runtime Database Credentials and the Audit Database

When you register a project with an audit database, the runtime credentials setting on the project applies to connections to the audit database. 

When Datical DB performs an operation, it writes records to the audit database. Credentials are used every time the audit database is accessed. 

Set the following environment variables:

  • DDB_AUDIT_USER
  • DDB_AUDIT_PASS

If the environment variables are not set, Datical DB prompts the user with a dialog to provide them every time an operation accesses the audit database.
