Datical DB v2019.2.2 - Release Notes

About Releases

  • Releases contain new features, bug fixes, and their own documentation space. 
  • Maintenance releases contain bug fixes. They are released between full releases. The documentation space for the last full release is used as documentation for the maintenance releases. For example, the documentation space for full release v2019.2 is used for v2019.2 and maintenance releases going forward.  

Go to https://software.datical.com to download files.

Datical GUI Client

The Datical GUI Client has been updated to use a more recent version of Eclipse that provides better functionality.

Note

When you move from v5.12 or before to v2019.2.2 or later, you must perform a new installation of the Datical DB GUI client. 

An upgrade cannot install it. 

Use the special instructions for the transition: Required Preparation for Installing the GUI Client in v2019.2 and Later. 

The special instructions are required in order to view your existing projects in the new GUI client and to preserve the ability to revert to a prior version. 

Oracle: Support for Oracle on Amazon RDS

Amazon RDS is a cloud environment that includes support for its own versions of databases. 

Setup for Oracle on Amazon RDS is mostly the same as setup for Oracle databases. 

See the following documentation:

  • Oracle on Amazon RDS Setup - for a summary of exceptions where Datical DB requires different configuration to support Oracle on Amazon RDS.
  • Oracle Database Setup - for all documentation of features supported and setup requirements for Oracle databases. 

Oracle: Authenticating Database Connections Using LDAP

You can configure Datical to use LDAP authentication when connecting to Oracle databases. The DbDef is configured to point to a client-side location of the ldap.ora file. 

See LDAP Connection to an Oracle Database. 

Oracle: Changes to DbDef Configuration for Normal, SSL, and LDAP Connections

When you configure a DbDef to use the TNS Alias field, it has the following effects:

  • You no longer need to specify Hostname and Port. You use TNS Alias instead.
  • Connection configuration is simplified. The Use Oracle Wallet checkbox has been removed.  The TNS Alias field points to a directory that contains configuration files for both normal and SSL connections. 
    • Normal connections use the tnsnames.ora file. 
    • SSL connections use additional Oracle Wallet configuration files. 
    • LDAP connections use the ldap.ora file. 

Specifying TNS Alias in the DbDef and setting up configuration files to support it is sufficient. 

Existing connections that specify Use Oracle Wallet are still supported. 

Documentation has been revised to reflect the change. 

  • Configuring a normal connection to an Oracle database: 
  • SSL Connection to an Oracle Database Through Oracle Wallet
  • LDAP Connection to an Oracle Database

Oracle: New Tool to Check Oracle Connections

A new command-line script helps you test normal, SSL, and LDAP connections to Oracle databases. 

hammer groovy OraConnect.groovy LDAP_WALLET


It reports on the following:

  • Use of environment variables to locate configuration files
  • Location of configuration files
  • tnsping results when the connection is tested
  • JDBC driver URL, version, and connection test

See Testing Oracle Connections.

Update Site Changes

The structure of repositories has changed on the update site, update.datical.com. Instead of a single repository, there are now repositories for three components.

  • GUI
  • CLI
  • Third Party (contains database drivers and SCM plug-ins)

What to Do - GUI

Use the GUI to change the configuration of the installation repositories to use. 

  1. Remove the repositories you have currently
  2. Add the two new repositories (for GUI and thirdparty). Do not add the CLI repository. 

See Configuring Installation Repositories for instructions on adding the new repositories. 

Effects

If you do not change the repositories, then you will see an error when you select Help > Check for Updates in the Datical DB GUI. 

What to Do - CLI

When using the installDrivers command, you will need to use the new URL. In previous versions, the command was 

hammer installDrivers http://update.datical.com/drivers/updates

In this version, the command should be

hammer installDrivers http://update.datical.com/latest/thirdparty

Documentation: Expanded Coverage for Installation Use Cases

Installation documentation now discusses theme use cases (preferred) and variation use cases (variants) for both Windows and Linux platforms. 

The documented use cases include the use of the Composite Repository for file-based installation without an internet connection.
