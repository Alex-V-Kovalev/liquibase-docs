Database Client Requirements and Features

Oracle Client Requirements

Client requirements are the same for on-premise Oracle and Oracle on RDS. 

The Oracle client version must match the version of Oracle you are running. See the versions of Oracle supported at Hardware, Software, and Database Requirements.

The following example is for Oracle Database 11g. 

Windows

  1. For the packaging database only, install database utilities expdp and impdp from the following link: http://www.oracle.com/technetwork/database/enterprise-edition/downloads/112010-win64soft-094461.html
    1. Select Oracle Database 11g Release 2 Client (11.2.0.1.0) for Microsoft Windows (x64) or current version.
    2. Select Install Type "Custom"
      • Select Component Name "Oracle Database Utilities"
  2. Make sure expdp and impdp are installed and in the PATH. These are Oracle programs that are used to create a full export and perform a full restore.
    1. Note: When packager runs impdp on a remote database, the export file (.DMP) will reside on the remote database.

Linux

  1. For the packaging database only, install database utilities expdp and impdp from the following link:
    http://www.oracle.com/technetwork/database/enterprise-edition/downloads/112010-linx8664soft-100572.html
    1. Select Oracle Database 11gRelease 2 Client (11.2.0.1.0) for Linux x86-64 or current version.
    2. Download linux.x64_11gR2_client.zip
    3. Unzip linux.x64_11gR2_client.zip
      • Refer to included Oracle Database Client Installation Guides e10841.pdf and e10861.pdf
  2. Make sure expdp and impdp are installed and in the PATH. These are Oracle programs that are used to create a full export and perform a full restore.
    1. Note: When packager runs impdp on a remote database, the export file (.DMP) will reside on the remote database.

Other Connection Properties

When connecting to an Oracle database, fields are set that identify Datical DB. 

  • JDBC connection
    • v$session.program is set to "Datical Client"
  • SQL*Plus connection
    • v$session.client_info is set to "Datical Client"

This information may be useful in the context of database connection audits.

EDB Postgres Client Requirements

  • Datical uses the following client and Postgres Command Line Utilities:
    • edbplus
    • pg_dump
    • pg_restore
  • Ensure that the desired version of the above are installed and in the PATH
  • The edbplus client is not included in the standalone Postgres Command Line Utilities. However, both can be obtained through the EDB installer
  • The latest version of the Postgres Command Line Utilities is backwards compatible to EDB Postgres 9.6
  • If using EDB Postgres 9.5 or earlier, ensure your version of Postgres Command Line Utilities matches that of the database
  • If packaging runs and fails, a .DMP file is left on the client host. 

PostgreSQL Client Requirements

  • Datical uses the following Postgres Command Line Utilities:
    • psql
    • pg_dump
    • pg_restore
  • Ensure that the desired version of the above are installed and in the PATH
  • The latest version of the Postgres Command Line Utilities is backwards compatible to Postgres 9.6 and works with Postgres Community, Amazon RDS, & Amazon Aurora databases
  • If using Postgres 9.5 or earlier, ensure your version of Postgres Command Line Utilities matches that of the database

DB2 Client Requirements

  1. Make sure db2 is installed and in the PATH.
  2. Locate and Install DB2 utility "IBM Data Server Runtime Client" from the IBM Support area. See Client Packages.  
    1. Untar the download file: tar zxvf <tar_file>
    2. Run setup:
      1. cd <untar_dir>
      2. ./db2setup (may need to update OS libraries by running "yum update")
  3. Verify db2 client setup
    1. At the command prompt, run:
      db2 ?
    2. On, success
      Db2 will display the version number of the db2 client
    3. On, error
      SQL10007N Message "-1390" could not be retrieved. Reason code: "3"
      • Your db2 client is not setup correctly. To fix this error, run the db2icrt command as documented at https://www.ibm.com/support/knowledgecenter/en/SSEPGG_11.1.0/com.ibm.db2.luw.admin.cmd.doc/doc/r0002057.html

        DB2DIR/instance/db2icrt -s ese -u < user> <db2 instance>
