Accessing Remote Linux Systems

Accessing a remote system is typically a use case when you have installed Datical DB for use by an automation system. 

The remote system may not be directly accessible, so you need to access them through network methods.

Accessing a Linux Installation from other Linux Systems

Make sure the following environment variables are set:

  • DISPLAY environment variable is set properly

export DISPLAY=:0.0

  • X AUTHORITY environment variable is set properly
  • PATH includes correct java home (Java must be version 1.8.x)
  • PATH includes correct hammer home (<datical_install_dir>/repl/)

Accessing a Linux Installation from Windows Systems

In order to launch the Datical DB client in a Windows environment and connect to a Linux installation, you will need a few pieces in place to enable the connection.

  • Linux (server): Datical DB installed in Linux environment, may need to open ports (22 for ssh) or enable port forwarding rules and ensure X11 packages are installed
  • Windows (cient): X server client (i.e. Xming) and SSH client (for example, PuTTY)

To test the connection, do the following:

  1. Start the X server client (for example, Xming)
  2. Use an SSH client (for example, PuTTY) to connect to the Linux environment
  3. Set theDISPLAY variable (i.e. export DISPLAY=<your IP address>:0.0).
  4. Test connectivity using the xterm or xclock command to make sure it displays properly.

If the tests pass, go to the datical installation folder on Linux and launch the Datical DB client using the ./datical command.
