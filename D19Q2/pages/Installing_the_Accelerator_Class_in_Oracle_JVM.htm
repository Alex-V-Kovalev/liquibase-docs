Installing the Accelerator Class in Oracle JVM

You need to remove and reinstall the Oracle JVM for all databases where you want to use the Datical Accelerator. Typically this will be on the database where you run Convert SQL scripts.
For more detailed information, see the following resources:

  • http://stackoverflow.com/questions/11788194/how-to-install-jvm-in-oracle
  • Knowledge Mgmt Note ID 757771.1 How to Reload the JVM in 10.1.0.X and 10.2.0.X in Oracle Support (formerly metalink ). More detailed instructions and video are available there. This note also consists of several other Notes that resolve different issues.


Testing that Oracle JVM is Installed

Run the following statement:
select * from dba_registry where COMP_ID='JAVAVM';
If there are no results, then the Oracle JVM is not installed. Go to the installation step.

Removing the Existing Oracle JVM


  1. As sys user, run the following:

startup mount;
alter system set "_system_trig_enabled" = false scope=memory;
alter system enable restricted session;
alter database open;
@?/rdbms/admin/catnojav.sql
@?/xdk/admin/rmxml.sql
@?/javavm/install/rmjvm.sql
truncate table java$jvm$status;

  1. Restart the database.

Installing OracleJVM


  1. As sys user, run the following:

startup mount
alter system set "_system_trig_enabled" = false scope=memory;
alter database open;
@?/javavm/install/initjvm.sql
@?/xdk/admin/initxml.sql
@?/xdk/admin/xmlja.sql
@?/rdbms/admin/catjava.sql
shutdown immediate;

  1. Restart the database.
  2. Execute the following script and resolve any INVALID objects.

@?/rdbms/admin/utlrp.sql
The JVM should be installed and functional. You can check it using the following statement:
SQL> select comp_name, version, status from dba_registry

Verifying Oracle Permissions

The Accelerator loads a Java class into an Oracle database schema and executes it as part of its operation.
The required permissions are CREATE and EXECUTE permissions for PROCEDURES in order to load and execute the Java code.
