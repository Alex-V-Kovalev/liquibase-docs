Host System Requirements

Version and Resource Requirements

  • Systems and versions. The following systems are supported: 

    • Centos 7.4+

    • Red Hat Linux 7.4+

  • Hostname is discoverable from local machine
  • Hostname and static IP address is assigned
  • Hardware/VM:
    • Minimum: 4 cores and 16 GB RAM
  • Internet access (to get installation files)
  • Disk space for downloaded installation files. Allow space for the installed application components as follows.
    • Application server: 250 GB miniumum (in /opt) 
    • Audit database (if external): 20 GB

Network Block Requirement

Datical requires the following network block to not already be in use or reserved on the machine that it is being installed on:

10.32.0.0/12

If that block is not free, the installation preflight check fails. 

If the network block is already in use, you may still continue with the install. Updating the network block can be addressed as a post-install configuration activity.

See Changing the Weave Network Block. 


Package Dependencies and Pre-installation Check

Datical Service requires a set of packages in order to install. The pre-installation check script checks for them. 

RPM Management

Datical Service maintains and manages the RPMs that it requires. 

Pre-installation Check for RPMs

You need to have the RPMs in the list below installed on your system. 

The pre-install check script checks for them and provides a list of the ones that are missing. 

The requirements include:

  • Third-party packages and their dependencies
  • Operating-system packages that are optional parts of the standard distribution of the operating system

Cloud Hosting

Amazon Web Services

  1. Use the following settings when you launch a new instance from the AWS dashboard:
    • AMI Type - RedHat server
    • Instance Type - t2.xlarge
    • Storage - 250 GB (root partition)
    • Security Groups - add a rule for https
    • Launch - Chose the existing pem file to access the the instance via ssh.  After launch, rename the instance to a name that is meaningful to your installation. 

  2. Connect via ssh and run the following command to enable additional resources:

    sudo yum-config-manager --enable rhui-REGION-rhel-server-extras rhui-REGION-rhel-server-optional
  3. Use the instructions for Installing Datical Service on a Server or VM.
