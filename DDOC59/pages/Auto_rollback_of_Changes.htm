Auto-rollback of Changes

Many change types such as "createTable" and "addColumn" can be automatically rolled back. If a changeset contains only statements that fit into this category, then it can be rolled back automatically. 

Rollback operations (GUI and CLI) use a rollback script if you have provided one. If a rollback script is not available, the rollback operation checks all of the changes to determine if all of them can be rolled back automatically. If any change in the rollback operation cannot be performed automatically, the rollback operation fails with an error. 

See Auto-rollback Support by Database Object Change Type for a list of the changes you can roll back automatically.
