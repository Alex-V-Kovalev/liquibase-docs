Overview of Datical Accelerator


Note

Starting in Datical DB v5.5, this feature no longer requires a separate license. If your see errors or warnings that you Accelerator license is expired or expiring, you must first uninstall all your licenses, and then re-install the Datical DB Admin or Datical DB Deployer license. See uninstallLicense command and installLicense command for details.


The Datical Accelerator for Oracle improves performance by compressing the results data produced by an Oracle database before sending them to Datical DB. Using the accelerator involves a tradeoff between host resource usage and network usage.

  • Requires additional CPU and memory resources on the database host (for compressing) and the Datical DB host (for uncompressing).
  • Reduces time and network resources for sending the results data over the wire.

The accelerator can be helpful in cases where the network connection is slow, such as when lower-speed networks are used or when the Oracle database and Datical DB application are far apart geographically. The performance depends on the results data generated. Generally speaking, performance can be improved by a factor of 10X or more.

You install the following components to enable the performance improvement.

  • Datical DB Driver for Oracle
  • Datical Accelerator, a Java class loaded into the Oracle JavaVM

Once the Datical Accelerator is enabled for a project or database, all Datical commands in that context use it.
