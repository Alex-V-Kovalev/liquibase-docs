Overriding Java Use

Default Behavior

Datical installs and uses Java as described in Java Version Use. You may need to override the default Java used: 

  • During the Datical transition from Java 7 to Java 8.
  • To comply with IT or technical requirements to use a particular Java location or version. 

If you need Datical to use an installation of Java other than the default installation, do the following:

  • Install the version you require, if needed.  If you are switching between versions that are installed with Datical, you do not need to install Java again. 
  • Configure Datical to use the desired version. 

Note

If you use a different version, the major version must match the required Datical version.  


Configuring Datical to Override the Default

Two Datical components use Java.  

  • GUI: Reads <datical-install>/datical.ini when starting.
  • CLI: Reads the value of an environment variable, DDB_JRE. It does not use -vm or other JRE settings in <datical-install>/repl/hammer-bin.ini

Configuring the GUI

To override the Java location used by the GUI, use a text editor to add the -vm option to  <datical-install>/datical.ini.

  • It must be first option in the file and on its own line.
  • The java path needs to be on its own line immediately after it.

  • Restart the GUI if it is running so that it will read the INI file. 

    -vm
/usr/Java/jdk1.8.0_73/bin/java
[ the rest of the INI file ]

Configuring the CLI

Set the environment variable DDB_JRE to the path to the home directory of the Java to use. All hammer CLI commands use this setting. 

  • To use a jre directory installed with Datical (in <datical-install>), set DDB_JRE to the name of the directory. 
    • Java 7: jre

    • Java 8: jre8

      Note

      See Java Version Use for information on changes to the Java versions required. Datical is transitioning from Java 7 to Java 8 (v4.32 to v4.35).  

  • To use a directory outside of the datical-install directory, set DDB_JRE to an absolute path.
