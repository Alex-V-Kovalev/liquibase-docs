Roles and UI Visibility

Go to the Users page (Admin > Users) to assign roles to users, as described in Datical Users. 

Main Menu

Access to the left-side menu is constrained by role. 

Projects

Click Projects > Projects List to see the list of projects.  

You can view only the projects assigned to you. 

  • Users with only System Administrator role cannot view projects (unless assigned additional project roles). The System Administrator role includes the permissions to assign roles to any user.
  • Users with Project Administrator role can use settings pages to edit project, pipeline, and step settings, including user and group assignments. Settings pages are not visible to other roles. 
  • Users with Project Creator role can view and select New Project, Import a Project, and the Imported Projects list.   

My Account

All users can view this information. 

Click My Account to view

  • Identity information
  • Roles assigned
  • Groups assigned

You can also access My Account from the menu for your login name. 

Admin

Only users with System Administrator or Project Administrator role can see this menu item. 

Click Admin and choose from the following:

  • Users - View and edit users. Administrators can change the roles, groups, and projects assigned to a user. 
  • Groups - View and edit groups. Administrators can change the roles and users assigned to the group. 
  • Server Settings - View and edit server settings:
    • Server identity
    • Application database connection
    • Audit database connection
  • New Project - Create a new project. Publish imported projects. 
  • Import a Project -  Import a project from a datical.project file on your local file system. 

Documentation

  • PDF - Opens a PDF assembled from the Datical Documentation space. Depending on your setup, the PDF is opened in a browser or in Acrobat Viewer. 
  • Confluence - Link to the Datical Documentation space on Confluence. Use the credentials provided by Datical.
