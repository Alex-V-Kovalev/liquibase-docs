Viewing the Dashboard

When you log in, you see a dashboard of operations statistics for all projects where you have Project Viewer access. 

Operations Statistics Section

The Success Rate by Step Type graphs show the count of operation results by step type. 

Operations

An operation is one of the following:

  • Package - Create changesets from SQL scripts and deploy them to the REF database. 
  • Forecast - Simulate a deployment and report results, but do not deploy.
  • Deploy - Deploy changesets to a specified step. Normally you deploy to steps one at a time, in release-pipeline order. 

See /wiki/spaces/DDD/pages/538673329. 

Step Types

A graph appears for the following step types.

  • All - Sum of results for all step types
  • Dev - Results for operations in steps having Step Type=DEV 
  • QA - Results for operations in steps having Step Type=QA
  • Stage - Results for operations in steps having Step Type=STAGE
  • Prod - Results for operations in steps having Step Type=PROD
  • Uncategorized - Results for operations in steps where Step Type is not assigned (uncategorized). 

If there is no data for a step type, the graph is greyed. 

A release pipeline is made up of steps that are defined in the project. Your release pipeline may be made up of any number of steps. Each corresponds to a database to be managed.

Each step has a Step Type attribute. You determine how to group steps under each Step Type. 

If you imported a Datical DB project that does not have a Step Type, you can apply it on the Step Settings page. See Viewing Datical Projects. 

Operation Results

  • Pass (green) - The operation completed successfully, without warnings or errors.
  • Warn (yellow) - The operation completed successfully but with warnings. You need to examine the results and you may need to make adjustments. 
  • Fail (red) - The operation failed. You need to examine the results and make adjustments. 

Projects can be configured to stop and fail upon warning rather than on error during a deploy operation.  In that case, the status for operations that fail on warning is shown under Fail rather than Warn. See the Deployment Threshold project setting in Configuring Project Settings. 

Search Section

Use search to constrain the displayed results. Click Reset to clear the search constraints.

Search by one or more of the following fields. When you specify more than one, the search criteria are interpreted together (AND). The page remembers the search criteria for the last search you made in the current logon session. 

  • Project Name
  • Result Type - choose from Pass, Warning, Fail
  • Step Type (Dev, QA, Stage, Prod, Uncategorized)
  • Report Type - choose from Packager, Forecast, Deploy, Rollback, Sync
  • Time - choose from Last Day, Last 7 Days, Last 14 Days, Last 30 Days, Last 90 Days
  • Start Date
  • End Date

You can choose either Time or Start Date and End Date to specify a time range for the search. 

Details Section

The Details section is a list of operations and reports. It is similar to the Reports pages for projects, pipelines, and steps.


The following information is given for each operation listed:

  • Result Icon - Pass, Warn, Fail
  • Date/Time - Date and time when the operation completed
  • Step - Name of the step
  • Step Type (Dev, QA, Stage, Prod, Uncategorized)
  • Report Type  (Packager, Forecast, Deploy, Rollback, Sync). Note that Rollback and Sync operations do not produce reports. The entry in this column serves as an operation category for search. 
  • More Files - If the menu appears, use it to choose any of the files that are available: Output Log, Deploy Script, or Rollback Script.
