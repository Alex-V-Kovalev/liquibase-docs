EDB Postgres Database Setup


EDB Postgres Roles and Permissions for Datical DB

The roles and permissions needed differ for managed databases and the REF database. 

Managed Database Permissions

The following example shows how to add permissions needed to a role. Assign the role to the desired database user.  

The database user (in Datical DB) needs to have access to the database but does not need to own it. 

GRANT USAGE ON SCHEMA my_schema TO DATICAL_ROLE;

<add list of all possible base schema permissions>

REF Database Permissions

The following example shows shows how to add permissions needed to a role. Assign the role to the desired database user.  

The REF database user in Datical DB also needs to be the the database owner. During packaging the database is backed up and restored, which requires owner access.  Alternatively you can assign superuser to the REF database user. 

GRANT USAGE ON SCHEMA my_schema TO DATICAL_PACKAGER_ROLE;

<add list of additional permissions needed for packaging>

Assigning Superuser to the REF Database User

You can assign superuser privileges to the user for Datical DB access rather than use the database owner. 

ALTER myuser WITH SUPERUSER
